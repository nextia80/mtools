# GitHub 업로드 및 Personal Access Token 가이드

## 1. 개요

mtools 프로젝트를 GitHub에 올리는 과정과, push 시 필요한 **Personal Access Token(PAT)** 발급 방법을 정리한다.

| 용어 | 설명 |
|------|------|
| **Git** | 소스 변경 이력을 관리하는 도구 |
| **GitHub** | Git 저장소를 클라우드에 올려두는 서비스 |
| **PAT (토큰)** | GitHub에 "내가 맞다"고 증명하는 일회용 비밀번호. `ghp_` 로 시작 |
| **origin** | GitHub 저장소 주소의 별칭 (remote 이름) |
| **push** | 로컬 커밋을 GitHub에 업로드 |

> **중요:** GitHub는 2021년부터 HTTPS push에 **계정 비밀번호를 사용하지 않는다.**  
> 반드시 **Personal Access Token** 을 Password 자리에 입력해야 한다.

## 3. Personal Access Token 만들기

### 3.1 토큰이란?

GitHub push/pull 시 로그인 대신 쓰는 **긴 비밀번호**다.

- `ghp_` 로 시작하는 문자열
- 화면에서 **한 번만** 표시됨 → 반드시 복사해 둘 것
- **MD 파일, Git, 채팅에 절대 저장하지 말 것**

### 3.2 토큰 발급 페이지

브라우저에서 GitHub 로그인 후 아래 주소로 이동:

- 목록: https://github.com/settings/tokens
- 바로 생성: https://github.com/settings/tokens/new

### 3.3 Note (이름)

토큰 구분용 메모. 아무 이름이나 가능.

| 추천 이름 | 의미 |
|-----------|------|
| `mtools` | mtools 프로젝트용 |
| `mtools-push` | push 전용 |
| `mac-push` | 맥 터미널 push용 |

### 3.4 Expiration (만료 기간)

| 설정 | 설명 |
|------|------|
| 30 days | 30일 후 만료 (기본값으로 충분) |
| 90 days | 더 길게 쓰고 싶을 때 |

만료되면 같은 방법으로 **새 토큰**을 다시 만든다.

### 3.5 Select scopes (권한)

**`repo` 하나만 체크**하면 push에 충분하다.

```
☑ repo    Full control of private repositories
```

아래 항목은 **체크하지 않아도 됨:**

- `workflow`
- `write:packages`
- `read:packages`
- 기타 scope

> **주의:** "Fine-grained token"이 아니라 **"Generate new token (classic)"** 을 선택한다.

### 3.6 생성

1. 페이지 맨 아래 **Generate token** 클릭
2. `ghp_...` 문자열 **복사**
3. 메모장 등 **로컬에만** 임시 보관 (Git/MD/채팅 금지)

## 4. push 하기 (다음 단계)

### 4.1 macOS 키체인 정리 (선택, 401 반복 시)

1. **키체인 접근** 앱 실행
2. `github.com` 검색
3. 예전에 저장된 GitHub 비밀번호 항목 **삭제**

### 4.2 Terminal.app에서 push

Cursor 터미널 대신 **macOS Terminal.app** 사용을 권장한다.

```bash
cd /Users/yujinseok/project/mtools
git push -u origin main
```

로그인 창이 뜨면:

| 항목 | 입력 |
|------|------|
| Username | `nextia80` |
| Password | **PAT 토큰** (`ghp_...`) — GitHub 비밀번호 아님 |

### 4.3 성공 확인

https://github.com/nextia80/mtools 에서 `mtools-api/`, `mtools-ui/` 폴더가 보이면 완료.

### 4.4 이후 변경사항 올리기

```bash
cd /Users/yujinseok/project/mtools
git add .
git commit -m "변경 내용 설명"
git push
```

## 5. 보안 주의사항

### 5.1 토큰 노출 시 조치

토큰이 **채팅, MD, Git, 스크린샷** 등에 노출되면 즉시 폐기한다.

1. https://github.com/settings/tokens 접속
2. 해당 토큰 옆 **Delete** 또는 **Revoke**
3. **새 토큰** 재발급

### 5.2 절대 하지 말 것

- PAT를 `.md`, `.env`, 소스 코드에 적기
- PAT를 GitHub에 push하기
- PAT를 채팅/메신저에 붙여넣기

### 5.3 `.gitignore`로 제외하는 파일

민감 정보 예시:

- `**/application-local.properties`
- `.env`, `.env.*`

## 6. 자주 나는 에러

| 에러 | 원인 | 해결 |
|------|------|------|
| `'origin' does not appear to be a git repository` | remote 이름 오타 (`origan`) | `git remote rename origan origin` |
| `401 Authentication failed` | PAT 없음 / 잘못된 비밀번호 | PAT 발급, Terminal.app에서 push |
| `No anonymous write access` | 로그인 안 됨 | Username + PAT 입력 |
| `git branch` 출력 없음 | 아직 커밋 없음 | `git status`로 `On branch main` 확인 |
