# GitHub push 명령어 정리

## 개요

mtools 변경 사항을 GitHub에 올릴 때 쓰는 **일상적인 git 명령**을 정리한다.

| 항목 | 값 |
|------|-----|
| 저장소 | https://github.com/nextia80/mtools.git |
| 브랜치 | `main` |
| remote 이름 | `origin` |

토큰 발급·401 오류 등은 [2026-06-17 GitHub 업로드 및 토큰 가이드](../2026-06-17/2026-06-17_08-30_GitHub_업로드_및_토큰_가이드.md) 참고.

---


## 3. 파일 추가 (`git add`)

### 전부 올리기

```bash
git add .
```

### 코드 + 문서만 (MD 개발 일지 제외)

```bash
git add README.md INSTALL.md mtools-api/ mtools-ui/
```

### 문서만

```bash
git add README.md INSTALL.md mtools-ui/README.md mtools-ui/UI.md mtools-api/API.md
```

### UI 변경 + 문서 + 개발 일지

```bash
git add README.md INSTALL.md mtools-ui/ MD/2026-06-19/
```

### 올리면 안 되는 것 (`.gitignore` 처리됨)

| 파일 | 이유 |
|------|------|
| `**/application-local.properties` | Google OAuth secret |
| `.env`, `.env.*` | 환경 변수 |
| `node_modules/` | npm 패키지 |
| `build/`, `dist/` | 빌드 결과물 |

> **PAT(토큰)** 도 MD·Git·채팅에 넣지 않는다.

## 4. 커밋

```bash
git commit -m "$(cat <<'EOF'
변경 내용을 한 줄로 요약

EOF
)"
```

### 커밋 메시지 예시

| 변경 | 메시지 예 |
|------|-----------|
| gc 기능 | `gc 명령어 및 캘린더 기능 개발` |
| 사이드바 접기 | `좌측 메뉴 접기 및 단축키 추가` |
| GitHub 문서 | `GitHub 설치/UI 문서 README 및 INSTALL 추가` |
| UI 문서 | `UI 설치·구조 문서 UI.md 추가` |

---

## 5. push

```bash
git push origin main
```

처음 push 또는 upstream 미설정 시:

```bash
git push -u origin main
```

로그인 창이 뜨면:

| 항목 | 입력 |
|------|------|
| Username | GitHub 아이디 (`nextia80`) |
| Password | **PAT 토큰** (`ghp_...`) — GitHub 비밀번호 아님 |

## 7. push 후 확인

1. https://github.com/nextia80/mtools 접속
2. 커밋·파일이 반영됐는지 확인
3. `README.md`가 저장소 메인 페이지에 표시되는지 확인

---

## 8. 자주 나는 문제

| 증상 | 해결 |
|------|------|
| `401 Authentication failed` | PAT를 Password 칸에 입력. [토큰 가이드](../2026-06-17/2026-06-17_08-30_GitHub_업로드_및_토큰_가이드.md) 참고 |
| `'origin' does not appear to be a git repository` | `git remote -v` 확인. 없으면 `git remote add origin https://github.com/nextia80/mtools.git` |
| Cursor 터미널 401 반복 | macOS **Terminal.app**에서 push |
| `nothing to commit` | `git status`로 변경 없음 — 이미 커밋됨 |
| push 거부 (rejected) | 원격에 새 커밋 있음 → `git pull origin main` 후 다시 push |

## 9. remote 확인

```bash
git remote -v
git branch -vv
```

정상 출력 예:

```text
origin  https://github.com/nextia80/mtools.git (fetch)
origin  https://github.com/nextia80/mtools.git (push)
* main  xxxxxxx [origin/main] 최근 커밋 메시지
```
