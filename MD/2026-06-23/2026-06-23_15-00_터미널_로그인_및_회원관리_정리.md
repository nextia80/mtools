# 터미널 로그인 및 회원관리 정리

## 개요

mTools 터미널에 **회원 로그인/로그아웃** 흐름을 추가하고, 로그인 상태에 따라 **좌측 메뉴·명령어 접근**을 제어한다. 회원 CRUD API·UI와 함께 `t_member` 기반 인증을 사용한다.

---

## 1. 로그인 명령

| 명령 | 동작 |
|------|------|
| `@@login` | 로그인 시작 → `(@@login) \|` 에서 아이디 → `(@@login pw) \|` 에서 비밀번호 |
| `@@exit` | 로그아웃 (세션·localStorage 제거, 환영 화면 복귀) |

- `cd login` / `cd exit` 는 **`@@login` / `@@exit`** 으로 변경됨
- UI 안내 문구에는 **`@@login`을 노출하지 않음** (명령은 직접 입력)

### 로그인 성공

```
━━━━━━━━ (CSS 구분선)
[MTOOLS ASCII 로고]
TERMINAL WEB SYSTEM
━━━━━━━━
** mTools 시스템에 접속하셨습니다.
➜
```

### 로그인 실패

- 상단 **환영 영역**에 아래 내용 표시 (명령 기록은 초기화)

```
정보가 올바르지 않습니다.

mTools에 오신것을 환영합니다.
사용을 하시려면 로그인을 하셔야 합니다.
```

### 비밀번호 표시

- `(@@login pw) \|` 입력 중: `type="password"` (●● 표시)
- Enter 후 기록: `**` 로만 남김

---

## 2. 비로그인 시 허용 명령

| 허용 | 차단 |
|------|------|
| `cls` | `?`, 도움말, `cd`, `bd`/`gc`/`gm`/`md`, `<`/`>` 등 |
| `@@login` (및 로그인 입력 단계) | |

차단 시 메시지:

> **사용을 하시려면 로그인을 하셔야 합니다.**

---

## 3. 좌측 메뉴·사이드바

| 항목 | 조건 |
|------|------|
| 메뉴 **1** (터미널) | 항상 표시 |
| 메뉴 **2~8** | 로그인 + **레벨 ≤ 1** (0=admin, 1=일반) |
| 사이드바 기본 상태 | **접힘** (`sidebarCollapsed = true`) |
| 로고 하단 사용자 정보 | 로그인 시 이름·아이디·레벨 표시 |

세션은 `localStorage` 키 `mtools-current-member` 에 저장 (새로고침 유지).

---

## 4. 환영 배너 (비로그인)

```
━━━━━━━━
[MTOOLS 블록체 ASCII 로고]
TERMINAL WEB SYSTEM
━━━━━━━━

mTools에 오신것을 환영합니다.
사용을 하시려면 로그인을 하셔야 합니다.
━━━━━━━━
```

- 구분선: 긴 `━` 문자 대신 **CSS `<hr>`** (가로 스크롤바·흰 네모 박스 방지)
- 로고: 초기 **블록체 MTOOLS** ASCII 유지

## 5. 백엔드 — 로그인 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/api/members/login` | 아이디·비밀번호 검증 |

**요청**

```json
{ "memberId": "nextia80", "password": "123456" }
```

**응답** — `Member` (비밀번호 미포함)

**DB 조건** — `t_member` 에서 `st_member`, `st_password`, `yn_use = 'Y'` 일치

### 시드 계정 (예시)

| 아이디 | 이름 | 레벨 | 비밀번호 |
|--------|------|------|----------|
| `admin` | 관리자 | 0 | `123456` |
| `nextia80` | 유진석 | 1 | `123456` |

> 비밀번호는 현재 **평문** 저장. 운영 환경에서는 해싱 권장.

### 기존 회원 CRUD

| 메서드 | 경로 |
|--------|------|
| `GET` | `/api/members` |
| `POST` | `/api/members` |
| `PUT` | `/api/members/{idMember}` |
| `DELETE` | `/api/members/{idMember}` (소프트 삭제) |

## 6. 주요 소스

### 터미널 (mtools-ui)

| 파일 | 역할 |
|------|------|
| `terminal/commands/loginCommand.ts` | `@@login`/`@@exit`, API 호출, 환영/실패 메시지 |

| `terminal/index.ts` | 명령 디스패치, 비로그인 차단 |
| `terminal/context.ts` | `@@login` 컨텍스트 입력 해석 |
| `components/TerminalView.vue` | 비밀번호 마스킹, welcome 영역 렌더 |
| `components/SidebarMenu.vue` | 메뉴 2~8 조건부 표시, 사용자 정보 |
| `components/MemberView.vue` | 회원관리 UI |
| `App.vue` | `currentMember`, `canAccessMenus`, 세션 복원 |

### API (mtools-api)

| 파일 | 역할 |
|------|------|
| `member/controller/MemberController.java` | `/api/members/login` 포함 |
| `member/service/MemberService.java` | `login()` |
| `member/dao/MemberDao.java` | `findByLogin` |
| `mapper/member/MemberMapper.xml` | 로그인 SQL |

## 7. 로그인 흐름 (요약)

```
접속 → 비로그인 환영 배너
  → @@login
  → (@@login) | {아이디}
  → (@@login pw) | {비밀번호}
  → 성공: 배너 갱신 + 메뉴 2~8 표시 + bd/gc/gm/md 사용 가능
  → 실패: 상단 환영 영역에 오류 + 안내 재표시
  → @@exit: 로그아웃
```

## 8. API 재기동

로그인 API·Mapper 추가 후 **mtools-api 재기동** 필요.

```bash
cd mtools-api && ./gradlew bootRun
```
