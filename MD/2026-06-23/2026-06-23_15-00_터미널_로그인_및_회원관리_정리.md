# 터미널 로그인 및 회원관리 정리

## 개요

mTools 터미널에 **회원 로그인/로그아웃** 흐름을 추가하고, 로그인 상태에 따라 **좌측 메뉴·명령어 접근**을 제어한다. 회원 CRUD API·UI와 함께 `t_member` 기반 인증을 사용한다.

---

## 1. 로그인·로그아웃 명령

| 명령 | 동작 |
|------|------|
| `@login` | 단계별 로그인 시작 |
| `@login -connect 아이디@비밀번호` | 한 줄 로그인 |
| `@exit` | 로그아웃 (세션 제거, 게스트 화면 복귀) |

> 예전 `@@login` / `@@exit` / `cd login` 형식은 **`@login` / `@exit`** 로 통일됨.

### 단계별 로그인 (`@login`)

```
$ @login
(id) | admin
(password) | ******
```

- 프롬프트 기호: **`$`** (기존 `➜` 대체)
- 내부 컨텍스트는 `@login`, `@login pw` 이지만 **화면 라벨은 `id`, `password`**
- 로그인 단계 줄 간격: **0.15rem** (일반 명령 0.65rem보다 좁음)

### 한 줄 로그인 (`@login -connect`)

```
$ @login -connect admin@123456
```

- 기록에는 비밀번호 마스킹: `@login -connect admin@**`

### 로그인 성공

```
━━━━━━━━ (CSS 구분선)
[MTOOLS ASCII 로고]
TERMINAL WEB SYSTEM
━━━━━━━━
** mTools 시스템에 접속하셨습니다.
$
```

- **로그인 후에만** ASCII 로고·배너 표시
- 터미널 하단 statusbar: 이름 · 레벨 · 접속 시각

### 로그인 실패

명령 기록은 초기화하고, 상단 환영 영역에 아래 내용 표시:

```
정보가 올바르지 않습니다.

* mTools Terminal System ver 1.0에 접속하였습니다.
* 메뉴를 사용하시려면 로그인을 해주시기 바랍니다.
```

### 비밀번호 표시

| 상황 | 처리 |
|------|------|
| `(password) \|` 입력 중 | `type="password"` (●● 표시) |
| Enter 후 기록 | `**` 로만 남김 |
| `-connect` 기록 | `아이디@**` |

## 2. 비로그인 환영 화면

비로그인 시 **로고 없음**. 구분선은 CSS `<hr>` (`bannerBar: true`).

### 직전 접속 기록 없음

```
---------
* mTools Terminal System ver 1.0에 접속하였습니다.
* 메뉴를 사용하시려면 로그인을 해주시기 바랍니다.
---------
$
```

### 직전 접속 기록 있음 (`@exit` 후)

```
* 직전 접속기록 : 1시간 10분
---------
* mTools Terminal System ver 1.0에 접속하였습니다.
* 메뉴를 사용하시려면 로그인을 해주시기 바랍니다.
---------
$
```

### 접속 시간 표시 규칙

| 조건 | 표시 |
|------|------|
| 1분 미만 (0일 0시간 0분) | **표시 안 함** |
| 10분 | `10분` |
| 1시간 | `1시간` |
| 1시간 10분 | `1시간 10분` |
| 1일 1시간 10분 | `1일 1시간 10분` |

- **0이 아닌 단위만** 붙여서 표시 (`0일 0시간 10분` → `10분`)
- localStorage에 예전 형식(`0일 0시간 0분`)이 남아 있으면 로드 시 **자동 삭제**
- `@exit` 시 계산 → `mtools-last-session-label` 키에 저장
- 다시 `@login` 성공 시 직전 접속 기록 **초기화**
- `cls` / 새로고침 후에도 유지

## 3. 비로그인 시 허용 명령

| 허용 | 차단 |
|------|------|
| `cls` | `?`, 도움말, `cd`, `bd`/`gc`/`gm`/`md`, `<`/`>` 등 |
| `@login` | |
| `@login -connect ...` | |

차단 시 메시지:

> **메뉴를 사용하시려면 로그인을 해주시기 바랍니다.**

## 4. 좌측 메뉴·사이드바

| 항목 | 조건 |
|------|------|
| 메뉴 **1** (터미널) | 항상 표시 |
| 메뉴 **2~8** | 로그인 + **레벨 ≤ 1** (0=admin, 1=일반) |
| 사이드바 기본 상태 | **접힘** (`sidebarCollapsed = true`) |
| 사이드바 사용자 정보 | **제거** (하단 statusbar로 이동) |

### localStorage 키

| 키 | 내용 |
|----|------|
| `mtools-current-member` | `{ member, loggedInAt }` — 로그인 세션 |
| `mtools-last-session-label` | 직전 접속 시간 문자열 (예: `1시간 10분`) |

## 5. 백엔드 — 로그인 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/api/members/login` | 아이디·비밀번호 검증 |

**요청**

```json
{ "memberId": "admin", "password": "123456" }
```

**응답** — `Member` (비밀번호 미포함)

**DB 조건** — `t_member` 에서 `st_member`, `st_password`, `yn_use = 'Y'` 일치

### 시드 계정 (예시)

| 아이디 | 이름 | 레벨 | 비밀번호 |
|--------|------|------|----------|
| `admin` | 관리자 | 0 | `123456` |
| `nextia80` | 유진석 | 1 | `123456` |

> 비밀번호는 현재 **평문** 저장. 운영 환경에서는 해싱 권장.

### 기존 회원 CRUD

| 메서드 | 경로 |
|--------|------|
| `GET` | `/api/members` |
| `POST` | `/api/members` |
| `PUT` | `/api/members/{idMember}` |
| `DELETE` | `/api/members/{idMember}` (소프트 삭제) |

## 7. 로그인 흐름 (요약)

```
접속 → 게스트 welcome (로고 없음, $ 프롬프트)
  → @login
  → $ @login
  → (id) | {아이디}
  → (password) | {비밀번호}
  → 성공: 로고 welcome + statusbar + 메뉴 2~8 + 명령 사용
  → 실패: 오류 + 게스트 welcome 재표시
  → @exit: 직전 접속기록 저장 + 게스트 welcome 복귀
```

### 한 줄 로그인

```
@login -connect admin@123456
  → 성공/실패 동일
```

## 8. 유틸 함수 (`textUtils.ts`)

| 함수 | 설명 |
|------|------|
| `formatSessionDurationLabel(startedAt)` | 접속 시작~종료 시간 → `1시간 10분` (0이면 `null`) |
| `normalizeLastSessionLabel(label)` | 저장된 문자열 정규화, `0일 0시간 0분` → `null` |
| `formatPromptContextLabel(context)` | `@login` → `id`, `@login pw` → `password` |
| `parseInlineLogin(input)` | `@login -connect id@pw` 파싱 |
| `maskLoginCommandForDisplay(...)` | 기록용 마스킹 |

## 9. API 재기동

```bash
cd mtools-api && ./gradlew bootRun
```
