# gm 터미널 UX 및 도움말 정리

## 개요

Gmail 터미널(`gm`) 사용성 개선과 도움말 명령 단순화를 반영했다.

---

## 1. 프롬프트 표시

상세 열람 시 메일 키를 **프롬프트 컨텍스트**에 포함한다.

| 이전 | 변경 후 |
|------|---------|
| `(gm r) \| gm r 0623-1` | `(gm r 0623-1) \|` |

- 입력창은 비우고 `/n`, `/d`, `/0` 등 서브커맨드만 입력
- `context: \`gm r ${ref}\`` — `inputDraft` 제거

---

## 2. 목록·열람 단축

| 명령 | 동작 |
|------|------|
| `(gm) \| r` | 조회된 목록 **첫 번째** 메일 열람 |
| `(gm) \| r <키>` | 지정 메일 열람 |

목록이 없으면 `gm 으로 목록을 먼저 조회하세요` 오류.

---

## 3. 본문 HTML 렌더링

터미널에서 `bodyHtml`이 있으면 HTML로 표시한다.

- 이미지(`https://` URL), 링크, 테이블 지원
- HTML 없으면 `bodyText` / `snippet` 텍스트 줄 출력
- `TerminalLineDraft.htmlContent` + `.terminal-mail-body-html` 스타일

> `cid:` 인라인 첨부 이미지는 미지원 (Gmail API 첨부 처리 필요).

---

## 4. 읽기 모드 서브커맨드

| 옵션 | 동작 |
|------|------|
| `/r` | 읽음 처리 |
| `/n` | 다음 메일 |
| `/b` | 이전 메일 |
| `/d` | 삭제 후 다음 |
| `/k` | 보관(INBOX 제거) 후 다음 |
| `/0` | **메일 맨 위**(제목·발신자)로 스크롤 |
| `/list` | 마지막 목록 조회 조건으로 복귀 |

`(gm r 0623-1) | /0` — `.terminal-output` 컨테이너 기준 수동 스크롤.

---

## 5. 도움말 명령 변경

| 이전 | 변경 후 |
|------|---------|
| `/?` | `?` (전체 목록) |
| `/? gc` | `? gc` |
| `(gc) \| -?` | `(gc) \| ?` |
| `gc -?` | `gc ?` |

---

## 6. 백엔드 — 보관 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/api/mail/messages/{id}/archive` | INBOX 라벨 제거(보관) |

`/k` 실행 시 호출. API 재기동 후 반영 필요.

## 8. 사용 예시

```text
cd gm
gm
r                    # 첫 메일
r 0623-1             # 또는 키 지정
/n                   # (gm r 0623-1) 에서 다음
/0                   # 메일 맨 위로 스크롤
/k                   # 보관
/list                # 목록 복귀
?                    # Gmail 도움말
```
