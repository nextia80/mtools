# gm 명령어 스펙 정리

## 개요

mtools 터미널에서 Gmail을 다루기 위한 `gm` 명령어를 정의·구현했다.

- 조회 정렬: **일자** 내림차순 (최신 우선)
- 메일 키: **`MMDD-순번`** (예: `0622-2` → 6월 22일 2번째 메일)
- 목록 출력: **키 · 제목** (`*` = 읽지 않음)
- 상세 열람: 프롬프트 `(gm r <키>)` — 입력창은 서브커맨드용

---

## 1. 목록 조회 (`gm`)

### 기본

```text
> gm
```

받은편지함 메일 **10건** (읽음 구분 없음, `-l a` 와 동일)

### `-l` 읽음 필터

| 값 | 설명 |
|----|------|
| `n` | 읽지 않은 메일 10건 |
| `r` | 읽은 메일 10건 |
| `a` | 전체 메일 10건 |

```text
gm -l n
gm -l r
gm -l a
```

### `-s` 일자 필터

| 종류 | 예시 |
|------|------|
| `yyyy` | `gm -s 2026` |
| `yyyy~yyyy` | `gm -s 2025~2026` |
| `yyyy.mm` | `gm -s 2026.06` |
| `yyyy.mm~yyyy.mm` | `gm -s 2026.01~2026.06` |
| `yyyy.mm.dd` | `gm -s 2026.06.22` |
| `yyyy.mm.dd~yyyy.mm.dd` | `gm -s 2026.06.01~2026.06.30` |
| `today` | 오늘 수신 메일 |
| `-week` | 오늘 기준 1주일 전 ~ 오늘 |
| `-month` | 오늘 기준 한 달 전 ~ 오늘 |

> `gc`와 달리 `week`, `month`(미래 방향)는 지원하지 않는다.

### `-k` 키워드

```text
gm -k "PT"
gm -s 2026.06 -k "회의"
```

Gmail 검색 `q` 파라미터로 서버에 전달한다.

### 출력 예

```text
메일 10건 (전체 45건) — gm r <키> 로 열람
0622-1    주간 보고 *
0622-2    PT 일정 안내
0621-1    뉴스레터
```

## 2. 상세 열람 (`gm r <키>`)

```text
gm r 0622-1
(gm) | r              # 목록 첫 번째 메일
(gm) | r 0622-1       # 키 지정
```

- 본문: `bodyHtml` 있으면 HTML(이미지 포함), 없으면 텍스트
- 열람 시 **자동 읽음 처리하지 않음** (`/r` 로 명시 처리)
- 프롬프트: `(gm r 0622-1) |` — 입력창은 `/n`, `/0` 등만 입력

### 액션 옵션

| 옵션 | 동작 |
|------|------|
| `/r` | 읽음 처리 |
| `/n` | 목록에서 다음 메일 (키 변경) |
| `/b` | 목록에서 이전 메일 |
| `/d` | 휴지통 이동 후 다음 메일 |
| `/k` | 보관(INBOX 제거) 후 다음 메일 |
| `/0` | 메일 맨 위(제목·발신자)로 스크롤 |
| `/list` | 마지막 목록 조회 조건으로 복귀 |

```text
(gm r 0622-1) | /r
(gm r 0622-1) | /n
(gm r 0622-1) | /0
(gm r 0622-1) | /d
(gm r 0622-1) | /list
```

다음/이전이 없으면 안내 메시지 후 `gm` 목록을 다시 표시한다.

---

## 3. 메일 키 (`MMDD-순번`)

### 형식

```text
{MMDD}-{순번}
```

같은 날짜 안에서 순번 부여 (목록 정렬 순서 기준).

### 클릭 연동

- 목록 행 클릭 → `r 0622-1` 입력 (`cd gm` 모드)
- `(gm r <키>)` 모드에서 `/n` 등 단독 입력 시 컨텍스트 키 자동 보완

---

## 4. 터미널 모드

| 명령 | 컨텍스트 |
|------|----------|
| `cd gm` | `(gm)` — `gm` 접두어 생략 |
| `cd /gm` | 상위 복귀 |
| `gm r <키>` | `(gm r <키>)` — 읽기 모드 |

---

## 5. 백엔드 API 확장

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/api/mail/messages` | `readFilter`(all/unread/read), `q`(Gmail 검색) 추가 |
| `GET` | `/api/mail/messages/{id}?markRead=false` | 터미널용 상세 (자동 읽음 생략) |
| `POST` | `/api/mail/messages/{id}/read` | 읽음 처리 |
| `POST` | `/api/mail/messages/{id}/archive` | 보관 처리 |

기존 `unreadOnly` 파라미터는 UI 호환을 위해 유지한다.

## 8. 제한 사항

1. **키 연도** — `MMDD`만 사용. 연말/연초 동일 월·일 메일은 키만으로 연도 구분 불가.
2. **10건 고정** — 목록은 항상 최대 10건 (Gmail API `maxResults`).
3. **받은편지함** — `in:inbox` 기준. 보관함·휴지통 목록은 미지원.
4. **인라인 이미지** — `cid:` 첨부 URL은 미지원. 외부 `https://` 이미지는 HTML 렌더링으로 표시.
5. **API 재시작** — 백엔드 변경 반영 시 `bootRun` 재기동 필요.
