# gc 명령어 스펙 재정의 정리

## 개요

mtools 터미널에서 Google Calendar를 다루기 위한 `gc` 명령어를 **플래그 기반**으로 재정의했다.

주요 변경:

1. **조회 기본값** — `gc` 단독 실행 시 **오늘** 일정
2. **옵션 키 통합** — `-l`, `-s`, `-c`, `-k` 조합으로 조회 범위·필터 지정
3. **일자 표기** — 점(`.`) 구분 (`2026.06.22`, `2026.01~2026.06` 등)
4. **일정 추가** — `gc a -t "<제목>"` + `-s` / `-c` / `-d` 옵션
5. **기간 종일·메모 생성** — 백엔드 `POST /api/calendar/events`에 `endDate`, `description` 지원 추가

조회 결과 정렬: **일자 → 카테고리(캘린더) → 종일 우선 → 시작 시각 → 제목**

---

## 1. 기본 규칙

| 항목 | 내용 |
|------|------|
| 기본 조회 | `gc` → 금일 모든 일정 |
| 옵션 조합 | `-s`, `-c`, `-k` 등 **중복 사용 가능** |
| `-s` 생략 | `-c`, `-k`만 써도 **오늘** 기준으로 조회 |
| 카테고리 | Google 캘린더 이름 앞 번호 (`0.(개인)특별한날` → 키 `0`) |
| 도움말 | `/? gc` 또는 `gc -?` |
| gc 모드 | `cd gc` 후 `gc` 접두어 생략 가능 (`week` → `gc -s week`) |

---

## 2. 조회 (`gc`)

### 2-1. 기본

```text
> gc
```

오늘 날짜의 전체 캘린더 일정을 반환한다.

### 2-2. 카테고리 목록 (`-l c`)

```text
> gc -l c
```

출력 예:

```text
0       0.(개인)특별한날
1       1.(개인)흔적
...
```

### 2-3. 일자 필터 (`-s`)

| 종류 | 설명 | 예시 명령 |
|------|------|-----------|
| `yyyy` | 해당 연 1/1 ~ 12/31 | `gc -s 2026` |
| `yyyy~yyyy` | 연도 범위 | `gc -s 2025~2026` |
| `yyyy.mm` | 해당 월 1일 ~ 말일 | `gc -s 2026.06` |
| `yyyy.mm~yyyy.mm` | 월 범위 | `gc -s 2026.01~2026.06` |
| `yyyy.mm.dd` | 특정일 | `gc -s 2026.06.22` |
| `yyyy.mm.dd~yyyy.mm.dd` | 일 범위 | `gc -s 2026.06.01~2026.06.30` |
| `today` | 오늘 | `gc -s today` |
| `week` | 오늘 ~ 7일 후 | `gc -s week` |
| `-week` | 7일 전 ~ 오늘 | `gc -s -week` |
| `month` | 오늘 ~ 한 달 후 | `gc -s month` |
| `-month` | 한 달 전 ~ 오늘 | `gc -s -month` |

일자 해석 예 (오늘이 2026-06-22인 경우):

```text
gc -s today     → 2026-06-22
gc -s week      → 2026-06-22 ~ 2026-06-28
gc -s -week     → 2026-06-15 ~ 2026-06-22
gc -s month     → 2026-06-22 ~ 2026-07-22
gc -s -month    → 2026-05-22 ~ 2026-06-22
```

### 2-4. 카테고리 필터 (`-c`)

```text
gc -c 5           # 5번 카테고리, 오늘 일정
gc -c 5,6,7       # 5·6·7번 카테고리
gc -s week -c 2   # 1주일 + 2번 카테고리
```

### 2-5. 키워드 검색 (`-k`)

제목·메모·캘린더 이름·장소를 LIKE 검색한다.

```text
gc -k "PT"
gc -s 2026 -k "PT"
gc -s week -c 1 -k "회의"
```

### 2-6. 조회 출력 형식

```text
[1-0622-1]  2026-06-22 종일 | 1.(개인)흔적 | 팀 미팅
    메모 내용 (있을 경우 들여쓰기)
[2-0622-1]  2026-06-22 09:00 | 2.(회사)업무 | 주간 회의
```

- 일정 행 클릭 → 입력창에 키(`1-0622-1`) 자동 입력
- 날짜가 바뀌면 출력 그룹 사이에 구분선

## 3. 일정 추가 (`gc a`)

`-t "<제목>"`은 **필수**이며 반드시 따옴표로 감싼다.

### 3-1. 기본

```text
gc a -t "테스트"
```

- 일자: **오늘**
- 시간: **종일**
- 카테고리: **목록 첫 번째**(키가 가장 작은 캘린더)

### 3-2. 일자 (`-s`, `-t`와 함께)

| 종류 | 설명 |
|------|------|
| `today` | 오늘 종일 |
| `yyyy.mm.dd` | 해당일 종일 |
| `yyyy.mm.dd~yyyy.mm.dd` | 기간 종일 (여러 날) |
| `yyyy.mm.dd:hh:mm` | 시작 시각, 기본 1시간 |
| `yyyy.mm.dd:hh:mm~:hh:mm` | 시작~종료 시각 |
| `yyyy.mm.dd:hh:mm+<n>h` | 시작 + 길이 (`+0.5h`, `+1h` 등) |

예시:

```text
gc a -t "휴가" -s 2026.01.01
gc a -t "연수" -s 2026.01.01~2026.01.30
gc a -t "미팅" -s 2026.01.01:09:00
gc a -t "미팅" -s 2026.01.01:09:00~:10:00
gc a -t "미팅" -s 2026.01.01:09:00+1h
gc a -t "메모" -s today
```

### 3-3. 카테고리 (`-c`, `-t`와 함께)

```text
gc a -t "테스트" -c 2
gc a -t "미팅" -s 2026.06.22:14:00+1h -c 2
```

### 3-4. 메모 (`-d`, `-t`와 함께)

```text
gc a -t "제목" -d "메모 내용"
gc a -t "제목" -c 2 -d "메모 내용"
```

추가 성공 메시지 예:

```text
일정 추가: 테스트 (2026-06-22 종일) → [1] 1.(개인)흔적
```

---

## 4. 일정 수정 (`gc m`)

```text
gc m <캘린더-MMDD-순번> -t "<제목>"
gc m <캘린더-MMDD-순번> -d "<메모>"
gc m <캘린더-MMDD-순번> -c <n>
```

- `-t`, `-d`, `-c` 중 **하나 이상** 필수
- **일자 변경(`-s`)은 지원하지 않음** (추가 명령과 다름)

예시:

```text
gc m 1-0622-2 -t "변경된 제목"
gc m 1-0622-2 -d "메모 수정"
gc m 1-0622-2 -c 3
```

---

## 5. 일정 삭제 (`gc r`)

```text
gc r <캘린더-MMDD-순번>
```

예: `gc r 1-0622-2`

키에 캘린더 번호가 포함되므로 `-c` 옵션은 없다.

---

## 6. 일정 식별 키 (`캘린더-MMDD-순번`)

### 형식

```text
{캘린더키}-{MMDD}-{순번}
```

예: `1-0622-2` → 1번 캘린더, 6월 22일, 그날 2번째 일정

### 순번 부여 (키 생성용)

같은 캘린더·같은 날짜 안에서:

1. 종일 일정 우선
2. 시작 시각 오름차순
3. 제목 오름차순

### 키 해석

1. 최근 `gc` 조회 결과 캐시(`lastListedEvents`)에서 조회
2. 없으면 키에서 캘린더·날짜를 파싱해 해당 일 API 재조회 후 순번 재계산
3. `MMDD`의 연도는 **현재 연도** 기준 (연말/연초 경계 제한)

## 7. 이전 스펙과의 차이

| 이전 (6/18 기준) | 현재 |
|------------------|------|
| `gc l` / `gc l <YYYYMMDD>` | `gc -s week`, `gc -s 2026.06.22` |
| `gc l --y 2026` | `gc -s 2026` |
| `gc c` | `gc -l c` |
| `gc a "<제목>" <날짜>` | `gc a -t "<제목>" -s <일자>` |
| `cd /gc/l` 하위 모드 | `cd gc` 모드에서 옵션 직접 입력 |

### 레거시 호환

아래 구형 명령은 내부에서 새 플래그로 변환되어 동작한다.

| 구형 | 변환 결과 |
|------|-----------|
| `gc c` | `gc -l c` |
| `gc l <일자>` | `gc -s <일자>` |
| `gc l -c 1,2` | `gc -c 1,2` (오늘 기준) |
| `gc l` 단독 | `gc`와 동일 (**오늘**) |

> 구 스펙에서 `gc l`은 1주일이었으나, 신규 스펙에서는 **`gc -s week`** 를 사용한다.

## 8. 구현 파일

### 프론트엔드 (`mtools-ui`)

| 파일 | 역할 |
|------|------|
| `src/terminal/commands/calendarCommand.ts` | gc 파싱·조회·추가·수정·삭제 전체 |
| `src/terminal/dateUtils.ts` | 점 구분 일자 파싱, week/month 범위 |
| `src/terminal/context.ts` | `cd gc` 모드에서 옵션 자동 보완 |
| `src/terminal/help.ts` | `/? gc` 도움말 |
| `src/components/TerminalView.vue` | 일정 행 클릭 → 키 입력 |

### 백엔드 (`mtools-api`)

| 파일 | 역할 |
|------|------|
| `GoogleCalendarService.java` | 조회·생성·수정·삭제, `endDate`/`description` 생성 |
| `CreateCalendarEventRequest.java` | `endDate`, `description` 필드 추가 |
| `CalendarController.java` | REST 엔드포인트 |

### API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/api/calendar/events` | 일정 조회 (`timeMin`, `timeMax`, `calendarKey`) |
| `POST` | `/api/calendar/events` | 일정 생성 (`date`, `endDate`, `time`, `description` 등) |
| `PATCH` | `/api/calendar/events/{eventId}` | 제목·메모·캘린더 이동 |
| `DELETE` | `/api/calendar/events/{eventId}?calendarKey=n` | 일정 삭제 |
| `GET` | `/api/calendar/calendars` | 카테고리 목록 |
| `GET` | `/api/calendar/status` | OAuth 연결 상태 |

## 9. 사용 예시 모음

```text
# 조회
gc
gc -l c
gc -s 2026
gc -s 2026.06
gc -s 2026.06.01~2026.06.30
gc -s week -c 1,2
gc -k "PT"
gc -s -month -k "병원"

# 추가
gc a -t "종일 일정"
gc a -t "회의" -s 2026.06.22:09:00+1h -c 2
gc a -t "연차" -s 2026.07.01~2026.07.05 -c 1 -d "가족 여행"

# 수정·삭제
gc m 1-0622-1 -t "제목 변경"
gc m 1-0622-1 -d "메모 추가"
gc r 1-0622-2
```

---

## 10. 알려진 제한 사항

1. **연도 경계** — 일정 키 `MMDD`는 현재 연도 기준. 전년/다음 해 일정은 키만으로 연도 구분 불가.
2. **수정 시 일자 변경** — `gc m`은 `-s` 미지원. 날짜 변경은 Google Calendar UI 또는 삭제 후 재추가.
3. **OAuth** — 일정 쓰기 권한 없으면 추가·수정·삭제 실패. 일정관리 메뉴에서 Google 연결 필요.
4. **백엔드 재시작** — `endDate`/`description` 생성 API 반영 시 `bootRun` 재기동 필요.

## 11. 서버 기동

```bash
# 백엔드 (8080)
cd mtools-api && ./gradlew bootRun

# 프론트엔드 (5173)
cd mtools-ui && npm run dev
```

- UI: `http://localhost:5173`
- API: `http://localhost:8080`
