# 메일관리 Gmail 연동 및 화면 UI 정리

일정관리 아래 **메일관리** 메뉴를 추가하고, Gmail OAuth 2.0으로 받은편지함 메일을 불러온 뒤 목록·상세·페이징·삭제·읽음 처리까지 구현했다.

터미널 명령어 연동은 추후 별도 설계 예정.

---

## 1. 메뉴 구조

```text
터미널(1) → 일자별LOG(2) → 일정관리(3) → 메일관리(4) → 게시판(5) → 회원관리(6) → API 스웨거(7) → API 테스트(8)
```

단축키: `Cmd+4` → 메일관리

| 파일 | 역할 |
|------|------|
| `SidebarMenu.vue` | 메일관리 메뉴 (아이콘: 4) |
| `MailView.vue` | 메일 목록 UI (필터, 페이징) |
| `MailDetailView.vue` | 메일 상세 UI (이전/다음/삭제) |
| `App.vue` | Gmail API 호출, OAuth 콜백, 목록·상세 상태 관리 |
| `types.ts` | `MailMessage`, `MailMessageDetail`, `MailMessagesPage` 등 |
| `app.css` | `.mail-*` 스타일 |

## 2. Google Cloud Console 설정

### 필수 작업

1. **Gmail API** 활성화 (API 및 서비스 → 라이브러리)
2. **OAuth 클라이언트 ID** (웹 애플리케이션) — 일정관리와 동일 클라이언트 사용 가능
3. **승인된 리디렉션 URI** 등록 (calendar URI와 함께 두 개 필요):

```text
http://localhost:8080/api/calendar/oauth/callback
http://localhost:8080/api/mail/oauth/callback
```

4. **Google 인증 플랫폼 → 브랜딩**: 앱 이름, 지원 이메일 설정
5. **Google 인증 플랫폼 → 대상**: 테스트 사용자에 본인 Gmail 추가 (Testing 상태)
6. **Google 인증 플랫폼 → 데이터 액세스**: scope 추가
   - `gmail.modify` (메일 읽기·읽음 처리·삭제)
   - `userinfo.email` (연결 계정 표시)

### scope 변경 후

삭제·읽음 처리에 `gmail.modify`가 필요하므로, 기존 `gmail.readonly`만으로 연결한 경우 **연결 해제 → 재연결** 필요.

## 3. 로컬 OAuth 설정

### application.properties

```properties
app.google.client-id=${GOOGLE_CLIENT_ID:}
app.google.client-secret=${GOOGLE_CLIENT_SECRET:}
app.google.redirect-uri=http://localhost:8080/api/calendar/oauth/callback
app.google.frontend-redirect-url=http://localhost:5173
app.google.mail-redirect-uri=http://localhost:8080/api/mail/oauth/callback
```

일정관리와 메일관리는 **별도 OAuth 연결**이다. 같은 Google 클라이언트 ID를 쓰되, 각 메뉴에서 따로 연결해야 한다.

## 4. OAuth 흐름

```text
[Gmail 연결] 클릭
    ↓
GET /api/mail/auth-url
    ↓
Google 로그인 · 권한 허용
    ↓
GET /api/mail/oauth/callback
    ↓
토큰 저장 (t_google_gmail_oauth)
    ↓
프론트 http://localhost:5173?view=mail&mail=connected
```

일정관리 OAuth 토큰(`t_google_oauth`)과 메일 OAuth 토큰(`t_google_gmail_oauth`)은 **테이블 분리**.

## 5. 백엔드 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/mail/auth-url` | Gmail OAuth URL |
| GET | `/api/mail/oauth/callback` | OAuth 콜백 |
| GET | `/api/mail/status` | 연결 상태 |
| GET | `/api/mail/messages` | 메일 목록 (페이징) |
| GET | `/api/mail/messages/{messageId}` | 메일 상세 (본문 포함, 열람 시 읽음 처리) |
| DELETE | `/api/mail/messages/{messageId}` | 메일 휴지통 이동 |
| DELETE | `/api/mail/disconnect` | Gmail 연결 해제 |

### 목록 API 쿼리 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `maxResults` | 10 | 페이지당 건수 (최대 100) |
| `unreadOnly` | false | true면 `is:unread in:inbox` 검색 |
| `pageToken` | (없음) | Gmail 다음 페이지 토큰 |

### 목록 API 응답 (`MailMessagesPage`)

```json
{
  "messages": [...],
  "pageSize": 10,
  "totalCount": 45,
  "unreadCount": 12,
  "inboxTotalCount": 1234,
  "nextPageToken": "...",
  "hasNext": true
}
```

- `totalCount`: 현재 필터 기준 전체 건수 (읽지 않음 필터면 unread, 전체면 inbox)
- `unreadCount`, `inboxTotalCount`: Gmail 라벨 API(`INBOX`, `UNREAD`)에서 조회

### 상세 API 응답 (`MailMessageDetail`)

```json
{
  "id": "...",
  "threadId": "...",
  "subject": "...",
  "from": "...",
  "date": "...",
  "snippet": "...",
  "unread": false,
  "bodyText": "...",
  "bodyHtml": "<html>..."
}
```

- 본문: multipart 메일에서 `text/plain` 우선, 없으면 `text/html`
- 상세 조회 시 `UNREAD` 라벨 제거 → Gmail·목록 모두 읽음 반영

## 7. 프론트 화면 동작

### 목록 (`MailView`)

- **읽지 않음** / **받은편지함 전체** 필터 (건수 표시)
- 페이지당 **10건** 조회
- 상·하단 **이전 / 다음** 페이징
- 상단 헤더: `받은편지함 전체 N건 · 1-10건 · 1 / M 페이지`
- 페이징 이동 시 로딩 오버레이 + `불러오는 중...` 표시
- **제목 클릭** → 상세 화면 이동

### 상세 (`MailDetailView`)

- 발신자, 날짜, 본문(HTML 또는 텍스트)
- **이전 / 다음**: 현재 페이지 목록 안에서 이동
- **삭제**: 휴지통 이동 후 다음 메일 또는 목록 복귀
- **목록**: 목록 화면으로 복귀
- 상세 열람 시 자동 **읽음** 처리 (Gmail + 목록 unread 뱃지 제거)

### App.vue 상태

| 상태 | 설명 |
|------|------|
| `mailViewMode` | `'list'` \| `'detail'` |
| `mailPageIndex` | 현재 페이지 (1부터) |
| `mailPageTokens` | Gmail pageToken 이력 (이전 페이지 복원용) |
| `mailUnreadOnly` | 읽지 않음 필터 (기본 true) |
| `selectedMailIndex` | 상세에서 목록 내 위치 |

### 키보드 단축키 (Option = ⌥)

`App.vue`의 `handleMailListKeyboardShortcut`, `handleMailDetailKeyboardShortcut`에서 처리한다.  
Mac **Option** 키는 `event.altKey`로 감지한다.

#### 목록 화면 (`mailViewMode === 'list'`)

| 단축키 | 동작 |
|--------|------|
| `⌥Enter` | 조회된 **첫 번째 메일** 상세 화면 열기 |

- 목록 하단 요약에 `⌥Enter 첫 메일 열기` 힌트 표시
- 로딩 중이거나 목록이 비어 있으면 동작하지 않음

#### 상세 화면 (`mailViewMode === 'detail'`)

| 단축키 | 동작 |
|--------|------|
| `⌥←` | 이전 메일 (현재 페이지 목록 내) |
| `⌥→` | 다음 메일 (현재 페이지 목록 내) |
| `⌥D` | 삭제 (휴지통 이동, 확인 대화상자) |

- 버튼 라벨: `이전 (⌥←)`, `다음 (⌥→)`, `삭제 (⌥D)`
- `Cmd+4`는 메일관리 메뉴 이동 (기존 전역 단축키)

#### 읽음 처리

상세 API(`GET /api/mail/messages/{id}`) 호출 시 백엔드에서 `UNREAD` 라벨 제거.  
프론트 `markMailMessageAsRead()`로 목록 unread 뱃지·건수도 즉시 반영.

```java
// GoogleGmailService.markMessageAsRead()
ModifyMessageRequest request = new ModifyMessageRequest()
    .setRemoveLabelIds(List.of("UNREAD"));
gmail.users().messages().modify("me", messageId, request).execute();
```

## 9. 자주 나는 오류

### redirect_uri_mismatch

Google Cloud Console **클라이언트**에 아래 URI가 있는지 확인:

```text
http://localhost:8080/api/mail/oauth/callback
```

### 메일 삭제·읽음 처리 권한 없음

```text
메일 삭제 권한이 없습니다. 메일관리 메뉴에서 Gmail 연결을 해제한 뒤 다시 연결하세요.
```

→ `gmail.modify` scope 추가 후 Gmail **재연결**.

### 읽지 않은 메일이 적게 보임

- 전체가 아니라 **현재 페이지 10건**만 표시
- **읽지 않음** 필터는 unread만 검색, **받은편지함 전체**는 최신 10건 중 unread 뱃지 표시

## 10. 터미널 연동 (미구현 · 검토 예정)

일정관리는 `gc` 명령어로 터미널에서 조회·추가·수정·삭제가 가능하다. 메일관리도 유사하게 터미널 명령을 붙일 수 있다.

### 후보 명령어 (아이디어)

| 명령 | 설명 |
|------|------|
| `mail` / `ml` | 연결 상태, 읽지 않음 건수 |
| `mail list` | 현재 페이지 목록 |
| `mail list --unread` | 읽지 않음만 |
| `mail read <id\|번호>` | 상세 보기 (읽음 처리) |
| `mail del <id\|번호>` | 휴지통 이동 |
| `mail connect` | OAuth 연결 URL 안내 |

### 참고할 기존 코드

- `mtools-ui/src/terminal/commands/calendarCommand.ts`
- `mtools-ui/src/terminal/apiClient.ts`
- `mtools-ui/src/terminal/help.ts`

터미널에서 메일 번호는 **현재 페이지 1~10 인덱스**로 매핑하는 방식이 자연스럽다.
