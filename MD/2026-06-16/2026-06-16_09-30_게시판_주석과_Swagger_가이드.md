# 게시판 소스 주석 + Swagger(OpenAPI) 가이드

## 개요

게시판(board) Java 소스에 **주석**과 **Swagger 어노테이션**을 추가했다.  
이 문서는 주석 규칙, Swagger 설정, API 문서화 방법을 정리한다.

---

## 1. 주석이 달린 파일

| 파일 | 주석 종류 |
|------|----------|
| `BoardController.java` | 클래스/메서드 JavaDoc, `@` 역할 설명 |
| `BoardService.java` | 원글·답글·st_order 로직 설명 |
| `BoardDao.java` | MyBatis 연결, `@Param` ↔ XML 설명 |
| `BoardPost.java` | 응답 DTO + `@Schema` |
| `BoardPostRequest.java` | 요청 DTO + `@Schema` |
| `BoardMapper.xml` | `<!-- -->` SQL/XML 설명 |

### 주석 규칙 (이 프로젝트)

| 위치 | 형식 | 예 |
|------|------|-----|
| 클래스 | `/** ... */` JavaDoc | 역할, 연결 대상 |
| public 메서드 | `/** ... */` | 무엇을 하는지, 규칙 |
| private 메서드 | `/** ... */` (핵심만) | st_order 계산 등 |
| XML | `<!-- -->` | id, namespace, SQL 목적 |
| `@` 어노테이션 | JavaDoc에서 이름 언급 | `{@code @Service}` |

> 자명한 getter/setter, 단순 위임에는 주석을 달지 않는다.

## 2. Swagger란?

**API 문서 + 브라우저에서 직접 테스트**할 수 있는 UI.

| 항목 | URL |
|------|-----|
| Swagger UI | http://localhost:8080/swagger-ui/index.html |
| OpenAPI JSON | http://localhost:8080/v3/api-docs |

mtools-api는 **springdoc-openapi** 라이브러리를 사용한다.

```gradle
// build.gradle
implementation 'org.springdoc:springdoc-openapi-starter-webmvc-ui:3.0.1'
```

```properties
# application.properties
springdoc.api-docs.enabled=true
springdoc.swagger-ui.enabled=true
```

별도 설정 클래스 없이도 **Controller가 기동되면 자동으로 API 목록**이 Swagger에 나타난다.

## 3. Swagger에 API 추가하는 방법

### 3-1. 최소 (어노테이션 없음)

`@RestController` + `@GetMapping` 등만 있어도 Swagger UI에 **자동 등록**된다.

```java
@GetMapping("/api/boards")
public List<BoardPost> boards() { ... }
```

→ Swagger에 `GET /api/boards` 가 보이지만, 설명·예시는 부족할 수 있다.

### 3-2. 권장 — 어노테이션으로 설명 추가

#### Controller — 그룹 + API 설명

```java
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.tags.Tag;

@Tag(name = "Board", description = "블로그형 무한 답글 게시판 API")
@RestController
public class BoardController {

    @Operation(summary = "게시글 목록 조회", description = "삭제 제외, 트리 정렬")
    @GetMapping("/api/boards")
    public List<BoardPost> boards() { ... }

    @Operation(summary = "게시글 등록", description = "parentId 생략=원글")
    @PostMapping("/api/boards")
    public BoardPost createBoard(@RequestBody BoardPostRequest request) { ... }

    @Operation(summary = "게시글 수정")
    @PutMapping("/api/boards/{idBoard}")
    public BoardPost updateBoard(
        @Parameter(description = "게시글 ID", example = "1")
        @PathVariable String idBoard,
        @RequestBody BoardPostRequest request) { ... }
}
```

| 어노테이션 | 붙는 곳 | 역할 |
|-----------|--------|------|
| `@Tag` | Controller 클래스 | Swagger UI **그룹 이름** (왼쪽 메뉴) |
| `@Operation` | Controller 메서드 | API **한 줄 요약** + 상세 설명 |
| `@Parameter` | `@PathVariable`, `@RequestParam` | URL 파라미터 설명·예시 |

#### Request / Response — 필드 설명

```java
import io.swagger.v3.oas.annotations.media.Schema;

@Schema(description = "게시글 등록/수정 요청")
public record BoardPostRequest(
    @Schema(description = "제목", example = "테스트2")
    String title,
    @Schema(description = "본문", example = "본문 내용")
    String text,
    @Schema(description = "부모 ID (답글만)", example = "2")
    String parentId
) {}
```

Swagger UI **Try it out** 시 예시 JSON에 `example` 값이 채워진다.

## 4. Swagger UI 사용법

1. API 서버 기동: `./gradlew bootRun`
2. 브라우저: http://localhost:8080/swagger-ui/index.html
3. 왼쪽에서 **Board** 그룹 클릭
4. API 선택 → **Try it out**
5. Request body 입력 → **Execute**
6. Response 확인

### POST /api/boards 예시 body

**원글**

```json
{
  "title": "테스트2",
  "text": "본문"
}
```

**답글**

```json
{
  "title": "테스트2-답글1",
  "text": "답글 내용",
  "parentId": "2"
}
```

## 5. 새 API를 Swagger에 추가하는 체크리스트

새 Controller/API를 만들 때:

- [ ] `@RestController` 클래스 생성
- [ ] `@GetMapping` / `@PostMapping` 등 URL 매핑
- [ ] (권장) 클래스에 `@Tag(name = "...", description = "...")`
- [ ] (권장) 메서드에 `@Operation(summary = "...")`
- [ ] (권장) Request/Response record에 `@Schema`
- [ ] (경로 변수 있으면) `@Parameter` on `@PathVariable`
- [ ] 서버 재기동 후 Swagger UI에서 확인

## 6. import 패키지 (springdoc / OpenAPI 3)

```java
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.tags.Tag;
import io.swagger.v3.oas.annotations.media.Schema;
```

> Swagger 2 (`io.swagger.annotations`) 가 아니라 **OpenAPI 3** (`io.swagger.v3.oas`) 패키지를 사용한다.

## 8. 주석 ↔ Swagger ↔ Vue 연결

```text
BoardPostRequest.java     @Schema → Swagger UI request body 스키마
        ↑ JSON
BoardController.java      @Operation → Swagger API 설명
        ↓
BoardService.java         JavaDoc → 개발자용 로직 설명 (Swagger 미표시)
        ↓
BoardDao.java + XML       JavaDoc/XML 주석 → SQL (Swagger 미표시)
        ↓
t_board (PostgreSQL)
        ↑ JSON
BoardPost.java            @Schema → Swagger response 스키마
        ↓
Vue BoardView.vue         fetch /api/boards
```

- **Swagger에 보이는 것:** Controller URL, `@Operation`, `@Schema` 필드
- **Swagger에 안 보이는 것:** Service 로직, MyBatis SQL (주석은 코드 읽을 때만)
