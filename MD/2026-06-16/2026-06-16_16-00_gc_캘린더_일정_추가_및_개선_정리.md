# gc 캘린더 일정 추가 및 개선 정리

## 개요

기존 `gc` 조회 명령에 이어 아래 작업을 추가·개선했다.

1. **`gc c` 키 규칙 수정** — 목록 순번이 아닌 캘린더 이름 앞 숫자를 키로 사용
2. **휴일 캘린더 제외** — `대한민국의 휴일` 등 목록·조회에서 제외
3. **일정 목록 날짜 구분** — 날짜가 바뀌면 빈 줄(여백)로 구분
4. **`gc lcb`** — 날짜 범위 조회 (기존 `gc lbc` 대체)
5. **`gc a` / `gc ad` / `gc at`** — Google Calendar 일정 추가
6. **OAuth 쓰기 권한** — 일정 추가를 위해 `calendar` 스코프로 변경

---

## 1. `gc c` — 캘린더 목록 키

### 규칙

| 항목 | 설명 |
|------|------|
| 키 | 캘린더 이름 앞 숫자 (`1.` → 키 `1`, `6.` → 키 `6`) |
| 표시 | `{키}  {Google 캘린더 이름}` |
| 제외 | 번호 없는 캘린더, `대한민국의 휴일` 등 휴일 캘린더 |

### 출력 예

```text
> gc c
0       0. (개인) 특별한 날
1       1. (개인) 흔적
2       2. (개인) 운동로그
6       6. (회사) 업무스케줄
7       7. (승현) 교육스케줄
```

`gc lc`, `gc lcb`, `gc a` 등에서 `<키>`는 위 왼쪽 숫자를 사용한다.

---

## 2. 일정 조회 명령 (`gc l` 계열)

### 전체 명령어

| 명령어 | 설명 |
|--------|------|
| `gc` | 오늘 일정 (전체 캘린더) |
| `gc l` | 1주일 일정 (전체 캘린더) |
| `gc l <YYYYMMDD>` | 해당 날짜 일정 (전체) |
| `gc c` | 내 캘린더 목록 |
| `gc lc <키>` | 해당 캘린더 1주일 일정 |
| `gc lc <키> <YYYYMMDD>` | 해당 캘린더 특정 날짜 일정 |
| `gc lcb <키> <YYYYMMDD> <YYYYMMDD>` | 해당 캘린더 날짜 범위 일정 |

날짜 형식: **`YYYYMMDD`** (예: `20260616`)

### 사용 예

```text
gc lc 6
gc lc 6 20260616
gc lcb 6 20260601 20260607
```

### 날짜별 구분 출력

같은 날짜 일정은 연속 출력하고, **날짜가 바뀌면 한 줄 여백**을 둔다.

```text
2026-06-16 종일 | 6. (회사) 업무스케줄 | 회의
2026-06-16 14:00 | 6. (회사) 업무스케줄 | 미팅

2026-06-17 09:00 | 6. (회사) 업무스케줄 | 출장
```

구현: `TerminalLineDraft.groupBreak` + CSS `terminal-line-group-break`

## 3. 일정 추가 명령 (`gc a` 계열)

### 명령어

| 명령어 | 설명 |
|--------|------|
| `gc a <키> <일정>` | 오늘 **종일** 일정 추가 |
| `gc ad <키> <YYYYMMDD> <일정>` | 해당 날짜 **종일** 일정 추가 |
| `gc at <키> <YYYYMMDD> <HHmm> <일정>` | 해당 날짜·시간 일정 추가 (**종료 = 시작 + 1시간**) |

`<일정>`은 공백 포함 가능 (제목 전체).

### 사용 예

```text
gc a 6 팀 미팅
gc ad 6 20260620 휴가
gc at 6 20260620 0930 고객 미팅
```

### 성공 메시지

```text
일정 추가: 팀 미팅 (2026-06-16 종일)
일정 추가: 고객 미팅 (2026-06-20 09:30)
```

### OAuth 재연결 (필수)

일정 추가 API는 **쓰기 권한(`calendar` 스코프)** 이 필요하다.  
예전에 **읽기 전용(`calendar.readonly`)** 으로 연결한 토큰은 그대로 두면 `gc a` 등이 **403 오류**로 실패한다.

**재연결 순서**

1. 사이드바 **일정관리** 메뉴 이동
2. **Google Calendar 연결 해제** 클릭
3. **Google Calendar 연결** 다시 진행 (OAuth 동의 화면)
4. 터미널에서 `gc a 6 테스트` 등 재시도

## 4. 백엔드 API

### 조회

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/calendar/calendars` | 캘린더 목록 (이름 앞 숫자 = key) |
| GET | `/api/calendar/events` | 일정 조회 |
| GET | `/api/calendar/events?calendarKey=6` | 6번 캘린더만 조회 |

Query: `timeMin`, `timeMax` (ISO 8601), `calendarKey` (선택)

### 추가

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/calendar/events` | 일정 추가 |

**Request Body**

```json
{
  "calendarKey": 6,
  "summary": "팀 미팅",
  "date": "20260620",
  "time": "0930"
}
```

| 필드 | 필수 | 설명 |
|------|------|------|
| calendarKey | Y | `gc c` 목록 키 |
| summary | Y | 일정 제목 |
| date | N | `YYYYMMDD` — 생략 시 오늘 |
| time | N | `HHmm` — 생략 시 종일, 있으면 1시간 일정 |

**Response**: `CalendarEvent` (기존 조회와 동일 형식)

### OAuth 스코프 변경

```java
// 변경 전
CalendarScopes.CALENDAR_READONLY

// 변경 후
CalendarScopes.CALENDAR
```

일정 추가(`gc a` 등) 사용 전 **Google Calendar 연결 해제 후 재연결** 필요.

403 발생 시 백엔드·터미널 모두 아래 메시지를 표시한다.

```text
일정 추가 권한이 없습니다. 일정관리 메뉴에서 Google Calendar 연결을 해제한 뒤 다시 연결하세요.
```

| 구분 | 처리 |
|------|------|
| Google API | `ACCESS_TOKEN_SCOPE_INSUFFICIENT` → `IllegalStateException` |
| 백엔드 | HTTP 400 + 위 한글 메시지 |
| 터미널 | `parseHttpError()` — JSON/trace 대신 위 메시지 출력 |

### 백엔드 파일

| 파일 | 내용 |
|------|------|
| `CreateCalendarEventRequest.java` | 일정 추가 요청 DTO |
| `GoogleCalendarService.java` | `createEvent()`, `parseCalendarKey()`, 휴일 제외, 키 조회 |
| `CalendarController.java` | `POST /api/calendar/events` |

### `executeGcCommand` 파싱 순서

1. `gc` (단독)
2. `gc c`
3. `gc at` → `gc ad` → `gc a` (추가)
4. `gc lcb`
5. `gc lc`
6. `gc l`

`at` / `ad` / `a`를 `lc`보다 먼저 검사해 접두어 충돌을 방지한다.

## 6. 트러블슈팅 — `gc a` 일정 추가 오류

### 증상

`gc a`, `gc ad`, `gc at` 실행 시 터미널에 에러가 표시되거나, 긴 JSON/trace가 출력된다.

### 원인

| 원인 | 설명 |
|------|------|
| **OAuth 스코프 부족** | 기존 토큰이 `calendar.readonly` — Google API 403 |
| **백엔드 미재시작** | `POST /api/calendar/events` API 미반영 → 404 |

Google API 원인 코드: `ACCESS_TOKEN_SCOPE_INSUFFICIENT`

### 해결

1. **일정관리 → 연결 해제 → 재연결** (스코프 부족 시)
2. **`./gradlew bootRun` 재시작** (404 시)
3. 재연결 후 `gc c`로 키 확인 → `gc a 6 테스트` 실행

### 에러 메시지 개선 (2026-06-16)

- `GoogleCalendarService.createEvent()` — 403 catch 후 한글 메시지
- `terminalCommands.parseHttpError()` — scope/permission 키워드 감지
- `TerminalView.runCommand()` — 예외 시 에러 줄 표시
