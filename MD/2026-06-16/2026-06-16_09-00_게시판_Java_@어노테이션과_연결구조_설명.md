# 게시판 Java — `@` 어노테이션과 연결 구조 (초보자용)

## 이 문서의 목적

Java/Spring을 잘 모르더라도, **게시판(board) 코드가 어떻게 서로 연결되는지** 이해할 수 있도록 정리한 문서다.

특히 `@`로 시작하는 **어노테이션(annotation)** 이 무엇을 의미하고,  
Vue → API → DB까지 **누가 누구를 호출하는지** 흐름 중심으로 설명한다.

---

## 0. 큰 그림 (한 장 요약)

```text
[Vue 브라우저]
    │  fetch("http://localhost:8080/api/boards")
    ▼
[BoardController]  ← @RestController, @GetMapping ...
    │  boardService.findAll()
    ▼
[BoardService]     ← @Service
    │  boardDao.findAll()
    ▼
[BoardDao]         ← @Mapper (인터페이스만 있음)
    │  MyBatis가 BoardMapper.xml SQL 실행
    ▼
[PostgreSQL t_board]
```

**핵심:** 개발자가 `new BoardService()` 같은 코드를 직접 쓰지 않아도,  
Spring이 `@` 표시를 보고 **자동으로 객체를 만들고 연결**해 준다.

---

## 1. 서버는 어떻게 켜지나?

### `@SpringBootApplication`

```java
// MtoolsApiApplication.java
@SpringBootApplication
public class MtoolsApiApplication {
    public static void main(String[] args) {
        SpringApplication.run(MtoolsApiApplication.class, args);
    }
}
```

| 항목 | 설명 |
|------|------|
| `@SpringBootApplication` | "이 프로젝트는 Spring Boot 앱이야"라고 선언 |
| `main()` | `./gradlew bootRun` 실행 시 **여기서 시작** |
| `SpringApplication.run(...)` | Tomcat(웹서버) + 각종 `@` 컴포넌트를 **한꺼번에 기동** |

기동 후 `http://localhost:8080` 에서 API 요청을 받을 준비가 된다.

---

## 2. `@` 어노테이션이란?

**클래스·메서드·필드 위에 붙이는 메타데이터(표시)** 다.

```java
@RestController   // ← "이 클래스는 HTTP API 창구야"
public class BoardController { ... }
```

Spring Boot는 기동할 때 프로젝트 안의 `.java` 파일을 스캔하면서  
`@RestController`, `@Service`, `@Mapper` 등이 붙은 클래스를 찾고,  
**역할에 맞게 객체를 만들어 등록**한다.

> 비유: `@` = **명찰**. Spring = **"명찰 보고 일 시키는 매니저"**.

---

## 3. 게시판 관련 `@` 정리표

| 어노테이션 | 붙는 위치 | 역할 |
|-----------|----------|------|
| `@RestController` | Controller 클래스 | HTTP 요청 받아 JSON 응답 |
| `@GetMapping` | Controller 메서드 | GET 요청 URL 매핑 |
| `@PostMapping` | Controller 메서드 | POST 요청 URL 매핑 |
| `@PutMapping` | Controller 메서드 | PUT 요청 URL 매핑 |
| `@DeleteMapping` | Controller 메서드 | DELETE 요청 URL 매핑 |
| `@RequestBody` | Controller 메서드 파라미터 | JSON body → Java 객체 변환 |
| `@PathVariable` | Controller 메서드 파라미터 | URL 경로의 `{id}` 값 추출 |
| `@Service` | Service 클래스 | 비즈니스 로직 담당 |
| `@Mapper` | Dao 인터페이스 | MyBatis DB 접근 |
| `@Param` | Dao 메서드 파라미터 | SQL `#{이름}` 과 연결 |

## 4. Controller — `@RestController` + URL 매핑

```java
@RestController
public class BoardController {

    private final BoardService boardService;

    public BoardController(BoardService boardService) {
        this.boardService = boardService;
    }

    @GetMapping("/api/boards")
    public List<BoardPost> boards() {
        return boardService.findAll();
    }

    @PostMapping("/api/boards")
    public BoardPost createBoard(@RequestBody BoardPostRequest request) {
        return boardService.create(request);
    }

    @PutMapping("/api/boards/{idBoard}")
    public BoardPost updateBoard(
            @PathVariable String idBoard,
            @RequestBody BoardPostRequest request) {
        return boardService.update(idBoard, request);
    }

    @DeleteMapping("/api/boards/{idBoard}")
    public Map<String, String> deleteBoard(@PathVariable String idBoard) {
        boardService.delete(idBoard);
        return Map.of("status", "ok");
    }
}
```

### `@RestController`

- 이 클래스 = **REST API 창구**
- 메서드 반환값(`BoardPost`, `List<BoardPost>`)을 **자동으로 JSON**으로 변환해 응답

### `@GetMapping("/api/boards")`

```text
브라우저 GET http://localhost:8080/api/boards
    → boards() 메서드 실행
```

URL과 Java 메서드를 **1:1로 연결**한다.

### `@PostMapping("/api/boards")`

```text
POST http://localhost:8080/api/boards
Body: { "title": "...", "text": "...", "parentId": "2" }
    → createBoard() 실행
```

### `@RequestBody BoardPostRequest request`

HTTP body의 JSON:

```json
{ "title": "제목", "text": "내용", "parentId": "2" }
```

을 Java 객체 `BoardPostRequest`로 **자동 변환**한다.

```java
public record BoardPostRequest(String title, String text, String parentId) {}
```

필드 이름이 JSON 키와 같으면 매핑된다.

### `@PathVariable String idBoard`

```text
PUT http://localhost:8080/api/boards/3
```

에서 URL의 `{idBoard}` 자리 값 `"3"`을 `idBoard` 변수에 넣는다.

## 5. Service — `@Service` + 비즈니스 로직

```java
@Service
public class BoardService {

    private final BoardDao boardDao;

    public BoardService(BoardDao boardDao) {
        this.boardDao = boardDao;
    }

    public BoardPost create(BoardPostRequest request) {
        String nextId = String.valueOf(boardDao.nextId());

        if (!StringUtils.hasText(request.parentId())) {
            // 원글
            boardDao.insert(nextId, nextId, "#001", request.title(), request.text());
            return findById(nextId);
        }

        // 답글
        BoardPost parent = findById(request.parentId().trim());
        String childOrder = nextChildOrder(parent.stPid(), parent.stOrder());
        boardDao.insert(nextId, parent.stPid(), childOrder, request.title(), request.text());
        return findById(nextId);
    }
}
```

### `@Service`

- **"주방"** 역할 — Controller는 SQL/규칙을 몰라도 됨
- `@RestController`와 같은 방식으로 Spring이 **객체 1개 생성** 후 보관

### Controller와 Service 연결 (의존성 주입, DI)

```java
public BoardController(BoardService boardService) {
    this.boardService = boardService;
}
```

| 일반 Java (직접 연결) | Spring (자동 연결) |
|---------------------|-------------------|
| `new BoardService(new BoardDao())` | 생성자에 타입만 적으면 Spring이 알아서 넣어줌 |

이것을 **의존성 주입(DI, Dependency Injection)** 이라고 부른다.

```text
Spring 기동
  → @Service BoardService 객체 생성
  → @Mapper BoardDao 구현체 생성
  → BoardService 생성자에 BoardDao 주입
  → @RestController BoardController 생성
  → BoardController 생성자에 BoardService 주입
```

개발자는 **`new`를 거의 쓰지 않는다**.

## 6. Dao — `@Mapper` + MyBatis

```java
@Mapper
public interface BoardDao {

    List<BoardPost> findAll();

    BoardPost findById(@Param("idBoard") String idBoard);

    long nextId();

    int insert(
        @Param("idBoard") String idBoard,
        @Param("stPid") String stPid,
        @Param("stOrder") String stOrder,
        @Param("title") String title,
        @Param("text") String text
    );
}
```

### `@Mapper`

- **DB 접근 전용** 인터페이스
- `BoardDao`에는 **메서드 선언만** 있고, **구현 코드는 없음**
- MyBatis가 `BoardMapper.xml`을 읽어 **구현체를 자동 생성**

### `@Param("idBoard")`

XML SQL의 `#{idBoard}` 와 연결:

```xml
WHERE id_board = #{idBoard}
```

Java에서 `findById("3")` 호출 → SQL에 `3`이 바인딩된다.

---

## 7. BoardMapper.xml — SQL과 Java 연결

```xml
<mapper namespace="com.mtools.api.board.dao.BoardDao">

    <select id="findAll" resultMap="BoardPostMap">
        SELECT id_board, st_pid, st_order, st_title, tx_board, ...
        FROM t_board
        WHERE COALESCE(yn_use, 'Y') != 'N'
        ORDER BY st_pid DESC, st_order ASC
    </select>

</mapper>
```

| XML 요소 | 의미 |
|---------|------|
| `namespace` | **어느 Dao 인터페이스**와 짝인지 (`BoardDao` 전체 경로) |
| `<select id="findAll">` | `BoardDao.findAll()` 메서드와 연결 |
| `#{idBoard}` | `@Param("idBoard")` 값 치환 |
| `resultMap="BoardPostMap"` | 조회 결과 → `BoardPost` 객체로 변환 규칙 |

### resultMap — DB 컬럼 → Java 필드

```xml
<resultMap id="BoardPostMap" type="com.mtools.api.board.service.BoardPost">
    <constructor>
        <arg column="id_board" javaType="java.lang.String" />
        <arg column="st_pid" javaType="java.lang.String" />
        ...
    </constructor>
</resultMap>
```

DB: `id_board`, `st_pid` (snake_case)  
Java: `idBoard`, `stPid` (camelCase)  
→ MyBatis가 **조회 결과를 BoardPost record로 조립**한다.

### application.properties 연결

```properties
mybatis.mapper-locations=classpath:mapper/**/*.xml
```

Spring Boot 기동 시 `mapper/board/BoardMapper.xml` 을 읽도록 설정.

## 8. record — 데이터 상자

```java
// 응답용
public record BoardPost(
    String idBoard,
    String stPid,
    String stOrder,
    String title,
    String text,
    String insertedAt,
    String updatedAt
) {}

// 요청용
public record BoardPostRequest(
    String title,
    String text,
    String parentId
) {}
```

| 특징 | 설명 |
|------|------|
| `record` | 필드만 선언하면 getter, equals 등 **자동 생성** |
| JSON 변환 | Spring이 `BoardPost` → JSON, JSON → `BoardPostRequest` 자동 처리 |
| Vue에서 받는 형태 | `{ "idBoard": "1", "stPid": "1", "title": "..." }` |

```text
         │
         ▼

         │                               │                              │
   @GetMapping 등                    create(), findAll()          interface만

         │                               │                              │
         ▼                               ▼                              ▼

```

## 10. 실제 요청 1건 따라가기 (답글 등록)

### 1) Vue

```javascript
fetch('http://localhost:8080/api/boards', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    title: '테스트2-답글1',
    text: '답글 내용',
    parentId: '2',
  }),
})
```

### 2) `@PostMapping` → `BoardController.createBoard()`

- URL `/api/boards` + POST → 이 메서드 선택
- `@RequestBody` → JSON을 `BoardPostRequest`로 변환

### 3) `boardService.create(request)`

- `parentId`가 있으므로 **답글 분기**
- `boardDao.nextId()` → `"5"`
- `findById("2")` → 부모 글 조회
- `nextChildOrder(...)` → `"#001#001"`
- `boardDao.insert("5", "2", "#001#001", ...)`

### 4) MyBatis → `BoardMapper.xml` `<insert id="insert">`

```sql
INSERT INTO t_board (id_board, st_pid, st_order, st_title, tx_board, ...)
VALUES ('5', '2', '#001#001', '테스트2-답글1', '답글 내용', ...)
```

### 5) `findById("5")` → SELECT → `BoardPost` → JSON 응답

```json
{
  "idBoard": "5",
  "stPid": "2",
  "stOrder": "#001#001",
  "title": "테스트2-답글1",
  ...
}
```

## 12. 자주 묻는 질문

### Q. `@` 없으면 동작 안 하나?

Spring이 관리하지 않는 일반 클래스는 **URL과 연결되지 않는다**.  
Controller/Service/Dao는 `@`로 Spring에게 "나를 등록해줘"라고 알려야 한다.

### Q. BoardDao 구현 클래스는 어디 있나?

**없다.** MyBatis가 `@Mapper` + XML을 보고 **런타임에 자동 생성**한다.

### Q. Controller에서 Dao 직접 호출하면?

기술적으로 가능하지만, **Service를 거치는 게 규칙**이다.  
로직(원글/답글 구분)이 Service에 모여 있어야 유지보수가 쉽다.

### Q. `@PathVariable` vs `@RequestBody` 차이?

| | 위치 | 예 |
|---|------|-----|
| `@PathVariable` | URL 경로 | `/api/boards/3` → `3` |
| `@RequestBody` | HTTP body (JSON) | `{ "title": "..." }` |

### Q. Vue의 `fetch`와 Java `@GetMapping` 관계?

```text
fetch GET /api/boards  ↔  @GetMapping("/api/boards")
```

**URL + HTTP 메서드**가 같으면 연결된다.
