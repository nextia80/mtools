# 블로그형 무한 답글 게시판 구현

## 개요

기존 평면(flat) CRUD 게시판을 **블로그형 + 무한 depth 답글** 구조로 재구현했다.

- `st_pid` + `st_order`로 트리 구조 표현
- SQL 정렬만으로 계층 출력 (`ORDER BY st_pid DESC, st_order ASC`)
- 원글 / 댓글 / 대댓글 무제한 생성

---

## 테이블 구조 (`t_board`)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id_board` | VARCHAR(100) PK | 게시글 고유키 (Sequence) |
| `st_pid` | VARCHAR(100) | 원글 그룹 ID |
| `st_order` | VARCHAR(10000) | 계층 정렬값 |
| `st_title` | VARCHAR(1000) | 제목 |
| `tx_board` | TEXT | 본문 |
| `yn_use` | VARCHAR(1) | Y: 사용, N: 삭제 |
| `id_insert` | VARCHAR(100) | 등록자 |
| `dt_insert` | TIMESTAMP | 등록일시 |
| `id_update` | VARCHAR(100) | 수정자 |
| `dt_update` | TIMESTAMP | 수정일시 |

인덱스: `(st_pid, st_order)`

---

## 데이터 규칙

### id_board

신규 등록 시 `MAX(id_board) + 1` Sequence 방식.

```text
1, 2, 3, ...
```

### st_pid

| 구분 | 규칙 |
|------|------|
| 원글 (1Level) | `st_pid = id_board` |
| 답글 (2Level+) | 최상위 원글의 `id_board` |

```text
id=2  pid=2   (원글 테스트2)
id=3  pid=2   (테스트2 답글)
id=5  pid=2   (테스트2 대댓글)
```

### st_order

3자리 단위 `#001` 세그먼트로 계층·순서 표현.

| 글 | st_order |
|----|----------|
| 원글 | `#001` |
| 1번째 답글 | `#001#001` |
| 2번째 답글 | `#001#002` |
| 대댓글 | `#001#001#001` |

### 예시 데이터

```text
ID  PID  ORDER            TITLE
------------------------------------------------
2   2    #001             테스트2
3   2    #001#001         테스트2-답글1
5   2    #001#001#001     테스트2-답글1-답글1
4   2    #001#002         테스트2-답글2
1   1    #001             테스트1
```

조회 정렬:

```sql
ORDER BY st_pid DESC, st_order ASC
```

## 답글 생성 로직 (백엔드)

### 원글 등록

```text
id_board = nextId
st_pid   = id_board
st_order = #001
```

### 답글 등록 (`parentId` 전달)

1. 부모 게시글 조회
2. `st_pid` = 부모의 `st_pid` (원글 그룹 유지)
3. `st_order` = 부모 `st_order` + `#` + `(형제 max + 1, 3자리)`

```java
// BoardService.nextChildOrder()
parentOrder + String.format("#%03d", maxSegment + 1)
```

직계 자식만 조회:

```sql
WHERE st_pid = #{stPid}
  AND st_order LIKE CONCAT(#{parentOrder}, '#%')
  AND LENGTH(st_order) = LENGTH(#{parentOrder}) + 4
```

## API

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/boards` | 전체 목록 (flat, 트리 순서) |
| POST | `/api/boards` | 등록 |
| PUT | `/api/boards/{idBoard}` | 수정 |
| DELETE | `/api/boards/{idBoard}` | 소프트 삭제 (`yn_use='N'`) |

### 요청 예시

**원글**

```json
{
  "title": "테스트2",
  "text": "본문"
}
```

**답글**

```json
{
  "title": "테스트2-답글1",
  "text": "답글 내용",
  "parentId": "2"
}
```

### 응답 예시

```json
{
  "idBoard": "3",
  "stPid": "2",
  "stOrder": "#001#001",
  "title": "테스트2-답글1",
  "text": "답글 내용",
  "insertedAt": "2026-06-16 08:02:09.048412",
  "updatedAt": null
}
```

## 화면 구성 (UI)

### 원글 작성

- 제목 / 내용 / [등록]

### 댓글 작성

- **댓글추가** 클릭 시 폼 표시
- 원글 ID (readonly) / 제목 / 내용 / [등록]

### 게시글 목록

- `st_pid DESC, st_order ASC` 순서로 flat 렌더링
- **depth별 들여쓰기** (`st_order`의 `#` 개수 − 1)
- 원글: 번호 + 제목, 답글: 제목만
- 수정일시 있으면 수정일시 우선 표시
- 각 글: **댓글추가** / 수정 / 삭제

### 출력 예시

```text
2  테스트2
   테스트2 본문
   (댓글추가)  2026. 6. 16. ...

       테스트2-답글1
       답글 내용
       2026. 6. 16. ...

           테스트2-답글1-답글1
           대댓글
           2026. 6. 16. ...
```

|------|------|
