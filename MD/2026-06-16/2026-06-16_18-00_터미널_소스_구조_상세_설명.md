# mtools 터미널 소스 구조 상세 설명

## 1. 개요

mTools 홈 화면의 **터미널**은 CLI(명령줄)처럼 텍스트 명령을 입력하고, 백엔드 API 결과를 텍스트로 출력하는 UI다.

| 구분 | 역할 |
|------|------|
| **화면 (Vue)** | 입력·출력·히스토리·클릭 UX |
| **로직 (TypeScript)** | 명령 파싱, API 호출, 결과 포맷 |
| **백엔드 (Spring Boot)** | 게시판·캘린더·MD 파일 데이터 제공 |

터미널 관련 프론트 코드는 **`mtools-ui/src/terminal/`** 폴더와 **`TerminalView.vue`** 로 나뉜다.

```text
사용자 입력
    ↓
TerminalView.vue          ← 화면 (Vue)
    ↓
terminal/index.ts         ← 명령 분기 (라우터)
    ↓
commands/*.ts             ← bd / gc / md 처리
    ↓
apiClient.ts              ← HTTP 호출
    ↓
http://localhost:8080     ← Spring Boot API
    ↓
결과 줄 (TerminalLineDraft[])
    ↓
TerminalView.vue          ← 화면 출력
```

---

## 2. TypeScript(TS)와 이 프로젝트

터미널 로직은 **TypeScript(`.ts`)** 로 작성되어 있다.

- JavaScript + **타입(형태) 정의**
- 브라우저 실행 전 JS로 변환
- `npm run type-check`로 오타·타입 오류 사전 검사

예: API 응답 형태를 `CalendarEvent`로 정의해 두면, `event.summry` 같은 오타를 IDE/빌드 단계에서 잡을 수 있다.

터미널에서 자주 쓰는 TS 개념:

| 문법 | 터미널에서의 예 |
|------|----------------|
| `type` | `TerminalLineDraft`, `TerminalAction` |
| `export` | 다른 파일에서 import 가능하게 내보내기 |
| `async/await` | API 호출 후 결과 기다리기 |
| `Promise<T>` | 비동기 함수 반환 타입 |
| `T` (제네릭) | `fetchJson<BoardPost[]>(url)` — 응답 타입 지정 |

## 4. 데이터 타입

### 4.1 출력 한 줄 — `TerminalLineDraft` (`src/types.ts`)

명령 실행 결과는 **줄(line) 배열**로 표현한다.

```typescript
export type TerminalLineDraft = {
  type: 'input' | 'output' | 'error'   // 줄 종류
  text: string                          // 화면에 보이는 텍스트
  clickValue?: string                   // 클릭 시 입력창에 붙일 값
  indented?: boolean                    // (예약) 들여쓰기
  groupBreak?: boolean                  // 날짜 구분용 위쪽 여백 (gc)
}
```

화면에 그릴 때 `id`를 붙여 `TerminalLine`으로 만든다 (`TerminalView.vue`의 `assignLineIds`).

| 필드 | 용도 | 예 |
|------|------|-----|
| `type: 'output'` | 일반 출력 | 일정 목록, 게시글 제목 |
| `type: 'error'` | 에러 (빨간색) | 사용법 오류, API 실패 |
| `clickValue` | 클릭 가능 줄 | `[7] 제목` → `7` 클릭 시 `bd g 7` 입력 |
| `groupBreak` | 날짜 바뀔 때 여백 | gc 일정 목록 |

### 4.2 명령 결과 — `TerminalCommandResult` (`terminal/types.ts`)

```typescript
export type TerminalCommandResult = {
  lines: TerminalLineDraft[]   // 출력 줄
  clear?: boolean              // true → cls, 화면 전체 지우기
  action?: TerminalAction      // 화면 전환 등 부가 동작
}
```

### 4.3 부가 동작 — `TerminalAction`

터미널만으로 끝나지 않고 **다른 화면으로 이동**할 때 사용한다.

```typescript
export type TerminalAction =
  | { type: 'open-md'; path: string }      // md g → 일자별 LOG 페이지
  | { type: 'board-add' }                 // bd a → 게시판 등록 (준비 중)
  | { type: 'board-edit'; id: string }    // bd m → 게시판 수정 (준비 중)
```

`TerminalView` → `emit('action', ...)` → `App.vue`의 `handleTerminalAction()`에서 처리한다.

## 5. 화면 — `TerminalView.vue`

### 5.1 역할

- 명령어 **입력·출력 UI**만 담당
- 명령 규칙(bd/gc/md)은 **모름** → `executeTerminalCommand()`에 위임

### 5.2 상태 변수

| 변수 | 설명 |
|------|------|
| `welcomeLines` | 최초 안내 문구 (2줄) |
| `commandBlocks` | `{ command, results }[]` — 명령별 출력 블록 |
| `commandInput` | 현재 입력창 텍스트 |
| `commandHistory` | ↑↓ 히스토리 |
| `isRunning` | API 호출 중 → 입력 비활성, "실행 중..." 표시 |

### 5.3 화면 구조 (템플릿)

```text
┌─ welcomeLines (안내) ─────────────────┐
│ mtools terminal — bd / gc / md ...   │
│ /? 를 입력하면 ...                    │
└──────────────────────────────────────┘

┌─ commandBlock 1 ──────────────────────┐
│ > gc c                    ← 입력했던 명령 │
│ ┌ result-box ─────────────────────┐  │
│ │ 1  1. (개인) 흔적  [클릭 가능]   │  │
│ │ 6  6. (회사) 업무스케줄          │  │
│ └─────────────────────────────────┘  │
└──────────────────────────────────────┘

┌─ commandBlock 2 ──────────────────────┐
│ > gc lc 6                             │
│ ...                                   │
└──────────────────────────────────────┘

> [입력창]  ← commandInput
```

### 5.4 `runCommand()` — 핵심 흐름

1. 입력값 trim, 빈 값·실행 중이면 무시
2. `commandBlocks`에 새 블록 추가 (명령만 먼저 표시)
3. `executeTerminalCommand(command, apiBaseUrl)` 호출
4. 결과 처리:
   - `clear: true` → `clearTerminal()` (cls)
   - `lines.length > 0` → `block.results`에 줄 할당
   - `action` 있으면 → `emit('action', action)`
5. 스크롤 맨 아래, 입력창 포커스

### 5.5 클릭으로 입력 채우기

조건: 입력창이 `bd g `, `bd m `, `bd d `, `md g ` 상태일 때만

목록 줄의 `clickValue`(게시판 ID, MD 파일키 등)를 클릭하면  
→ 입력창 끝에 값이 붙는다. (예: `bd g` + 클릭 `7` → `bd g 7`)

### 5.6 키보드

| 키 | 동작 |
|----|------|
| Enter | `runCommand()` |
| ↑ | 이전 명령 |
| ↓ | 다음 명령 / 빈 입력 |

### 5.7 외부 연동

- `defineExpose({ focusInput })` — `App.vue`에서 ⌘+H로 홈 이동 시 포커스
- `props.active` — 홈 탭 활성화 시 입력창 포커스

## 6. 명령 라우터 — `terminal/index.ts`

모든 명령의 **진입점**이다.

```typescript
export const executeTerminalCommand = async (input, apiBaseUrl) => {
  // 1. /?        → help.ts
  // 2. cls       → { clear: true }
  // 3. bd ...    → boardCommand.ts
  // 4. gc ...    → calendarCommand.ts
  // 5. md ...    → mdCommand.ts
  // 6. 그 외     → "알 수 없는 명령어"
}
```

첫 단어(`group`)로 분기한다.

```typescript
const group = trimmed.split(/\s+/)[0]?.toLowerCase()
// "gc lc 6" → group === "gc"
```

`try/catch`로 API 예외를 잡아 `{ type: 'error', text: message }` 한 줄로 반환한다.

## 9. 날짜 공통 — `terminal/dateUtils.ts`

gc 명령에서 날짜를 다룰 때 사용한다.

| 함수 | 설명 | 예 |
|------|------|-----|
| `parseYmd(raw)` | `YYYYMMDD` 유효성 검사 | `20260616` → OK, `20261399` → null |
| `buildDayRange(ymd)` | 하루 → ISO timeMin/timeMax | API 쿼리용 |
| `buildDateRange(from, to)` | 기간 범위 | `gc lcb`용 |
| `getTodayYmd()` | 오늘 `YYYYMMDD` | `gc`, `gc a` |

백엔드 `/api/calendar/events`는 `timeMin`, `timeMax`를 ISO 8601 문자열로 받는다.

## 11. 게시판 명령 — `commands/boardCommand.ts`

### 11.1 명령어表

| 명령 | API | 설명 |
|------|-----|------|
| `bd l` | GET `/api/boards` | 상위 20개 제목 |
| `bd l <n>` | GET `/api/boards` | 상위 n개 |
| `bd lb <n>~<m>` | GET `/api/boards` | n~m번째 제목 |
| `bd g <id>` | GET `/api/boards` | 전체 정보 |
| `bd a` | — | 등록 팝업 (action) |
| `bd m <id>` | GET `/api/boards` | 수정 팝업 (action) |
| `bd d <id>` | DELETE `/api/boards/{id}` | 삭제 |

### 11.2 출력 포맷

- 목록: `[id] 제목` — 답글 depth만큼 들여쓰기 (`stOrder`의 `#` 개수)
- `clickValue: id` — 클릭 시 `bd g/m/d`에 ID 채움

### 11.3 파싱

`bd` 접두어 제거 후 `rest` 문자열을 정규식으로 매칭한다.

```typescript
const rest = trimmed.replace(/^bd\s+/i, '').trim()
// "bd l 10" → rest = "l 10"
```

## 12. 캘린더 명령 — `commands/calendarCommand.ts`

가장 명령·기능이 많은 모듈이다.

### 12.1 조회 명령

| 명령 | API | 설명 |
|------|-----|------|
| `gc` | GET `/api/calendar/events?timeMin&timeMax` | 오늘, 전체 캘린더 |
| `gc l` | GET events (기본 1주) | 1주일 |
| `gc l <YYYYMMDD>` | GET events + 날짜 범위 | 해당 날짜 |
| `gc c` | GET `/api/calendar/calendars` | 캘린더 목록 |
| `gc lc <키>` | GET events?calendarKey= | 키 기준 1주 |
| `gc lc <키> <YYYYMMDD>` | GET events?calendarKey&날짜 | 키+날짜 |
| `gc lcb <키> <from> <to>` | GET events?calendarKey&기간 | 키+기간 |

### 12.2 추가 명령

| 명령 | API | 설명 |
|------|-----|------|
| `gc a <키> <일정>` | POST `/api/calendar/events` | 오늘 종일 |
| `gc ad <키> <날짜> <일정>` | POST | 해당 날짜 종일 |
| `gc at <키> <날짜> <HHmm> <일정>` | POST | 1시간 일정 |

POST Body 예:

```json
{
  "calendarKey": 6,
  "summary": "팀 미팅",
  "date": "20260620",
  "time": "0930"
}
```

### 12.3 캘린더 키

`gc c` 목록의 **왼쪽 숫자** = 캘린더 이름 앞 번호 (`6.` → 키 `6`).  
백엔드 `GoogleCalendarService.parseCalendarKey()`와 동일 규칙.

### 12.4 일정 목록 포맷

```text
2026-06-16 종일 | 6. (회사) 업무스케줄 | 회의
2026-06-16 14:00 | 6. (회사) 업무스케줄 | 미팅
                                    ← groupBreak (날짜 바뀔 때 여백)
2026-06-17 09:00 | 6. (회사) 업무스케줄 | 출장
```

### 12.5 파싱 순서 (중요)

접두어가 겹치지 않도록 **아래 순서**로 검사한다.

1. `gc` (단독)
2. `gc c`
3. `gc at` → `gc ad` → `gc a`
4. `gc lcb`
5. `gc lc`
6. `gc l`

`at`/`ad`/`a`를 `lc`보다 먼저 검사하는 이유: `ad`, `at`가 `a`로 시작하기 때문.

### 12.6 Google Calendar 연결 확인

gc 명령 실행 전 `ensureCalendarConnected()`:

- GET `/api/calendar/status`
- `connected: false` → 에러 줄 반환, API 호출 중단

## 16. 실행 예시 — `gc lc 6` 전체 흐름

```text
[1] 사용자: "gc lc 6" 입력 → Enter

[2] TerminalView.runCommand()
    - commandBlocks에 { command: "gc lc 6", results: [] } 추가
    - isRunning = true

[3] executeTerminalCommand("gc lc 6", "http://localhost:8080")
    - group = "gc"
    - executeGcCommand() 호출

[4] executeGcCommand()
    - rest = "lc 6"
    - lcMatch → calendarKey=6, 날짜 없음
    - ensureCalendarConnected() → /api/calendar/status
    - fetchGcEvents(..., calendarKey=6)
      → GET /api/calendar/events?calendarKey=6
    - formatCalendarEvents() → TerminalLineDraft[]

[5] TerminalView
    - block.results = 줄 배열 (id 부여)
    - isRunning = false
    - scrollToBottom()
```

## 17. 에러 처리 경로

```text
API 실패
  → apiClient.parseHttpError()
  → throw new Error(한글 메시지)
  → index.ts catch → { type: 'error', lines: [...] }
  → TerminalView 표시

TerminalView runCommand catch
  → executeTerminalCommand 밖 예외도 에러 줄로 표시
```
