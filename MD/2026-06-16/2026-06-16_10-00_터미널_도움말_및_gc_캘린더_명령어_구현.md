# 터미널 도움말 및 gc 캘린더 명령어 구현

## 개요

홈 터미널에 아래 기능을 추가·개선했다.

1. **도움말 (`/?`)** — Map 기반 명령어 도움말
2. **`gc c`** — 내 캘린더 목록 (키 부여)
3. **`gc lc` / `gc lcb`** — 캘린더 키 기준 일정 조회
4. **에러 표시 개선** — API 실패·404 시 터미널에 메시지 출력

프론트엔드(`mtools-ui`)와 백엔드(`mtools-api`) 모두 변경했다.

---

## 1. 도움말 (`/?`)

### 동작

| 입력 | 출력 |
|------|------|
| `/?` | 등록된 최상위 명령어 목록 |
| `/? <키워드>` | 해당 명령어 상세 도움말 |
| `/? unknown` | `도움말을 찾을 수 없습니다 : unknown` |

### 출력 예시

**`/?`**

```text
cls     화면지우기
bd      게시판
gc      구글캘린더
md      일자별로그
```

**`/? gc`**

```text
gc              오늘일정
gc l            구글 캘린더 1주일 일정
gc l <YYYYMMDD> 해당 날짜 일정
gc c            내 캘린더 목록
gc lc <키>      캘린더 키 기준 1주일 일정
gc lc <키> <YYYYMMDD> 캘린더 키 기준 해당 날짜 일정
gc lcb <키> <YYYYMMDD> <YYYYMMDD> 캘린더 키 기준 날짜 범위 일정
```

### 구현 구조

도움말 데이터는 **`Map` 한 곳**에서 관리한다. 새 명령 추가 시 `TERMINAL_HELP`에 항목만 넣으면 `/?`, `/? <키워드>`에 자동 반영된다.

| 파일 | 역할 |
|------|------|
| `mtools-ui/src/terminalHelp.ts` | `TERMINAL_HELP` Map, `executeHelpCommand()` |
| `mtools-ui/src/terminalCommands.ts` | `isHelpCommand()`로 `/?` 분기 |

```typescript
export type CommandHelpEntry = {
  description: string           // /? 목록용 짧은 설명
  detailLines: readonly string[] // /? <키워드> 상세 줄
}

export const TERMINAL_HELP = new Map<string, CommandHelpEntry>([
  ['cls', { description: '화면지우기', detailLines: ['cls             화면지우기'] }],
  ['bd',  { /* ... */ }],
  ['gc',  { /* ... */ }],
  ['md',  { /* ... */ }],
])
```

### 새 명령 도움말 추가 방법

`terminalHelp.ts`의 `TERMINAL_HELP`에만 항목을 추가한다.

```typescript
[
  'api',
  {
    description: 'API 테스트',
    detailLines: [
      'api echo <id>     echo API 호출',
    ],
  },
],
```

## 2. Google Calendar 명령어 (`gc`)

### 전체 명령어表

| 명령어 | 설명 |
|--------|------|
| `gc` | 오늘 일정 (전체 캘린더) |
| `gc l` | 1주일 일정 (전체 캘린더) |
| `gc l <YYYYMMDD>` | 해당 날짜 일정 (전체 캘린더) |
| `gc c` | 내 캘린더 목록 (0, 1, 2… 키 부여) |
| `gc lc <키>` | 해당 캘린더 키 기준 1주일 일정 |
| `gc lc <키> <YYYYMMDD>` | 해당 캘린더 키 기준 특정 날짜 일정 |
| `gc lcb <키> <YYYYMMDD> <YYYYMMDD>` | 해당 캘린더 키 기준 날짜 범위 일정 |

날짜 형식은 **`YYYYMMDD`** (예: `20260616`).

### `gc c` — 캘린더 목록

`gc c` 실행 시 Google Calendar에 연결된 캘린더 목록을 출력한다.  
목록 앞 숫자(0, 1, 2…)가 **캘린더 목록 키**이며, `gc lc`, `gc lcb`에서 사용한다.

```text
> gc c
0       0. (개인) 특별한 날
1       1. (개인) 흔적
2       2. (개인) 운동로그
6       6. (회사) 업무스케줄
7       7. (승현) 교육스케줄
```

- 왼쪽: 목록 키 — 캘린더 이름 앞 숫자 (`1.` → 키 `1`, `6.` → 키 `6`)
- 오른쪽: Google Calendar에 등록된 캘린더 이름 그대로 표시
- `대한민국의 휴일` 등 번호 없는 캘린더·휴일 캘린더는 목록에서 제외

### `gc lc` / `gc lcb` — 키 기준 일정

```text
> gc lc 0
(0번 캘린더의 1주일 일정)

> gc lc 0 20260616
(0번 캘린더의 2026-06-16 일정)

> gc lcb 6 20260601 20260607
(0번 캘린더의 2026-06-01 ~ 2026-06-07 일정)
```

잘못된 키 입력 시:

```text
캘린더 목록 키를 찾을 수 없습니다: 999
```

## 3. 백엔드 API

### 추가·변경 API

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/calendar/calendars` | 캘린더 목록 (키 포함) |
| GET | `/api/calendar/events?calendarKey={n}` | 특정 캘린더만 일정 조회 |

기존 `/api/calendar/events`는 `timeMin`, `timeMax`와 함께 `calendarKey`를 선택적으로 받는다.

### 응답 예시

**GET /api/calendar/calendars**

```json
[
  {
    "key": 0,
    "calendarId": "xxx@group.calendar.google.com",
    "calendarName": "1. (개인) 흔적"
  }
]
```

**GET /api/calendar/events?calendarKey=0**

- `calendarKey` 생략: 기존과 동일, 전체 캘린더 일정
- `calendarKey=0`: 0번 캘린더만 조회
- 잘못된 키: HTTP 400, `캘린더 목록 키를 찾을 수 없습니다: {키}`

### 백엔드 파일

| 파일 | 변경 내용 |
|------|-----------|
| `CalendarListItem.java` | 캘린더 목록 DTO (`key`, `calendarId`, `calendarName`) |
| `GoogleCalendarService.java` | `fetchCalendarList()`, `fetchEvents(..., calendarKey)` |
| `CalendarController.java` | `/api/calendar/calendars`, `calendarKey` 쿼리 파라미터 |

캘린더 목록 키는 `listVisibleCalendars()` 결과의 **배열 인덱스(0부터)** 와 동일하다.  
`gc c`와 `gc lc`가 같은 순서·같은 키를 사용하도록 서버·클라이언트 모두 이 인덱스를 기준으로 한다.

## 4. 프론트엔드 변경

| 파일 | 변경 내용 |
|------|-----------|
| `terminalHelp.ts` | **신규** — Map 기반 도움말 |
| `terminalCommands.ts` | `gc c`, `gc lc`, `gc lcb` 파싱·API 호출, `fetchJson` 404 메시지 개선 |
| `terminalCommands.ts` | `fetchGcEvents(..., calendarKey?)` — `calendarKey` 쿼리 전달 |
| `types.ts` | `CalendarListItem` 타입 추가 |
| `TerminalView.vue` | `runCommand` catch — 예외 시 에러 줄 표시 |

### 명령 파싱 순서 (`executeGcCommand`)

`lc`, `lcb`가 `l`과 겹치지 않도록 아래 순서로 매칭한다.

1. `gc` (단독)
2. `gc c`
3. `gc lcb <키> <from> <to>`
4. `gc lc <키> [YYYYMMDD]`
5. `gc l [YYYYMMDD]`

## 5. 트러블슈팅 — `gc c` 무반응

### 원인

백엔드에 `/api/calendar/calendars` API 코드는 추가됐지만, **서버를 재시작하지 않아** 404가 발생했다.  
기존 `gc`, `gc l`은 이미 있던 API라 정상 동작했고, `gc c`만 실패한 것처럼 보였다.

### 조치

1. Spring Boot 재시작 (`./gradlew bootRun`)
2. `fetchJson` — 404 시 짧은 안내 메시지 출력
3. `TerminalView.vue` — `runCommand`에 `catch` 추가

백엔드 Java 코드 수정 후에는 **반드시 API 서버 재시작**이 필요하다.

## 6. 확인 방법

```text
/?              # 전체 도움말
/? gc           # gc 상세 도움말
gc c            # 캘린더 목록
gc lc 0         # 0번 캘린더 1주일
gc lc 0 20260616
gc lcb 6 20260601 20260607
```

Google Calendar 연결이 되어 있어야 `gc c`, `gc lc`, `gc lcb`가 동작한다.  
미연결 시: `Google Calendar가 연결되지 않았습니다. 일정관리 메뉴에서 연결하세요.`
