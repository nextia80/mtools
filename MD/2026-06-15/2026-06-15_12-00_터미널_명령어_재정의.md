# 홈 터미널 명령어 재정의

## 개요

홈 터미널 명령어를 **짧은 접두어(`bd`, `gc`, `md`)** 기준으로 재정의했다.

- 구현 위치: `mtools-ui/src/terminalCommands.ts`
- 백엔드 터미널 API는 없으며, 기존 REST API를 호출한다.
- `bd a`, `bd m` 팝업 UI는 **추후 구현** (현재는 안내 메시지 + 임시 화면 이동)

---

## 공통

| 명령어 | 설명 |
|--------|------|
| `/?` | 도움말 |
| `cls` | 화면 지우기 |

## 게시판 (`bd`)

| 명령어 | 설명 |
|--------|------|
| `bd l` | 게시판 상위 **20개** 제목만 조회 |
| `bd l <n>` | 게시판 상위 **n개** 제목만 조회 |
| `bd lb <n>~<m>` | 상위 정렬 기준 **n~m번째** 게시물 제목만 출력 (예: `bd lb 1~3`) |
| `bd g <id>` | 해당 ID 게시물 **전체 정보** 출력 |
| `bd a` | 게시글 등록 팝업 *(준비 중)* |
| `bd m <id>` | 게시글 수정 팝업 *(준비 중)* |
| `bd d <id>` | 해당 ID 게시글 삭제 |

### 출력 예시

```text
> bd l
1. 첫 번째 글 제목
2. 두 번째 글 제목
...

> bd lb 1~3
1. 첫 번째 글 제목
2. 두 번째 글 제목
3. 세 번째 글 제목

> bd g 1
[1] 제목
내용...
생성: 2026. 6. 15. ...
```

### API

- `GET /api/boards` — 목록·상세 조회
- `DELETE /api/boards/{id}` — 삭제

## Google Calendar (`gc`)

| 명령어 | 설명 |
|--------|------|
| `gc l` | Google Calendar **1주일** 일정 조회 |
| `gc l <YYYYMMDD>` | 해당 날짜 일정 조회 (예: `gc l 20260615`) |

### API

- `GET /api/calendar/status` — 연결 여부 확인
- `GET /api/calendar/events` — 기본 7일
- `GET /api/calendar/events?timeMin=...&timeMax=...` — 특정 날짜

## 일자별 LOG (`md`)

### 키워드 검색 규칙

- `*`는 **LIKE 검색** (와일드카드)
- 예: 폴더 `2026-06-01` → `*01*` 입력 시 `01`이 포함된 항목 검색
- 키워드 생략 시 전체 목록

### 파일키 형식

MD 파일 목록 출력 시 **년월일_시분** 형식의 파일키를 함께 표시한다.

- 예: `2026-06-14/2026-06-14_23-59_홈_터미널_명령어_구현.md`
- 파일키: `20260614_2359`

| 명령어 | 설명 |
|--------|------|
| `md ld [*키워드*]` | 일자별 로그 **폴더(날짜) 목록** |
| `md l [*키워드*]` | MD **파일 검색** (파일키 + 경로 출력) |
| `md g <파일키워드>` | 해당 MD 파일을 **일자별 LOG 페이지**에서 열기 |

### 출력 예시

```text
> md ld
2026-06-15
2026-06-14
...

> md ld *06-14*
2026-06-14

> md l *터미널*
20260614_2359  2026-06-14/2026-06-14_23-59_홈_터미널_명령어_구현.md

> md g 20260614_2359
일자별 LOG 페이지로 이동: 2026-06-14/2026-06-14_23-59_홈_터미널_명령어_구현.md
```

### API

- `GET /api/md-files` — 파일 목록
- `GET /api/md-files/content?path=...` — 내용 조회 (페이지 이동 시 UI에서 호출)

## 명령 실행 흐름

```text
TerminalView.runCommand()
  → executeTerminalCommand(input, apiBaseUrl)
      → bd / gc / md 분기
      → API fetch
      → TerminalLineDraft[] + (선택) action 반환
  → 출력 영역에 결과 표시
  → action 있으면 App.vue handleTerminalAction() 처리
```

### action 종류

| action | 동작 |
|--------|------|
| `open-md` | 일자별 LOG(`docs`) 페이지로 이동 후 파일 선택 |
| `board-add` | 게시판 페이지로 이동, 등록 폼 초기화 *(팝업 전 임시)* |
| `board-edit` | 게시판 페이지로 이동, 해당 글 수정 모드 *(팝업 전 임시)* |

## 제거·변경된 명령어

이전 명령어는 아래와 같이 대체되었다.

| 이전 | 변경 |
|------|------|
| `help`, `?` | `/?` |
| `clear` | `cls` |
| `menu` | 제거 |
| `board list`, `board get` … | `bd l`, `bd g` … |
| `schedule status/list` | `gc l` |
| `md` (단독) | `md ld`, `md l` |
| `echo` | 제거 |
