# 홈 터미널 UX 및 명령어 작업 정리

## 개요

홈 터미널을 중심으로 아래 작업을 진행했다.

1. **⌘ + H** 단축키 — 홈 이동 + 입력창 포커스
2. **명령어 재정의** — `bd` / `gc` / `md` 접두어 체계
3. **목록 출력 형식** — 게시판 번호 표시
4. **검색 결과 클릭** — 입력창 자동 채우기
5. **상태 유지** — 다른 메뉴 이동 후에도 터미널 내용 보존

구현은 프론트엔드(`mtools-ui`)만 변경했으며, 백엔드 터미널 전용 API는 없다.

---

## 1. ⌘ + H 단축키

어느 메뉴에 있든 **⌘ + H** 를 누르면 홈(터미널)으로 이동하고 입력창에 포커스가 간다.

### 구현

| 파일 | 내용 |
|------|------|
| `App.vue` | `handleKeyboardShortcut`에서 `metaKey + h` 처리 |
| `App.vue` | `focusHomeTerminal()` — `setActiveView('home')` 후 `terminalViewRef.focusInput()` |
| `TerminalView.vue` | `defineExpose({ focusInput })` — 부모에서 입력창 포커스 호출 |

### 참고

macOS에서 **⌘ + H**는 시스템 기본 단축키(앱 숨기기)와 겹칠 수 있다. 브라우저가 단축키를 가로채면 동작하지 않을 수 있다.

## 2. 명령어 재정의

`terminalCommands.ts`에서 명령어를 짧은 접두어로 통일했다.

### 공통

| 명령어 | 설명 |
|--------|------|
| `/?` | 도움말 |
| `cls` | 화면 지우기 |

### 게시판 (`bd`)

| 명령어 | 설명 |
|--------|------|
| `bd l` | 상위 **20개** 제목 조회 |
| `bd l <n>` | 상위 **n개** 제목 조회 |
| `bd lb <n>~<m>` | 상위 **n~m번째** 제목 ( `-` / `~` 모두 가능 ) |
| `bd g <id>` | 게시물 전체 정보 |
| `bd a` | 등록 팝업 *(준비 중)* |
| `bd m <id>` | 수정 팝업 *(준비 중)* |
| `bd d <id>` | 삭제 |

**목록 출력 형식** (순번 없음, 게시판 번호 포함):

```text
> bd l
[7] 게시물1
[8] 게시물2
```

### Google Calendar (`gc`)

| 명령어 | 설명 |
|--------|------|
| `gc l` | 1주일 일정 |
| `gc l <YYYYMMDD>` | 해당 날짜 일정 (예: `gc l 20260615`) |

### 일자별 LOG (`md`)

| 명령어 | 설명 |
|--------|------|
| `md ld [*키워드*]` | 날짜 폴더 목록 (`*` = LIKE 검색) |
| `md l [*키워드*]` | MD 파일 검색 (파일키 + 경로) |
| `md g <파일키워드>` | 일자별 LOG 페이지에서 파일 열기 |

**파일키 형식:** `년월일_시분` (예: `20260614_2359`)

```text
> md l *터미널*
20260614_2359  2026-06-14/2026-06-14_23-59_홈_터미널_명령어_구현.md
```

### 제거된 명령어

| 이전 | 변경 |
|------|------|
| `help`, `?` | `/?` |
| `clear` | `cls` |
| `menu` | 제거 |
| `board list/get/...` | `bd l`, `bd g`, ... |
| `schedule status/list` | `gc l` |
| `echo` | 제거 |

### action (페이지 연동)

명령 실행 결과에 따라 UI 페이지로 이동할 수 있다.

| action | 트리거 | 동작 |
|--------|--------|------|
| `open-md` | `md g` | 일자별 LOG 페이지 + 파일 선택 |
| `board-add` | `bd a` | 게시판 페이지 + 등록 폼 *(팝업 전 임시)* |
| `board-edit` | `bd m` | 게시판 페이지 + 수정 모드 *(팝업 전 임시)* |

## 3. 검색 결과 클릭 → 입력창 자동 채우기

`bd l`, `bd lb`, `md l` 등 목록 출력 줄을 클릭하면, 입력창에 **부분 입력 중인 명령**에 번호/파일키가 붙는다.

### 사용 예

```text
> bd l
[7] 게시물1
[8] 게시물2

입력창: bd g     →  [7] 게시물1 클릭  →  bd g 7
입력창: md g     →  파일 줄 클릭      →  md g 20260614_2359
```

### 동작 조건

입력창이 아래 패턴일 때만 클릭이 동작한다.

- `bd g`, `bd m`, `bd d`
- `md g`

### 구현

| 파일 | 내용 |
|------|------|
| `types.ts` | `TerminalLineDraft`에 `clickValue?: string` 추가 |
| `terminalCommands.ts` | 목록 줄에 `clickValue` 설정 (게시판 ID, MD 파일키) |
| `TerminalView.vue` | `handleLineClick()` — 조건 맞으면 입력창에 값 추가 |
| `app.css` | `.terminal-text-clickable` — hover 시 녹색 하이라이트 |

## 4. 터미널 상태 유지

다른 메뉴로 이동했다가 홈으로 돌아와도 **출력 내역·입력 중 텍스트·명령 히스토리**가 유지된다.

### 원인

`TerminalView`가 `v-if`로 마운트/언마운트되면서 컴포넌트 상태가 초기화되었다.

### 해결

`App.vue`에서 `TerminalView`를 **`v-show`** 로 변경했다. DOM에서 숨기기만 하고 인스턴스는 유지한다.

```vue
<TerminalView
  v-show="activeView === 'home'"
  ref="terminalViewRef"
  ...
/>
```

다른 뷰는 `v-else-if` 대신 **`v-if`** 로 각각 독립 표시한다.

### 유지되는 상태

- 명령 실행 결과 (`lines`)
- 입력창 미완성 텍스트 (`commandInput`)
- ↑/↓ 명령 히스토리 (`commandHistory`)

## 6. 명령 실행 흐름

```text
[사용자] 입력 + Enter
    ↓
TerminalView.runCommand()
    ├── 입력 줄 추가 (type: input)
    ├── executeTerminalCommand() → API fetch → lines + action
    ├── 출력 줄 추가 (type: output | error)
    └── action → App.vue handleTerminalAction() (페이지 이동)

[사용자] 목록 줄 클릭
    ↓
TerminalView.handleLineClick()
    └── bd g / md g 등 부분 명령 + clickValue → 입력창

[사용자] ⌘ + H
    ↓
App.vue focusHomeTerminal()
    └── 홈 표시 + 입력창 포커스 (기존 상태 유지)
```

## 7. 추후 작업

- `bd a` — 게시글 등록 **팝업**
- `bd m` — 게시글 수정 **팝업**
- ⌘ + H 브라우저 단축키 충돌 시 대체 키 검토
- `md ld` 결과 클릭 → `md l` 키워드 채우기 (필요 시)

---

## 8. 확인 방법

1. 홈에서 `/?` — 새 도움말 확인
2. `bd l` → `bd g` 입력 후 목록 줄 클릭 → `bd g 7` 형태 확인
3. `md l *키워드*` → `md g` + 클릭 → 파일키 채워짐 확인
4. 게시판 등 다른 메뉴 이동 → ⌘ + H → 이전 출력·입력 유지 확인
