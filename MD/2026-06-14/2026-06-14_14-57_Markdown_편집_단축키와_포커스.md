# 2026-06-14 14:57 - Markdown 편집 단축키와 포커스

## 질문 1. 어떤 요청이 있었나?

Markdown 편집 화면에 키보드 단축키를 추가하고 싶다는 요청이 있었다.

요청한 단축키는 아래와 같다.

```text
Command + E : 편집 모드로 전환
Command + S : 저장
Command + V : 리뷰 모드로 전환
```

이후 추가로 `Command + E`를 눌렀을 때 에디터에 바로 포커스가 가면 좋겠다는 요청이 있었다.

## 질문 2. 단축키는 어떻게 동작하나?

선택된 MD 파일이 있을 때만 단축키가 동작한다.

```text
⌘E
- 일자별 MD 화면으로 전환
- 편집 모드로 전환
- Markdown textarea에 바로 포커스

⌘S
- 브라우저 기본 저장 동작 방지
- 현재 편집 내용을 MD 파일로 저장

⌘V
- 브라우저 기본 붙여넣기 동작 방지
- 리뷰 모드로 전환
```

단축키는 `window`의 `keydown` 이벤트로 처리했다.

## 질문 3. 구현 코드는 어떻게 되어 있나?

`mtools-ui/src/App.vue`에 키보드 이벤트 핸들러를 추가했다.

```ts
const handleKeyboardShortcut = (event: KeyboardEvent) => {
  if (!event.metaKey || !selectedMdPath.value) {
    return
  }

  const key = event.key.toLowerCase()

  if (key === 'e') {
    event.preventDefault()
    activeView.value = 'docs'
    setMdViewMode('edit')
    return
  }

  if (key === 's') {
    event.preventDefault()
    activeView.value = 'docs'
    void saveMdFile()
    return
  }

  if (key === 'v') {
    event.preventDefault()
    activeView.value = 'docs'
    setMdViewMode('review')
  }
}
```

`onMounted`에서 이벤트를 등록하고, `onUnmounted`에서 제거한다.

```ts
onMounted(() => {
  window.addEventListener('keydown', handleKeyboardShortcut)
})

onUnmounted(() => {
  window.removeEventListener('keydown', handleKeyboardShortcut)
})
```

## 질문 4. 에디터 포커스는 어떻게 처리했나?

Vue의 `ref`와 `nextTick`을 사용했다.

```ts
const mdEditorRef = ref<HTMLTextAreaElement | null>(null)

const focusMdEditor = async () => {
  await nextTick()
  mdEditorRef.value?.focus()
}
```

편집 모드로 전환할 때 에디터가 렌더링된 뒤 포커스를 보낸다.

```ts
const setMdViewMode = (mode: MdViewMode) => {
  mdViewMode.value = mode

  if (mode === 'edit') {
    void focusMdEditor()
  }
}
```

textarea에는 아래처럼 ref를 연결했다.

```vue
<textarea
  ref="mdEditorRef"
  v-model="editedMdContent"
  class="md-editor"
></textarea>
```

## 질문 5. 버튼 UI는 어떻게 바뀌었나?

리뷰, 편집, 저장 버튼에 단축키 힌트를 표시했다.

```text
리뷰 ⌘V
편집 ⌘E
저장 ⌘S
```

힌트는 `.shortcut` 클래스로 작게 표시한다.

## 질문 6. 편집 영역 높이는 어떻게 조정했나?

편집 모드 textarea가 너무 짧게 보이는 문제가 있었다.

그래서 문서 뷰어와 편집 영역을 뷰포트 기준 높이로 늘렸다.

```css
.md-viewer {
  min-height: calc(100vh - 11rem);
}

.md-editor {
  min-height: calc(100vh - 17rem);
}
```

이제 편집 모드에서도 화면 하단 쪽까지 넓게 편집 영역을 사용할 수 있다.
