## 질문 2. SQL 파일은 어떻게 정리했나?

처음에는 `schema.sql`, `data.sql`이 아래 위치에 있었다.

```text
mtools-api/src/main/resources/schema.sql
mtools-api/src/main/resources/data.sql
```

SQL 파일을 따로 관리하기 위해 `sql` 폴더를 만들고 아래처럼 옮겼다.

```text
mtools-api/src/main/resources/sql/schema.sql
mtools-api/src/main/resources/sql/data.sql
```

Spring Boot가 새 위치의 SQL 파일을 읽도록 `application.properties`에 아래 설정을 추가했다.

```properties
spring.sql.init.schema-locations=classpath:sql/schema.sql
spring.sql.init.data-locations=classpath:sql/data.sql
```

## 질문 3. 어떤 테이블을 만들었나?

기존 테스트용 `tbl_test` 외에 `t_board`, `t_member` 테이블을 추가했다.

### t_board

```sql
CREATE TABLE IF NOT EXISTS t_board (
	id_board NUMERIC(10),
	st_title VARCHAR(1000),
	st_text TEXT,
	yn_use VARCHAR(1),
	id_member NUMERIC(10),
	id_insert NUMERIC(10),
	dt_insert TIMESTAMP,
	id_update NUMERIC(10),
	dt_update TIMESTAMP
);
```

처음에는 제목 컬럼이 `st_titel`로 잘못 들어갈 수 있어서, 실제 DB 컬럼명을 `st_title`로 변경했다.

확인 결과 현재 DB의 `t_board` 컬럼은 아래처럼 잡혀 있다.

```text
id_board
st_title
st_text
yn_use
id_member
id_insert
dt_insert
id_update
dt_update
```

### t_member

```sql
CREATE TABLE IF NOT EXISTS t_member (
	id_member NUMERIC(10),
	st_member VARCHAR(100),
	st_name VARCHAR(1000),
	st_email VARCHAR(1000),
	st_level VARCHAR(10) DEFAULT '99',
	yn_use VARCHAR(1),
	id_insert NUMERIC(10),
	dt_insert TIMESTAMP,
	id_update NUMERIC(10),
	dt_update TIMESTAMP
);
```

`st_level`은 기본값을 `99`로 설정했다.

## 질문 4. API 테스트 화면은 어떻게 바꿨나?

기존 API 테스트 화면은 `id` 하나만 받아서 `/api/echo`를 호출하는 구조였다.

```text

```

예를 들어 Echo API는 아래처럼 테스트한다.

```text

```

```json
  "id": "a"
}
```

DB 조회 API는 입력값 없이 테스트한다.

```text
End point
/api/tbl-test
```

```text

```

응답 결과는 JSON 형태로 화면에 출력된다.

## 질문 5. API Swagger는 어떻게 연결했나?

좌측 메뉴에 `API 스웨거` 메뉴를 추가했다.

Spring Boot에는 `springdoc-openapi` 의존성을 추가했다.

```gradle
implementation 'org.springdoc:springdoc-openapi-starter-webmvc-ui:3.0.1'
```

Swagger 활성화 설정도 추가했다.

```properties
springdoc.api-docs.enabled=true
springdoc.swagger-ui.enabled=true
```

Swagger UI 주소는 아래다.

```text
http://localhost:8080/swagger-ui/index.html
```

Vue 화면에서는 `API 스웨거` 메뉴를 누르면 오른쪽에 Swagger UI를 iframe으로 보여주고, 별도 창으로 열 수 있는 링크도 제공한다.

## 질문 7. 게시판 조회 API는 어떻게 만들었나?

게시판 조회 API는 아래다.

```text
GET /api/boards
```

조회 대상 컬럼은 아래다.

```text
id_board  -> 번호
st_title  -> 제목
st_text   -> 내용
dt_insert -> 생성시간
dt_update -> 수정시간
```

조회 조건은 `yn_use`가 `N`이 아닌 글만 보여준다.

```sql
WHERE COALESCE(yn_use, 'Y') <> 'N'
```

정렬은 수정 시간이 있으면 수정 시간, 없으면 생성 시간을 기준으로 최신순으로 정렬한다.

```sql
ORDER BY
	COALESCE(dt_update, dt_insert) DESC NULLS LAST,
	id_board DESC NULLS LAST
```

## 질문 8. 게시판 CRUD API는 어떻게 구성했나?

게시판 API는 아래처럼 구성했다.

```text
GET    /api/boards          목록 조회
POST   /api/boards          등록
PUT    /api/boards/{id}     수정
DELETE /api/boards/{id}     삭제
```

등록 요청 body는 아래 형태다.

```json
{
  "title": "제목",
  "text": "내용"
}
```

수정 요청 body도 동일하다.

삭제는 실제 row 삭제가 아니라 `yn_use = 'N'`으로 변경하는 soft delete 방식이다.

```sql
UPDATE t_board
SET
	yn_use = 'N',
	id_update = 1,
	dt_update = CURRENT_TIMESTAMP
WHERE id_board = ?
```

## 질문 9. 게시판 화면은 어떻게 바뀌었나?

게시판 화면에는 아래 기능을 추가했다.

```text
제목 입력
```

게시글은 블로그처럼 표시한다.

```text

                       수정 2026. 6. 14. 오후 4:10:12
```

수정 시간이 있으면 수정 시간을 보여주고, 수정 시간이 없으면 생성 시간을 보여준다.

```text
수정 있음 -> 수정 시간 표시
수정 없음 -> 생성 시간 표시
```

## 질문 11. 중간에 어떤 문제가 있었나?

### 문제 1. SQL 초기화 오류

`schema.sql`에 PostgreSQL의 `DO $$ ... $$` 블록을 넣었더니 Spring Boot SQL 초기화기가 세미콜론 기준으로 SQL을 잘라 실행하면서 오류가 발생했다.

오류 핵심은 아래였다.

```text
Unterminated dollar quote
```

해결:

```text
schema.sql에서 DO $$ 블록 제거
단순 CREATE TABLE 문만 유지
필요한 컬럼 rename은 psql로 직접 처리
```

### 문제 2. st_title 컬럼 오류

서버 로그에서 아래 오류가 발생했다.

```text
ERROR: column "st_title" does not exist
Hint: Perhaps you meant to reference the column "t_board.st_titel".
```

원인은 DB에는 `st_titel` 컬럼이 남아 있고, 코드에서는 `st_title`을 조회하고 있었기 때문이다.

해결:

```sql
ALTER TABLE mtools.t_board RENAME COLUMN st_titel TO st_title;
```

이후 `/api/boards` 조회가 정상 응답했다.

### 문제 3. POST /api/boards 405

처음 CRUD API를 추가한 뒤 실행 중인 백엔드 서버에는 새 Java 코드가 반영되지 않아 `POST /api/boards`가 405를 반환했다.

해결:

```text
백엔드 bootRun 재시작
새 Controller 코드 반영
POST/PUT/DELETE 정상 확인
```
