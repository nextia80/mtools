# API 엔드포인트 정리

상세 API 문서는 프로젝트 루트의 `mtools-api/API.md`에도 정리되어 있다.

## 기본 정보

- Base URL: `http://localhost:8080`
- Swagger UI: `http://localhost:8080/swagger-ui/index.html`

## 엔드포인트 목록

| Method | Path | 설명 | 사용 화면 |
|--------|------|------|-----------|
| GET | `/api/echo` | 파라미터 echo 테스트 | API 테스트 |
| GET | `/api/boards` | 게시글 목록 | 게시판 |
| POST | `/api/boards` | 게시글 등록 | 게시판 |
| PUT | `/api/boards/{idBoard}` | 게시글 수정 | 게시판 |
| DELETE | `/api/boards/{idBoard}` | 게시글 삭제 | 게시판 |
| GET | `/api/md-files` | MD 파일 목록 | 일자별 MD |
| GET | `/api/md-files/content` | MD 파일 내용 조회 | 일자별 MD |
| PUT | `/api/md-files/content` | MD 파일 내용 저장 | 일자별 MD |
| GET | `/api/tbl-test` | tbl_test 조회 | (직접 호출) |

## 백엔드 구조

```text
controller → service → dao → mapper xml → DB
```

- `board`, `tbltest`: MyBatis
- `md`: 파일 시스템
- `echo`: 단순 응답

## CORS

`application.properties`에서 허용 origin 관리.

```properties
app.cors.allowed-origins=http://localhost:5173,http://localhost:5174
```

컨트롤러별 `@CrossOrigin` 하드코딩은 제거하고 `CorsConfig`에서 `/api/**` 전역 처리.

## 참고

- JPA 의존성은 있으나 현재 API 코드에서는 미사용
- `t_member` 테이블은 생성되어 있으나 API는 아직 없음
- 회원관리 화면은 placeholder 상태
