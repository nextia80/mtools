# 홈 터미널 명령어 구현 정리

## 개요

홈 메뉴에 **터미널 형태 UI**를 추가했다.

- **상단:** 입력한 명령어와 실행 결과가 쌓이는 출력 영역 (스크롤)
- **하단:** 명령어 입력창 (패널 하단 고정)
- **목적:** 게시판, 일정, MD 등 기존 메뉴 데이터를 **명령어로 조회**

스타일은 게시판(`board-form`, `board-post`)과 같은 흰색 카드 + 라운드 테두리를 사용한다.

## 명령어는 어디서 정의하나?

**모든 명령어 로직은 `terminalCommands.ts` 한 파일에 있다.**

백엔드에 별도 터미널 API는 없다. 프론트에서 문자열을 파싱한 뒤, 기존 REST API를 `fetch`로 호출한다.

### 파일 내부 구조

```text
terminalCommands.ts
├── format*()           # API 응답 → 터미널 출력 문자열 변환
├── helpLines()         # help 명령 출력 텍스트
├── menuLines()         # menu 명령 출력 텍스트
├── fetch*()            # API 호출 + format 적용
├── executeTerminalCommand()  # ★ 진입점: 명령 파싱·분기·실행
└── terminalWelcomeLines()    # 최초 환영 메시지
```

### 명령 분기 위치

`executeTerminalCommand(input, apiBaseUrl)` 함수에서 **첫 단어(command)** 와 **두 번째 단어(subCommand)** 로 분기한다.

```typescript
const parts = trimmed.split(/\s+/)
const command = parts[0]?.toLowerCase() ?? ''
const subCommand = parts[1]?.toLowerCase()
const rest = trimmed.slice(parts[0]?.length ?? 0).trim()

// 예: command === 'board' → fetchBoards()
// 예: command === 'schedule' && subCommand === 'list' → fetchScheduleList()
```

| command | subCommand | 처리 함수 | 호출 API |
|---------|------------|-----------|----------|
| `help`, `?` | — | `helpLines()` | (로컬 텍스트) |
| `clear`, `cls` | — | `{ clear: true }` 반환 | (화면만 지움) |
| `menu`, `menus`, `메뉴` | — | `menuLines()` | (로컬 텍스트) |
| `board`, `게시판` | 없음 / `list` | `fetchBoards()` | `GET /api/boards` |
| `board`, `게시판` | 숫자 id | `fetchBoardDetail()` | `GET /api/boards` 후 id 필터 |
| `member`, `회원` | — | 고정 메시지 | (API 미구현) |
| `schedule`, `calendar`, `일정` | 없음 / `status` | `fetchScheduleStatus()` | `GET /api/calendar/status` |
| `schedule`, `calendar`, `일정` | `list` / `events` | `fetchScheduleList()` | `GET /api/calendar/status` → `GET /api/calendar/events` |
| `md`, `docs`, `log` | — | `fetchMdList()` | `GET /api/md-files` |
| `echo` | JSON 문자열 | `fetchEcho()` | `GET /api/echo?...` |

## 명령어는 어떻게 수행되나?

### 전체 흐름

```text
[사용자] 하단 입력창에 명령 입력 + Enter
    ↓
TerminalView.runCommand()
    ├── 입력 줄을 lines에 type: 'input' 으로 추가
    ├── executeTerminalCommand(command, apiBaseUrl) 호출
    │       ├── 명령 파싱 (split, toLowerCase)
    │       ├── 분기에 맞는 fetch*() 실행
    │       │       └── fetchJson() → fetch(API)
    │       └── TerminalLineDraft[] 반환
    ├── clear면 출력 배열 초기화
    └── 결과 줄을 lines에 type: 'output' | 'error' 로 추가
    ↓
terminal-output 영역 스크롤을 맨 아래로 이동
```

### TerminalView.vue의 역할

UI와 **상태**만 담당한다. 명령 해석은 하지 않는다.

| 상태 | 설명 |
|------|------|
| `lines` | 화면에 보이는 모든 줄 (입력/출력/에러) |
| `commandInput` | 하단 입력창 값 |
| `commandHistory` | ↑/↓ 키로 불러올 이전 명령 |
| `isRunning` | API 호출 중 중복 실행 방지 |

`runCommand()` 핵심 코드:

```typescript
appendLines([{ type: 'input', text: command }])
const result = await executeTerminalCommand(command, props.apiBaseUrl)

if (result.clear) {
  lines.value = []
} else if (result.lines.length > 0) {
  appendLines(result.lines)
}
```

### 반환 타입

```typescript
export type TerminalCommandResult = {
  lines: TerminalLineDraft[]   // 출력할 줄들
  clear?: boolean              // true면 화면 전체 지우기
}

export type TerminalLineDraft = {
  type: 'input' | 'output' | 'error'
  text: string
}
```

- `input`: 사용자가 친 명령 (TerminalView가 추가)
- `output`: 정상 결과
- `error`: 오류·사용법 안내 (빨간색 표시)

## API 호출 공통 처리

```typescript
const fetchJson = async <T>(url: string): Promise<T> => {
  const response = await fetch(url)
  if (!response.ok) {
    const message = await response.text()
    throw new Error(message || `요청 실패: ${response.status}`)
  }
  return (await response.json()) as T
}
```

- HTTP 오류는 `executeTerminalCommand`의 `catch`에서 `type: 'error'` 줄로 변환
- `echo` 명령의 JSON 파싱 실패는 `SyntaxError`로 별도 메시지 처리

## 결과 포맷 예시

### board (게시글 목록)

API: `GET /api/boards` → `BoardPost[]`

터미널 출력:

```text
[1] 제목
    내용...
    생성: 2026. 6. 14. ...

[2] 다른 글
    ...
```

### board 3 (게시글 상세)

목록 API로 전체 조회 후 `idBoard === 3` 인 항목만 필터 (별도 GET by id API 없음)

### schedule status

API: `GET /api/calendar/status`

```text
Google Calendar: 연결됨 (user@gmail.com)
```

### schedule list

1. `GET /api/calendar/status` — 미연결이면 에러 메시지
2. `GET /api/calendar/events` — 7일 일정

```text
2026-06-14 09:00 | 개인 | 회의
    메모 내용
```

### md

API: `GET /api/md-files`

```text
2026-06-14/파일명.md (1234 bytes, 2026. 6. 14. ...)
```

### echo

API: `GET /api/echo?id=a` (JSON 파라미터를 query string으로 변환)

```text
echo {"id":"a"}
```

## 새 명령어 추가 방법

1. **`terminalCommands.ts`에 fetch 함수 추가**  
   - `fetchJson`으로 API 호출  
   - `format*` 함수로 `TerminalLineDraft[]` 생성

2. **`executeTerminalCommand`에 분기 추가**  
   ```typescript
   if (command === 'member') {
     return { lines: await fetchMembers(apiBaseUrl) }
   }
   ```

3. **`helpLines()`에 사용법 한 줄 추가**

4. (선택) **`menuLines()`에 메뉴 매핑 추가**

백엔드 API가 새로 필요하면 `mtools-api`에 Controller를 추가한 뒤, 위 1~3 단계만 프론트에서 연결하면 된다.

## 정리

| 질문 | 답 |
|------|-----|
| 명령어는 어디서 정의? | `mtools-ui/src/terminalCommands.ts`의 `executeTerminalCommand()` |
