# 2026-06-14 14:30 - 좌측 메뉴 홈/API/일자별 MD 분리

## 질문 1. 어떤 요청이 있었나?

좌측 메뉴를 아래 3개로 정리하자는 요청이 있었다.

```text
홈
API 테스트
일자별 MD
```

각 메뉴의 동작은 아래처럼 정했다.

```text
홈        : 빈 화면
API 테스트 : Echo 호출 테스트 화면
일자별 MD : 상단에 MD 리스트, 하단에 Markdown 내용 표시
```

## 질문 2. UI 구조는 어떻게 바뀌었나?

기존에는 좌측 사이드바에 MD 파일 목록이 바로 표시됐다.

이번 변경 후에는 좌측 사이드바가 주요 메뉴 역할만 한다.

```text
좌측 사이드바
├─ 홈
├─ API 테스트
└─ 일자별 MD
```

MD 파일 목록은 `일자별 MD` 메뉴를 눌렀을 때 오른쪽 화면 상단에 표시된다.

## 질문 3. 홈 화면은 어떻게 처리했나?

홈 화면은 요청대로 빈 화면으로 두었다.

메뉴 상태의 기본값은 `home`이다.

```ts
const activeView = ref<ActiveView>('home')
```

따라서 처음 화면을 열면 오른쪽 작업 영역에는 별도 콘텐츠가 표시되지 않는다.

## 질문 4. API 테스트 화면은 어떻게 분리했나?

기존 `Echo 호출 테스트` 카드를 `API 테스트` 메뉴에서만 보이도록 분리했다.

동작은 기존과 동일하다.

```text
id 입력
-> 호출하기 버튼 클릭
-> /api/echo 호출
-> 응답 또는 오류 표시
```

## 질문 5. 일자별 MD 화면은 어떻게 구성했나?

`일자별 MD` 화면은 위아래 구조로 구성했다.

```text
상단
- 날짜별 MD 파일 리스트
- 새로고침 버튼

하단
- 선택한 Markdown 파일 내용
```

MD 파일은 `path`의 첫 번째 폴더명이 `YYYY-MM-DD` 형식이면 해당 값을 날짜로 사용한다.

예시:

```text
2026-06-14/2026-06-14_14-21_UI_좌측메뉴_레이아웃_정리.md
```

위 파일은 `2026-06-14` 그룹에 표시된다.

만약 path에서 날짜를 찾지 못하면 `modifiedAt` 기준 날짜를 사용한다.

## 질문 6. 변경한 파일은 무엇인가?

변경한 파일은 아래 하나다.

```text
mtools-ui/src/App.vue
```

주요 변경 내용은 아래와 같다.

```text
script
- ActiveView 타입 추가
- activeView 상태 추가
- mdFilesByDate computed 추가
- setActiveView 함수 추가

template
- 좌측 메뉴를 홈/API 테스트/일자별 MD로 변경
- 홈 빈 화면 추가
- API 테스트 전용 화면 추가
- 일자별 MD 전용 화면 추가

style
- primary-menu 스타일 추가
- date-md-list, date-group, md-chip 스타일 추가
- 문서 뷰어를 MD 리스트 아래에 배치
```
