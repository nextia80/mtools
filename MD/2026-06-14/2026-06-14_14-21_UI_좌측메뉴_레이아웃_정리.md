# 2026-06-14 14:21 - UI 좌측 메뉴 레이아웃 정리

## 질문 1. 어떤 작업을 했나?

`mtools-ui` 화면을 문서 탐색 중심의 좌측 메뉴 레이아웃으로 정리했다.

처음에는 API 테스트 카드와 MD 뷰어가 한 페이지에 카드 형태로 배치되어 있었다.

이후 사용자가 참고 사이트처럼 좌측에 메뉴가 있으면 좋겠다고 요청해서, MD 파일 목록을 좌측 사이드바 메뉴로 옮기고 오른쪽에는 작업 영역을 배치했다.

## 질문 2. 최종 화면 구조는 어떻게 바뀌었나?

최종 화면은 아래 구조로 구성된다.

```text
전체 화면
├─ 좌측 사이드바
│  ├─ mtools 브랜드 영역
│  ├─ MD 파일 개수
│  ├─ 새로고침 버튼
│  ├─ MD 파일 메뉴 목록
│  └─ API 서버 상태 안내
└─ 우측 작업 영역
   ├─ 상단 소개 영역
   ├─ Echo API 호출 테스트 카드
   ├─ 선택 문서 요약 카드
   └─ MD 문서 뷰어
```

즉, 문서 목록은 항상 좌측 메뉴에서 찾고, 선택한 문서 내용은 오른쪽에서 보는 방식이다.

## 질문 3. 좌측 메뉴에는 무엇이 들어가나?

좌측 메뉴에는 `MD` 폴더에서 읽어온 Markdown 파일 목록이 표시된다.

각 메뉴 항목은 아래 정보를 보여준다.

```text
MD 아이콘
파일명
MD 폴더 기준 상대 경로
```

선택된 파일은 초록색 배경과 강조 색상으로 표시된다.

목록은 기존과 동일하게 5초마다 자동 갱신되고, 사용자가 `새로고침` 버튼을 눌러 즉시 갱신할 수도 있다.

## 질문 4. 우측 작업 영역에는 무엇이 들어가나?

우측 작업 영역에는 아래 기능을 배치했다.

```text
상단 소개 영역
- 화면의 목적 안내
- 좌측 메뉴에서 문서를 선택하라는 안내

Echo API 호출 테스트
- id 입력
- API 호출 버튼
- 성공 또는 오류 응답 표시

문서 요약 카드
- 선택한 MD 파일명
- 마지막 수정 시간

MD 문서 뷰어
- 선택한 Markdown 파일을 HTML로 렌더링
- 파일명과 경로 표시
```

API 테스트 기능은 유지하되, 문서 탐색 화면의 보조 기능처럼 보이도록 정리했다.

## 질문 5. 변경한 파일은 무엇인가?

이번 작업에서 주로 변경한 파일은 아래 하나다.

```text
mtools-ui/src/App.vue
```

변경 범위는 크게 두 가지다.

```text
template
- 기존 카드 중심 구조 제거
- 좌측 sidebar 추가
- 우측 workspace 추가
- MD 목록을 sidebar nav 메뉴로 이동
- 선택 문서 요약 카드 추가

style
- app-shell 2단 그리드 구성
- sidebar 고정 레이아웃 추가
- menu-item active 스타일 추가
- workspace, hero, top-grid, md-card 스타일 정리
- 모바일 반응형 레이아웃 추가
```

## 질문 6. 참고한 UX 방향은 무엇인가?

사용자가 예시로 든 UPA 사이트처럼 좌측 탐색 메뉴가 먼저 보이고, 오른쪽에서 콘텐츠를 확인하는 구조를 참고했다.

목표는 아래와 같다.

```text
문서 목록을 찾기 쉽게 만들기
선택된 문서를 명확히 보여주기
API 테스트와 문서 뷰어를 한 화면에 유지하기
작은 화면에서도 사용할 수 있게 만들기
```

## 질문 7. 모바일에서는 어떻게 보이나?

화면 폭이 좁아지면 2단 레이아웃을 유지하지 않고 아래처럼 바뀐다.

```text
상단
- 사이드바 영역
- MD 메뉴 목록

하단
- 소개 영역
- API 테스트
- 문서 요약
- MD 뷰어
```

좌측 사이드바는 모바일에서 고정 높이를 해제하고, 최대 높이 안에서 스크롤되도록 처리했다.

## 질문 9. 현재 남아 있는 참고 사항은?

현재 `App.vue`에서 Markdown 렌더링을 위해 `marked`를 import하고 있다.

```ts
import { marked } from 'marked'
```

프로젝트 루트 `package.json`에는 `marked` 의존성이 있고, `mtools-ui/package.json`에는 직접 의존성으로 들어가 있지 않다.

현재 로컬 빌드는 성공하지만, UI 패키지를 독립적으로 관리하려면 나중에 `mtools-ui/package.json`에도 `marked`를 직접 추가하는 것이 더 깔끔하다.
