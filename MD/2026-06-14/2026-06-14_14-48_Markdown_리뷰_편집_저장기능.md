# 2026-06-14 14:48 - Markdown 리뷰/편집/저장 기능

## 질문 1. 어떤 요청이 있었나?

Markdown 문서를 화면에서 편집할 수 있도록 해달라는 요청이 있었다.

원하는 방식은 아래와 같다.

```text
상단에 리뷰 / 편집 버튼 추가
리뷰를 누르면 기존처럼 Markdown 렌더링 화면 표시
편집을 누르면 Markdown 원문을 수정할 수 있음
```

## 질문 2. UI는 어떻게 변경했나?

`일자별 MD` 화면의 문서 뷰어 상단에 모드 버튼을 추가했다.

```text
리뷰
편집
저장
```

동작은 아래와 같다.

```text
리뷰
- Markdown을 HTML로 렌더링해서 보여줌

편집
- Markdown 원문을 textarea에서 수정

저장
- 수정한 내용을 백엔드 API로 전송
- 실제 MD 파일에 반영
- 저장 후 리뷰 모드로 전환
```

## 질문 4. 저장은 어떻게 처리하나?

프론트에서는 `PUT /api/md-files/content` API를 호출한다.

```ts
await fetch(`${API_BASE_URL}/api/md-files/content?path=${encodeURIComponent(selectedMdPath.value)}`, {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ content: editedMdContent.value }),
})
```

저장 성공 후에는 아래 처리를 한다.

```text
mdContent 갱신
저장 완료 메시지 표시
리뷰 모드로 전환
MD 목록 새로고침
```

## 질문 5. 백엔드는 어떻게 변경했나?

`MdFileController`에 저장 API를 추가했다.

```java
@PutMapping("/api/md-files/content")
public Map<String, String> updateMdFileContent(
		@RequestParam String path,
		@RequestBody MdFileContentRequest request
) throws IOException {
	mdFileService.write(path, request.content());
	return Map.of("status", "ok");
}
```

`MdFileService`에는 파일 쓰기 메서드를 추가했다.

```java
public void write(String relativePath, String content) throws IOException {
	Path targetPath = resolveMdFile(relativePath);
	Files.writeString(targetPath, content, StandardCharsets.UTF_8);
}
```

경로 검증은 기존 `resolveMdFile`을 그대로 사용한다.

따라서 `MD` 폴더 밖의 파일이나 `.md`가 아닌 파일은 저장할 수 없다.

public record MdFileContentRequest(String content) {
}

## 질문 7. 검증은 어떻게 했나?

프론트는 아래 명령으로 빌드했다.

백엔드는 아래 명령으로 컴파일을 시도했다.

```bash

./gradlew compileJava
```

```text
Unable to locate a Java Runtime.
```

백엔드 저장 API를 사용하려면 Spring Boot 서버를 재시작해야 한다.
