## 질문 1. 어떤 요청이 있었나?

좌측 `일자별 MD` 메뉴 동작에 대해 아래 요청이 있었다.

```text
MD 목록 자동 갱신은 5초마다 할 필요 없음
새로고침 버튼이 있으니 1분 단위로 자동 갱신
가장 최근 날짜 폴더만 기본으로 열기
다른 날짜를 클릭하면 해당 날짜만 열고 나머지는 닫기
```

}, 5000)

## 질문 3. 날짜 폴더 열림 상태는 어떻게 관리하나?

열려 있는 날짜를 하나만 기억하도록 `openMdDate` 상태를 추가했다.

```ts
const openMdDate = ref('')
```

MD 파일 목록을 가져온 뒤 날짜 목록을 만들고, 현재 열려 있는 날짜가 없거나 사라졌으면 가장 최근 날짜를 연다.

```ts
const dates = Array.from(new Set(files.map(getMdFileDate))).sort((dateA, dateB) =>
  dateB.localeCompare(dateA),
)

if (!openDateExists) {
  openMdDate.value = dates[0] ?? ''
}
```

## 질문 4. 날짜를 클릭하면 어떻게 동작하나?

날짜 그룹은 `details` 태그를 사용하지만, 브라우저 기본 토글 대신 Vue 상태로 열림 여부를 제어한다.

```vue
<details :open="openMdDate === group.date">
  <summary @click.prevent="openDateGroup(group.date)">
```

사용자가 다른 날짜를 클릭하면 `openMdDate`가 그 날짜로 바뀐다.

```ts
const openDateGroup = (date: string) => {
  openMdDate.value = date
}
```

그래서 한 번에 하나의 날짜 폴더만 열린다.

## 질문 5. 파일 선택 시에는 어떻게 처리하나?

MD 파일을 클릭하면 `일자별 MD` 화면으로 전환되고, 해당 파일이 속한 날짜 폴더가 열린다.

```ts
const selectMdFile = async (path: string) => {
  activeView.value = 'docs'
  selectedMdPath.value = path

  const selectedFile = mdFiles.value.find((file) => file.path === path)

  if (selectedFile) {
    openMdDate.value = getMdFileDate(selectedFile)
  }
}
```

## 질문 6. 홈 화면 자동 전환 문제도 정리했나?

정리했다.

이전 구조에서는 MD 목록을 처음 가져올 때 첫 번째 파일을 자동 선택하면서 `일자별 MD` 화면으로 전환될 수 있었다.

이번 변경에서는 자동 선택 조건을 명확히 해서, 기본 화면인 `홈`이 빈 화면으로 유지되도록 했다.

```ts
if (selectFirstFile && firstFile) {
  await selectMdFile(firstFile.path)
}
```
