# 2026-06-14 15:46 - API 스웨거 메뉴와 Swagger UI 연결

## 질문 1. 어떤 요청이 있었나?

좌측 메뉴에서 `API 테스트` 위에 `API 스웨거` 메뉴를 추가하고, Spring Boot API에 Swagger UI를 연결해달라는 요청이 있었다.

목표는 아래와 같다.

```text
좌측 메뉴
├─ 홈
├─ API 스웨거
├─ API 테스트
└─ 일자별LOG
```

`API 스웨거`를 누르면 오른쪽 화면에서 Swagger UI를 볼 수 있게 한다.

## 질문 2. Swagger는 어떤 라이브러리를 사용했나?

Spring Boot 4.1.0 프로젝트라 `springdoc-openapi` 3.x 계열을 사용했다.

`mtools-api/build.gradle`에 아래 의존성을 추가했다.

```gradle
implementation 'org.springdoc:springdoc-openapi-starter-webmvc-ui:3.0.1'
```

Spring Boot 4는 Jackson 3 기반이므로 Swagger 관련 라이브러리도 Spring Boot 4 대응 버전을 사용해야 한다.

## 질문 3. Swagger 활성화 설정은 어디에 추가했나?

`mtools-api/src/main/resources/application.properties`에 아래 설정을 추가했다.

```properties
springdoc.api-docs.enabled=true
springdoc.swagger-ui.enabled=true
```

Swagger UI 기본 접속 주소는 아래와 같다.

```text
http://localhost:8080/swagger-ui/index.html
```

## 질문 4. UI에는 어떤 메뉴를 추가했나?

`mtools-ui/src/App.vue`의 좌측 메뉴에 `API 스웨거` 항목을 추가했다.

```text
홈
API 스웨거
API 테스트
일자별LOG
```

메뉴 타입도 기존 `home`, `api`, `docs`에서 `swagger`를 추가했다.

```ts
type ActiveView = 'home' | 'swagger' | 'api' | 'docs'
```

## 질문 5. Swagger 화면은 어떻게 연결했나?

Vue 화면에서는 Swagger UI URL을 상수로 선언했다.

```ts
const API_BASE_URL = 'http://localhost:8080'
const SWAGGER_UI_URL = `${API_BASE_URL}/swagger-ui/index.html`
```

`API 스웨거` 메뉴를 선택하면 오른쪽 화면에 iframe으로 Swagger UI를 표시한다.

```vue
<iframe class="swagger-frame" :src="SWAGGER_UI_URL" title="API Swagger UI"></iframe>
```

iframe이 정상 동작하지 않거나 별도 창에서 보고 싶을 때를 위해 `새 창으로 열기` 링크도 추가했다.

## 질문 6. 확인 방법은 무엇인가?

Swagger 의존성은 백엔드에 추가된 것이므로 Spring Boot 서버를 재시작해야 한다.

```bash
cd /Users/yujinseok/project/mtools/mtools-api
./gradlew bootRun
```

서버가 뜬 뒤 브라우저에서 직접 확인할 수 있다.

```text
http://localhost:8080/swagger-ui/index.html
```

또는 mtools UI에서 좌측 `API 스웨거` 메뉴를 클릭하면 된다.

## 질문 7. 검증은 어떻게 했나?

프론트는 아래 명령으로 빌드했다.

```bash
cd /Users/yujinseok/project/mtools/mtools-ui
npm run build
```

결과는 성공이다.

```text
vue-tsc --build 성공
vite build 성공
```

`App.vue`, `build.gradle`, `application.properties` 기준 linter 오류도 없었다.

백엔드 컴파일은 현재 실행 환경에서 Java Runtime을 찾지 못해 확인하지 못했다.

```text
Unable to locate a Java Runtime.
```

다만 사용자의 터미널에서는 Java 21로 `bootRun`이 실행된 이력이 있으므로, 백엔드 터미널에서 재시작하면 Swagger UI를 확인할 수 있다.

## 질문 8. 참고한 내용은 무엇인가?

Spring Boot 4에서는 springdoc 3.x 계열을 사용해야 한다.

참고 자료:

```text
https://springdoc.org/modules.html
https://github.com/springdoc/springdoc-openapi/issues/3200
```

Spring Boot 4와 Jackson 3 전환 시점에는 일부 springdoc 버전에서 Jackson 호환 이슈가 보고되어 있으므로, 문제가 발생하면 springdoc 버전 또는 Jackson 호환 의존성을 추가로 확인해야 한다.
