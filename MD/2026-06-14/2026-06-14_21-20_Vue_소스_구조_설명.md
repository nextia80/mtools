# Vue 소스 구조 설명

## 개요

`mtools-ui`는 Vue 3 + TypeScript + Vite로 구성된 프론트엔드다.

화면은 `App.vue`가 전체 상태와 API 호출을 관리하고, 메뉴별 화면은 `components` 아래 View 컴포넌트가 담당한다.

```text
mtools-ui/src/
├── main.ts              # 앱 진입점
├── App.vue              # 상태 관리 + 화면 조립
├── types.ts             # 공통 타입
├── assets/
│   ├── main.css         # 전역 CSS 진입점
│   ├── base.css         # 기본 스타일
│   └── app.css          # mtools 화면 스타일
└── components/
    ├── SidebarMenu.vue  # 좌측 메뉴
    ├── BoardView.vue    # 게시판
    ├── MemberView.vue   # 회원관리
    ├── SwaggerView.vue  # API 스웨거
    ├── ApiTestView.vue  # API 테스트
    └── DocsView.vue     # 일자별 MD
```

## 실행 흐름

```text
main.ts
  ↓
App.vue 마운트
  ↓
onMounted
  ├── loadMdFiles()        # MD 목록 최초 로딩
  ├── keydown 이벤트 등록   # ⌘E / ⌘S / ⌘V
  └── 1분마다 MD 목록 갱신
```

## main.ts

앱 시작 파일이다.

```typescript
import './assets/main.css'
import { createApp } from 'vue'
import App from './App.vue'

createApp(App).mount('#app')
```

CSS를 불러오고 `App.vue`를 `#app`에 마운트한다.

## types.ts

여러 컴포넌트에서 공통으로 쓰는 타입을 정의한다.

| 타입 | 설명 |
|------|------|
| `MdFile` | MD 파일 정보 (path, name, size, modifiedAt) |
| `MdFileGroup` | 날짜별 MD 파일 묶음 |
| `BoardPost` | 게시글 데이터 |
| `ActiveView` | 현재 선택된 메뉴 |
| `MdViewMode` | MD 보기 모드 (`review` / `edit`) |

## App.vue

앱의 중심 파일이다. 역할은 아래 3가지다.

1. **상태 관리** — ref, computed로 화면 데이터 보관
2. **API 호출** — fetch로 백엔드와 통신
3. **화면 조립** — 메뉴별 View 컴포넌트에 props/events 연결

### 주요 상태

| 상태 | 용도 |
|------|------|
| `activeView` | 현재 메뉴 (home, board, member, swagger, api, docs) |
| `boardPosts` | 게시글 목록 |
| `boardFormTitle`, `boardFormText` | 게시글 입력 폼 |
| `editingBoardId` | 수정 중인 게시글 ID |
| `mdFiles` | MD 파일 목록 |
| `selectedMdPath` | 선택된 MD 파일 경로 |
| `openMdDate` | 펼쳐진 날짜 그룹 |
| `editedMdContent` | 편집 중인 MD 내용 |
| `mdViewMode` | MD 리뷰/편집 모드 |
| `apiEndpoint`, `apiInput` | API 테스트 입력값 |

### 주요 함수

| 함수 | 설명 |
|------|------|
| `setActiveView()` | 메뉴 전환. board/docs 진입 시 데이터 로딩 |
| `loadBoardPosts()` | `GET /api/boards` |
| `saveBoardPost()` | `POST /api/boards` 또는 `PUT /api/boards/{id}` |
| `deleteBoardPost()` | `DELETE /api/boards/{id}` |
| `loadMdFiles()` | `GET /api/md-files` |
| `selectMdFile()` | `GET /api/md-files/content` |
| `saveMdFile()` | `PUT /api/md-files/content` |
| `callApi()` | API 테스트 화면에서 임의 endpoint 호출 |
| `handleKeyboardShortcut()` | ⌘E 편집, ⌘S 저장, ⌘V 리뷰 |

### computed

| 이름 | 설명 |
|------|------|
| `selectedMdFile` | 선택된 MD 파일 객체 |
| `mdFilesByDate` | 날짜별로 그룹핑된 MD 목록 |
| `renderedMdContent` | marked로 파싱한 HTML + 코드 복사 버튼 |

### template 구조

```text
app-shell
├── SidebarMenu
└── workspace
      ├── 홈 (빈 section)
      ├── BoardView
      ├── MemberView
      ├── SwaggerView
      ├── ApiTestView
      └── DocsView
```

`v-if` / `v-else-if`로 `activeView`에 따라 하나의 화면만 표시한다.

## SidebarMenu.vue

좌측 사이드바 전체를 담당한다.

### props

| prop | 설명 |
|------|------|
| `activeView` | 현재 메뉴 |
| `mdFiles` | MD 파일 전체 목록 |
| `mdFilesByDate` | 날짜별 그룹 |
| `openMdDate` | 열린 날짜 |
| `selectedMdPath` | 선택된 파일 |
| `isMdLoading` | 로딩 상태 |
| `mdErrorMessage` | 오류 메시지 |

### events

| event | 설명 |
|-------|------|
| `setActiveView` | 메뉴 클릭 |
| `refreshMdFiles` | MD 새로고침 |
| `openDateGroup` | 날짜 그룹 열기 + 첫 파일 선택 |
| `selectMdFile` | MD 파일 선택 |

화면 표현만 담당하고, API 호출은 `App.vue`에서 처리한다.

## BoardView.vue

게시판 화면.

- 게시글 작성/수정 폼
- 게시글 목록
- 수정/삭제 버튼
- 생성/수정 일시 표시

입력값은 `updateTitle`, `updateText` 이벤트로 `App.vue`에 전달하고, 저장/삭제는 `save`, `delete` 이벤트로 위임한다.

## ApiTestView.vue

API 호출 테스트 화면.

- End point 입력 (`/api/echo` 등)
- JSON 입력값
- 실행 버튼
- 응답/오류 결과 표시

실제 URL 조합과 fetch 호출은 `App.vue`의 `buildApiUrl()`, `callApi()`가 처리한다.

## SwaggerView.vue

Swagger UI를 iframe으로 표시한다.

- `swaggerUrl` prop으로 `http://localhost:8080/swagger-ui/index.html` 전달
- 새 창으로 열기 링크 제공

## DocsView.vue

일자별 MD 뷰어.

- 리뷰 모드: marked HTML 렌더링
- 편집 모드: textarea 편집기
- 편집 모드 진입 시 자동 포커스
- 코드 블록 복사 버튼은 `App.vue`에서 HTML 생성, 클릭 처리도 `App.vue`에서 담당

### props

| prop | 설명 |
|------|------|
| `selectedMdFile` | 선택된 파일 |
| `mdViewMode` | review / edit |
| `editedMdContent` | 편집 내용 |
| `renderedMdContent` | 렌더링된 HTML |
| `isMdContentLoading` | 내용 로딩 중 |
| `isMdSaving` | 저장 중 |

## MemberView.vue

회원관리 placeholder 화면. 아직 API 연동 없음.

## 스타일 구조

| 파일 | 역할 |
|------|------|
| `base.css` | 브라우저 기본 리셋 |
| `app.css` | mtools 레이아웃, 사이드바, 게시판, MD 뷰어 등 전체 UI |
| `main.css` | base + app import, `#app` 기본 설정 |

컴포넌트별 scoped style은 사용하지 않고, 공통 class명으로 스타일을 관리한다.

## 데이터 흐름 요약

```text
사용자 클릭/입력
  ↓
View 컴포넌트 (화면)
  ↓ event
App.vue (로직)
  ↓ fetch
Spring Boot API (localhost:8080)
  ↓ response
App.vue (상태 갱신)
  ↓ props
View 컴포넌트 (화면 갱신)
```

## 키보드 단축키

MD 파일 선택 상태에서만 동작한다.

| 단축키 | 동작 |
|--------|------|
| ⌘E | 편집 모드 |
| ⌘S | 저장 |
| ⌘V | 리뷰 모드 |

## API 연결 위치

프론트엔드 API 기본 주소는 `App.vue`에 정의되어 있다.

```typescript
const API_BASE_URL = 'http://localhost:8080'
```

CORS 허용은 백엔드 `application.properties`의 `app.cors.allowed-origins`에서 관리한다.

## 앞으로 개선할 수 있는 부분

현재 API 로직이 `App.vue`에 모여 있다. 다음 단계로 아래처럼 분리할 수 있다.

```text
composables/
├── useBoard.ts
├── useMdFiles.ts
└── useApiTest.ts
```

그러면 `App.vue`는 화면 조립과 상태 연결만 담당하는 더 가벼운 구조가 된다.
