# 일정관리 Google Calendar 연동 및 화면 UI 정리

회원관리 아래 **일정관리** 메뉴를 추가하고, Google Calendar OAuth 2.0으로 일정을 불러온 뒤 날짜별·한 줄 형식 UI로 표시하도록 구현·개선했다.

---

## 1. 메뉴 구조

```text
홈 → 게시판 → 회원관리 → 일정관리 → API 스웨거 → API 테스트 → 일자별LOG
```

| 파일 | 역할 |
|------|------|
| `SidebarMenu.vue` | 일정관리 메뉴 (아이콘: S) |
| `ScheduleView.vue` | 일정 목록 UI |
| `App.vue` | 캘린더 API 호출, OAuth 콜백 처리 |
| `types.ts` | `CalendarEvent`, `CalendarConnectionStatus` 타입 |
| `app.css` | `.schedule-*` 스타일 |

## 2. Google Cloud Console 설정

### 필수 작업

1. **Google Calendar API** 활성화 (API 및 서비스 → 라이브러리)
2. **OAuth 클라이언트 ID** 생성 (웹 애플리케이션)
3. **승인된 리디렉션 URI** 등록:

```text
http://localhost:8080/api/calendar/oauth/callback
```

4. **Google 인증 플랫폼 → 대상**: 테스트 사용자에 본인 Gmail 추가 (Testing 상태)
5. **Google 인증 플랫폼 → 데이터 액세스**: scope 추가
   - 필수: `calendar.readonly` (Google Calendar 읽기)
   - 권장: `userinfo.email` (연결 계정 표시)
6. **인증 센터**: Testing 상태면 Google 심사 불필요 (정상)

### scope 추가 후

기존 토큰에는 Calendar 권한이 없을 수 있으므로 **연결 해제 → 재연결** 필요.

## 3. 로컬 OAuth 설정

### application.properties

```properties
spring.config.import=optional:classpath:application-local.properties

app.google.client-id=${GOOGLE_CLIENT_ID:}
app.google.client-secret=${GOOGLE_CLIENT_SECRET:}
app.google.redirect-uri=http://localhost:8080/api/calendar/oauth/callback
app.google.frontend-redirect-url=http://localhost:5173
```

### application-local.properties (Git 제외)

```properties
app.google.client-id=발급받은-클라이언트-ID
app.google.client-secret=GOCSPX-xxxxxxxx
```

환경 변수 `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`로도 설정 가능.

## 4. OAuth 흐름

```text
[Google Calendar 연결] 클릭
    ↓
GET /api/calendar/auth-url
    ↓
Google 로그인 · 권한 허용
    ↓
GET /api/calendar/oauth/callback
    ↓
토큰 DB 저장 → http://localhost:5173/?view=schedule&calendar=connected
    ↓
GET /api/calendar/events → 화면 표시
```

토큰은 `t_google_oauth` 테이블에 저장 (refresh token 지원).

## 5. API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/calendar/auth-url` | Google 로그인 URL |
| GET | `/api/calendar/oauth/callback` | OAuth 콜백 |
| GET | `/api/calendar/status` | 연결 상태 |
| GET | `/api/calendar/events` | 일정 목록 (기본: 오늘 ~ 7일) |
| DELETE | `/api/calendar/disconnect` | 연결 해제 |

## 6. 일정 조회 규칙 (백엔드)

| 항목 | 내용 |
|------|------|
| 대상 | Google Calendar **구독 캘린더 전체** (숨김 캘린더 제외) |
| 기간 | **오늘 0시 ~ 7일 후** |
| 캘린더당 | 최대 100건 |
| 정렬 | 시작 시간 기준 |

### 제외 캘린더 (`isExcludedCalendar`)

캘린더 이름에 아래 문자열이 포함되면 **조회하지 않음**:

| 키워드 | 예시 |
|--------|------|
| `달의 위상` | Google 기본 위상 캘린더 |
| `phases of the moon` | 영문 위상 캘린더 |
| `moon phases` | 영문 위상 캘린더 |
| `운동시설정보` | `(개인)운동시설정보` 등 |

## 7. 화면 표시 형식

### 레이아웃

- **날짜 제목**: 크게 표시 (예: `2026년 6월 15일 월요일`)
- **일정**: 날짜 아래 **들여쓰기** (`padding-left: 1.25rem`)
- **번호(1, 2, 3…)**: 사용하지 않음
- **쉼표(,)**: 일정 한 줄 안에서 사용하지 않음

### 한 줄 구성 (순서)

```text
[시간/날짜 뱃지] [캘린더 뱃지] [제목]
```

**시간 있는 일정 예시**

```text
2026년 6월 15일 월요일
    오전8:30~오후5:30 (회사) 업무스케줄 (6차수,6) Modeling & Analytics (ML이해/분석 활용)
```

**기간 있는 종일 일정 예시**

```text
    11월 21일~1월21일 (개인)운동시설정보 에이블짐
```
(운동시설정보 캘린더는 제외 설정됨 — 형식 참고용)

**메모**

- Google Calendar **설명(description)** 이 있으면 **한 줄 아래**에 회색 작은 글씨로 표시

```text
    오전8:30~오후5:30 (회사) 업무스케줄 제목
    메모 내용
```

## 8. 색상·스타일

| 요소 | 스타일 |
|------|--------|
| **날짜 제목** | `1.35rem`, 굵게 |
| **시간/날짜 뱃지** | 초록 배경 `#ecfdf5`, 글자 `#0f766e` |
| **캘린더 뱃지** | Google `backgroundColor`를 **16% 투명도**로 연하게 + Google 색 **테두리**, 글자 `#0f172a` (가독성) |
| **제목** | 기본 본문색 |
| **메모** | `#64748b`, `0.78rem` |

Google에서 내려준 `foregroundColor`(검정)는 그대로 쓰지 않고, 연한 배경 + 진한 글자로 통일했다.

## 10. 자주 발생한 오류

| 증상 | 원인 | 해결 |
|------|------|------|
| `access_denied` | 테스트 사용자 미등록 | **대상** → 테스트 사용자 추가 |
| `ACCESS_TOKEN_SCOPE_INSUFFICIENT` | **데이터 액세스**에 `calendar.readonly` 없음 | scope 추가 후 **재연결** |
| `redirect_uri_mismatch` | 리디렉션 URI 미등록 | Console URI 확인 |
| Google OAuth 설정 필요 | client secret 비어 있음 | `application-local.properties` 설정 |
| 운동시설/위상 일정만 보임 | primary만 조회했던 초기 버전 | 현재는 전체 캘린더 + 제외 필터 적용 |
