# 터미널 UI · 메뉴 재배치 · zsh · Git 정리

## 개요

mTools 웹 터미널 UI를 Dracula/zsh 스타일로 개선하고, 좌측 메뉴 순서를 조정했다.  
Cursor 통합 터미널(zsh) 프롬프트는 Git 상태만 보이도록 별도 설정했다.

| 구분 | 대상 | 비고 |
|------|------|------|
| 웹 터미널 | `TerminalView.vue`, `app.css` | mTools 앱 내부 |
| zsh 프롬프트 | `~/.zshrc` | Cursor/터미널 앱 — **repo 밖** |
| 메뉴·단축키 | `SidebarMenu.vue`, `App.vue` | |
| API | `apiClient.ts` | 요청 타임아웃 15초 |

## 1. mTools 웹 터미널 UI

### 스타일

- macOS 터미널 창: 신호등(빨/노/초) + `-zsh` 타이틀바
- Dracula 색상: 배경 `#282A36`, 프롬프트 `➜` 녹색
- 폰트: Menlo / monospace
- 입력창: 둥근 박스 없음, 프롬프트 바로 옆 인라인 입력

### 레이아웃

```text
➜ gc
  (결과 출력 — ➜ 오른쪽 들여쓰기)
➜ █
```

- 입력줄은 **결과 맨 아래**에 자연스럽게 이어짐 (하단 고정 아님)
- `➜`와 명령어/입력 사이 간격: `1ch` (스페이스 한 칸)
- 명령 실행 후 자동 스크롤 → 입력줄이 보이도록
- 터미널 높이: `calc(100vh - 4rem)` 고정 → 스크롤 영역만 움직임

### `cls` 동작

| 항목 | 동작 |
|------|------|
| 명령 기록 | 전부 삭제 |
| 환영 메시지 | **유지** |

```text
mtools terminal — bd / gc / md 명령어로 데이터를 조회합니다.
/? 를 입력하면 사용 가능한 명령어를 볼 수 있습니다.
➜
```

### API 타임아웃

- `apiClient.ts`: fetch 15초 초과 시 `요청 시간이 초과되었습니다.` 오류
- `isRunning` stuck 방지

## 3. Cursor 통합 터미널 (zsh)

> **웹 터미널과 별개.** `~/.zshrc` 설정은 Git repo에 포함되지 않는다.

### 테마

- Oh My Zsh + Dracula (`~/.oh-my-zsh/custom/themes/dracula`)

### 프롬프트 커스터마이즈

- `➜` 화살표: **숨김**
- 디렉터리명(`mtools` 등): **숨김**
- Git 브랜치·상태: **표시** — 예: `(main) ✓`

### `~/.zshrc` 핵심 설정

```zsh
ZSH_THEME="dracula"
source $ZSH/oh-my-zsh.sh

dracula_arrow() { :; }
dracula_directory() { :; }
```

> `DRACULA_ARROW_ICON=""` 만으로는 zsh `${var:-기본값}` 규칙 때문에 `➜`가 다시 나온다.  
> **함수 override**가 필요하다.

### 적용 방법

```bash
source ~/.zshrc
# 또는 새 터미널 탭 열기
```

### 화면 지우기 (zsh)

| 방법 | 설명 |
|------|------|
| `clear` | 화면 클리어 |
| `⌘K` | Cursor 터미널 클리어 |

(`cls`는 **mTools 웹 터미널** 전용 명령)

## 4. Git — 변경 올리기

### `git status` 해석

```text
Changes not staged for commit:   → 수정됐지만 git add 안 함
Untracked files:               → Git이 처음 보는 새 파일
no changes added to commit       → 아직 add/commit 안 함
Your branch is up to date...     → 마지막 push 시점과 원격은 같음 (로컬 미커밋 변경은 별개)
```

→ **에러가 아니라**, 커밋·push 전 정상 상태.

### 기본 순서

```bash
cd /Users/yujinseok/project/mtools
git status
git add .
git commit -m "$(cat <<'EOF'
터미널 UI 개선 및 메뉴 순서 변경

Dracula 스타일 터미널, cls 환영 메시지 유지, 메뉴 재배치, API 타임아웃
EOF
)"
git push origin main
```
