# 2026-06-13 23:31 - MD 뷰어 동작 방식

## 질문 1. MD 뷰어는 전체적으로 어떻게 동작하나?

Vue 화면에서 `MD` 폴더의 Markdown 파일을 보여주는 기능을 만들었다.

전체 흐름은 아래와 같다.

```text
Vue 화면 접속
-> Spring Boot에 MD 파일 목록 요청
-> Spring Boot가 프로젝트 MD 폴더를 읽음
-> .md 파일 목록을 JSON으로 반환
-> Vue가 왼쪽 목록에 파일 표시
-> 파일 클릭
-> Spring Boot에 해당 파일 내용 요청
-> Markdown 원문을 받아옴
-> Vue에서 Markdown을 HTML로 변환
-> 오른쪽 viewer 영역에 문서처럼 표시
```

즉, Vue가 직접 로컬 폴더를 읽는 것이 아니라 Spring Boot API를 통해 `MD` 폴더를 읽는다.

## 질문 2. 왜 Vue에서 직접 MD 폴더를 읽지 않나?

Vue는 브라우저에서 실행된다.

브라우저는 보안상 로컬 프로젝트 폴더를 마음대로 읽을 수 없다.

그래서 아래처럼 백엔드가 중간에서 파일을 읽어줘야 한다.

```text
Vue
-> API 요청
-> Spring Boot
-> 로컬 MD 폴더 읽기
-> Vue로 응답
```

이 구조를 사용하면 브라우저가 직접 파일 시스템에 접근하지 않아도 된다.

## 질문 3. MD 파일 목록 API는 무엇인가?

MD 파일 목록을 가져오는 API는 아래와 같다.

```text
GET http://localhost:8080/api/md-files
```

이 API는 `MD` 폴더 아래의 `.md` 파일들을 찾아서 목록으로 반환한다.

응답 형태는 아래와 같다.

```json
[
  {
    "path": "2026-06-13/2026-06-13_22-54_endpoint_적용과_서비스종료_QA.md",
    "name": "2026-06-13_22-54_endpoint_적용과_서비스종료_QA.md",
    "size": 3079,
    "modifiedAt": "2026-06-13T13:55:15.963Z"
  }
]
```

각 값의 의미는 아래와 같다.

```text
path       : MD 폴더 기준 상대 경로
name       : 파일명
size       : 파일 크기
modifiedAt : 마지막 수정 시간
```

## 질문 4. 백엔드 Controller는 어떻게 되어 있나?

`MdFileController`에서 MD 관련 API를 제공한다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/java/com/mtools/api/MdFileController.java
```

목록 조회 API는 아래 코드로 동작한다.

```java
@GetMapping("/api/md-files")
public List<MdFileInfo> mdFiles() throws IOException {
  return mdFileService.findAll();
}
```

파일 내용 조회 API는 아래 코드로 동작한다.

```java
@GetMapping("/api/md-files/content")
public Map<String, String> mdFileContent(@RequestParam String path) throws IOException {
  return Map.of("content", mdFileService.read(path));
}
```

즉, Controller는 요청을 받고 실제 파일 읽기는 `MdFileService`에 맡긴다.

## 질문 5. 백엔드 Service는 MD 폴더를 어떻게 찾나?

`MdFileService`에서 `MD` 폴더 위치를 잡는다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/java/com/mtools/api/MdFileService.java
```

현재 기준 경로는 아래와 같다.

```java
private final Path mdRoot = Paths.get("../MD").toAbsolutePath().normalize();
```

Spring Boot 서버는 보통 `mtools-api` 폴더에서 실행된다.

그래서 `../MD`는 아래 위치를 의미한다.

```text
mtools-api
../MD

결과:
mtools/MD
```

## 질문 6. MD 파일 목록은 어떻게 찾나?

`findAll()` 메서드가 MD 파일 목록을 찾는다.

핵심 코드는 아래와 같다.

```java
try (var stream = Files.walk(mdRoot)) {
  return stream
      .filter(Files::isRegularFile)
      .filter(path -> path.getFileName().toString().endsWith(".md"))
      .map(this::toMdFileInfo)
      .sorted(Comparator.comparing(MdFileInfo::path).reversed())
      .toList();
}
```

동작 순서는 아래와 같다.

```text
1. Files.walk(mdRoot)로 MD 폴더 하위까지 탐색
2. 일반 파일만 남김
3. 파일명이 .md로 끝나는 파일만 남김
4. 파일 정보를 MdFileInfo로 변환
5. 경로 기준 역순 정렬
6. 목록으로 반환
```

그래서 `MD/2026-06-13` 같은 하위 폴더 안에 있는 파일도 자동으로 찾을 수 있다.

## 질문 7. 새 MD 파일이 생기면 어떻게 자동으로 리스트업되나?

Vue에서 5초마다 목록 API를 다시 호출한다.

`App.vue`의 핵심 코드는 아래와 같다.

```ts
onMounted(() => {
  void loadMdFiles(true)
  mdListTimer = window.setInterval(() => {
    void loadMdFiles()
  }, 5000)
})
```

동작은 아래와 같다.

```text
화면 처음 열림
-> loadMdFiles(true) 실행
-> 첫 목록 조회
-> 5초마다 loadMdFiles() 반복 실행
-> 새 파일이 생기면 다음 조회 때 목록에 반영
```

그래서 새 `.md` 파일을 만들고 조금 기다리면 왼쪽 목록에 자동으로 나타난다.

## 질문 8. 파일을 클릭하면 어떻게 내용이 보이나?

왼쪽 목록의 파일 버튼을 클릭하면 `selectMdFile(file.path)`가 실행된다.

템플릿 코드는 아래와 같다.

```vue
<button
  v-for="file in mdFiles"
  :key="file.path"
  class="md-list-item"
  :class="{ active: file.path === selectedMdPath }"
  type="button"
  @click="selectMdFile(file.path)"
>
  <strong>{{ file.name }}</strong>
  <span>{{ file.path }}</span>
</button>
```

`selectMdFile()`은 선택한 파일 경로를 가지고 내용 API를 호출한다.

```ts
const response = await fetch(
  `${API_BASE_URL}/api/md-files/content?path=${encodeURIComponent(path)}`,
)
```

여기서 호출되는 API는 아래와 같다.

```text
GET http://localhost:8080/api/md-files/content?path=파일경로
```

응답으로 받은 `content` 값을 `mdContent`에 저장한다.

```ts
const data = (await response.json()) as { content: string }
mdContent.value = data.content
```

## 질문 9. Markdown 원문은 어떻게 문서처럼 보이나?

처음에는 Markdown 원문을 `<pre>`로 그대로 표시했다.

그래서 `# 제목`, ``` 코드블록 ``` 같은 문법이 문서처럼 보이지 않고 텍스트 그대로 보였다.

이를 해결하기 위해 `marked` 패키지를 추가했다.

```bash
npm install marked
```

`App.vue`에서는 아래처럼 import한다.

```ts
import { marked } from 'marked'
```

그리고 Markdown 원문을 HTML로 변환한다.

```ts
const renderedMdContent = computed(() => marked.parse(mdContent.value, { async: false }) as string)
```

화면에서는 `v-html`로 렌더링한다.

```vue
<div v-else-if="mdContent" class="md-content" v-html="renderedMdContent"></div>
```

이렇게 하면 Markdown 문법이 HTML로 변환되어 오른쪽 viewer에 문서처럼 표시된다.

## 질문 10. Markdown 스타일은 어디서 처리하나?

Markdown 렌더링 결과는 `.md-content` 안에 들어간다.

그래서 `App.vue`의 CSS에서 `.md-content :deep(...)` 형태로 스타일을 적용했다.

예를 들어 제목은 아래처럼 보이게 했다.

```css
.md-content :deep(h1) {
  padding-bottom: 0.5rem;
  border-bottom: 1px solid #d8dee4;
  font-size: 1.75rem;
}
```

코드블록은 아래처럼 보이게 했다.

```css
.md-content :deep(pre) {
  padding: 1rem;
  overflow: auto;
  border-radius: 8px;
  background: #f6f8fa;
}
```

`:deep()`을 쓰는 이유는 `App.vue`가 scoped style을 사용하기 때문이다.

`v-html`로 들어온 HTML에는 Vue의 scoped 속성이 자동으로 붙지 않기 때문에 `:deep()`으로 안쪽 HTML에 스타일을 적용한다.

## 질문 11. 오른쪽 viewer 세로 높이는 어떻게 동작하나?

처음에는 아래처럼 최대 높이 제한이 있었다.

```css
max-height: 65vh;
```

이 설정 때문에 내용 영역이 짧게 잘린 것처럼 보였다.

지금은 오른쪽 viewer를 flex 구조로 바꿨다.

```css
.md-viewer {
  display: flex;
  flex-direction: column;
  min-width: 0;
  overflow: hidden;
}
```

그리고 내용 영역은 남은 세로 공간을 채우게 했다.

```css
.md-content {
  flex: 1;
  min-height: 0;
  overflow: auto;
}
```

이제 viewer가 가능한 세로 공간을 끝까지 사용하고, 내용이 길면 viewer 내부에서 스크롤된다.

## 질문 12. 파일 경로 보안 처리는 왜 필요한가?

내용 조회 API는 사용자가 `path` 값을 넘긴다.

예를 들어 아래처럼 호출한다.

```text
GET /api/md-files/content?path=2026-06-13/test.md
```

문제는 누군가 아래처럼 `../`를 넣어 다른 폴더 파일을 읽으려 할 수 있다는 점이다.

```text
GET /api/md-files/content?path=../application.properties
```

그래서 `MdFileService`에서 아래 검사를 한다.

```java
Path targetPath = mdRoot.resolve(relativePath).normalize();

if (!targetPath.startsWith(mdRoot) || !targetPath.getFileName().toString().endsWith(".md")) {
  throw new IllegalArgumentException("허용되지 않는 MD 파일 경로입니다.");
}
```

검사 기준은 아래와 같다.

```text
1. 최종 경로가 MD 폴더 안에 있어야 한다
2. 파일명이 .md로 끝나야 한다
3. 실제 존재하는 일반 파일이어야 한다
```

이렇게 해서 MD 폴더 밖의 파일은 읽지 못하게 했다.

## 정리

MD 뷰어 동작의 핵심은 아래와 같다.

```text
1. Spring Boot가 MD 폴더를 읽는다
2. Vue는 API로 MD 파일 목록을 받는다
3. Vue는 5초마다 목록을 다시 받아 자동 갱신한다
4. 파일 클릭 시 해당 파일 내용을 API로 받는다
5. 받은 Markdown 원문을 marked로 HTML 변환한다
6. 오른쪽 viewer에 문서 형태로 표시한다
7. viewer는 남은 세로 공간을 채우고 내부 스크롤을 사용한다
```
