## 질문 2. `tbl_test` 테이블은 어떻게 만들었나?

`schema.sql`을 추가해서 Spring Boot 서버가 시작될 때 테이블을 만들도록 했다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/resources/schema.sql
```

내용은 아래와 같다.

```sql
CREATE TABLE IF NOT EXISTS tbl_test (
  v01 VARCHAR(100),
  v02 VARCHAR(100),
  v03 VARCHAR(100),
  v04 VARCHAR(100),
  v05 VARCHAR(100),
  v06 VARCHAR(100),
  v07 VARCHAR(100),
  v08 VARCHAR(100),
  v09 VARCHAR(100),
  v10 VARCHAR(100)
);
```

`data.sql`도 추가해서 테스트 데이터를 넣었다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/resources/data.sql
```

모든 컬럼에 `'1'` 값을 넣었다.

중요한 점은 서버를 재시작할 때마다 데이터가 계속 중복으로 들어가지 않도록 아래 조건을 사용했다.

```sql
WHERE NOT EXISTS (
  SELECT 1 FROM tbl_test
);
```

## 질문 3. Spring Boot에서 SQL 초기화 파일을 실행하려면?

PostgreSQL을 사용할 때 `schema.sql`, `data.sql`이 실행되도록 아래 설정을 추가했다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/resources/application.properties
```

추가한 설정은 아래와 같다.

```properties
spring.sql.init.mode=always
```

이 설정이 있어야 서버 시작 시 `schema.sql`, `data.sql`을 실행한다.

## 질문 4. 실제 DB 조회 코드는 어디에 있나?

DB 조회는 `TblTestService`에서 처리한다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/java/com/mtools/api/TblTestService.java
```

`JdbcTemplate`을 사용해서 SQL을 직접 실행한다.

```java
public List<Map<String, Object>> findAll() {
  return jdbcTemplate.queryForList("""
      SELECT
        v01,
        v02,
        v03,
        v04,
        v05,
        v06,
        v07,
        v08,
        v09,
        v10
      FROM tbl_test
      """);
}
```

`JdbcTemplate`은 Spring에서 제공하는 DB 조회 도구다.

SQL 실행 결과를 `List<Map<String, Object>>` 형태로 받아서 JSON 응답으로 반환한다.

## 질문 5. Controller에서는 어떻게 연결했나?

`EchoController`에 `/api/tbl-test` endpoint를 추가했다.

## 질문 6. `@CrossOrigin`은 무엇인가?

`@CrossOrigin`은 CORS 허용 설정이다.

현재 Controller에는 아래처럼 설정되어 있다.

```java
@CrossOrigin(origins = {"http://localhost:5173", "http://localhost:5174"})
```

뜻은 아래 주소에서 실행 중인 Vue 화면이 Spring Boot API를 호출할 수 있도록 허용한다는 의미다.

```text
Vue 개발 서버: http://localhost:5173
Spring Boot API 서버: http://localhost:8080
```

브라우저는 포트가 다르면 다른 출처로 본다.

그래서 Vue에서 Spring Boot API를 호출하려면 CORS 허용이 필요하다.

## 질문 7. Vue에서 MD 폴더를 직접 읽을 수 있나?

브라우저에서 실행되는 Vue는 로컬 프로젝트의 `MD` 폴더를 직접 읽을 수 없다.

그래서 Spring Boot에 MD 파일을 읽는 API를 추가했다.

추가한 API는 아래와 같다.

```text
GET http://localhost:8080/api/md-files
GET http://localhost:8080/api/md-files/content?path=파일경로
```

첫 번째 API는 MD 파일 목록을 반환한다.

두 번째 API는 선택한 MD 파일의 내용을 반환한다.

## 질문 8. MD 파일 목록 API는 어떻게 동작하나?

`MdFileService`에서 프로젝트의 `MD` 폴더를 재귀적으로 탐색한다.

파일 위치는 아래와 같다.

```text
mtools-api/src/main/java/com/mtools/api/MdFileService.java
```

핵심 동작은 다음과 같다.

```text
../MD 폴더 찾기
-> 하위 폴더까지 탐색
-> .md 파일만 필터링
-> 파일 경로, 파일명, 크기, 수정 시간을 응답
```

MD 파일 내용 조회 시에는 경로 검사를 추가했다.

`../` 같은 경로로 `MD` 폴더 밖의 파일을 읽지 못하게 하기 위해서다.

## 질문 9. Vue 화면에는 무엇을 만들었나?

`mtools-ui/src/App.vue`에 MD 문서 보기 영역을 추가했다.

화면 구성은 아래와 같다.

```text
상단: 기존 API 호출 테스트
하단: MD 문서 보기
  왼쪽: MD 파일 목록
  오른쪽: 선택한 MD 파일 내용
```

처음 화면에 들어가면 MD 목록을 불러오고, 첫 번째 파일을 자동 선택한다.

또한 5초마다 목록을 다시 불러온다.

그래서 `MD` 폴더에 새 `.md` 파일이 생성되면 자동으로 리스트에 반영된다.

## 질문 10. 화면에서 검은 영역이 많이 보였던 이유는?

처음에는 `App.vue`의 카드 폭만 넓혔지만, 실제 원인은 Vue 기본 전역 CSS였다.

`src/assets/main.css`에 아래 설정이 있었다.

```css
#app {
  max-width: 1280px;
  margin: 0 auto;
  padding: 2rem;
}
```

그리고 큰 화면에서는 `#app`을 2컬럼 grid로 만드는 설정도 있었다.

```css
@media (min-width: 1024px) {
  #app {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}
```

이 설정 때문에 화면 전체가 아니라 가운데 일부만 앱 영역처럼 보였다.

## 질문 11. 화면을 전체 폭으로 보이게 하려면 어떻게 수정했나?

`src/assets/main.css`에서 `#app` 설정을 아래처럼 바꿨다.

```css
#app {
  width: 100%;
  min-height: 100vh;
  margin: 0;
  padding: 0;
  font-weight: normal;
}
```

그리고 큰 화면에서 grid로 바꾸는 설정도 제거했다.

```css
@media (min-width: 1024px) {
  body {
    display: block;
  }

  #app {
    display: block;
    padding: 0;
  }
}
```

추가로 `src/assets/base.css`에서 OS 다크 모드일 때 배경을 검게 바꾸는 Vue 기본 설정도 제거했다.

이제 화면이 브라우저 전체 가로폭을 사용한다.
