# Vue 3 프로젝트 생성 가이드

작성일: 2026-06-13 21:29

## 목표

Vue 3 기반 사이트 프로젝트를 새로 생성한다.

이번 단계에서는 프론트엔드 프로젝트 생성까지만 진행한다.
PostgreSQL 연결은 이후 백엔드 구성 단계에서 진행한다.

## 사전 준비

먼저 Node.js가 설치되어 있어야 한다.

권장 버전:

- Node.js 20 이상
- npm 10 이상

버전 확인 명령어:

```sh
node -v
npm -v
```

Node.js가 없다면 아래 사이트에서 LTS 버전을 설치한다.

```text
https://nodejs.org
```

## 1. Vue 3 프로젝트 생성

원하는 작업 폴더로 이동한 뒤 아래 명령어를 실행한다.

```sh
npm create vue@latest
```

명령어를 실행하면 프로젝트 설정 질문이 나온다.

추천 선택:

```text
Project name: 원하는 프로젝트 이름 입력
Add TypeScript? Yes
Add JSX Support? No
Add Vue Router? Yes
Add Pinia? Yes
Add Vitest? No
Add an End-to-End Testing Solution? No
Add ESLint? Yes
Add Prettier? Yes
```

예시 프로젝트 이름:

```text
my-vue-site
```

## 2. 프로젝트 폴더로 이동

프로젝트 생성이 끝나면 생성된 폴더로 이동한다.

```sh
cd my-vue-site
```

프로젝트 이름을 다르게 만들었다면 `my-vue-site` 대신 본인이 입력한 이름을 사용한다.

## 3. 패키지 설치

필요한 의존성을 설치한다.

```sh
npm install
```

## 4. 개발 서버 실행

아래 명령어로 개발 서버를 실행한다.

```sh
npm run dev
```

정상 실행되면 터미널에 다음과 비슷한 주소가 표시된다.

```text
http://localhost:5173/
```

브라우저에서 해당 주소를 열면 Vue 기본 화면을 확인할 수 있다.

## 5. 기본 폴더 구조

Vue 프로젝트 생성 후 주요 구조는 보통 아래와 같다.

```text
my-vue-site/
├─ src/
│  ├─ assets/
│  ├─ components/
│  ├─ router/
│  ├─ stores/
│  ├─ views/
│  ├─ App.vue
│  └─ main.ts
├─ public/
├─ index.html
├─ package.json
├─ vite.config.ts
└─ tsconfig.json
```

주요 파일 설명:

- `src/main.ts`: Vue 앱이 시작되는 진입점
- `src/App.vue`: 최상위 Vue 컴포넌트
- `src/router/`: 페이지 라우팅 설정
- `src/stores/`: Pinia 상태 관리 파일
- `src/components/`: 재사용 컴포넌트
- `src/views/`: 페이지 단위 화면

## 6. 다음 단계 예고

Vue 프로젝트 생성이 끝나면 다음 단계에서 PostgreSQL을 직접 Vue에 연결하지 않고, 보통 백엔드 API 서버를 만든다.

권장 구조:

```text
Vue 3 Frontend
    ↓ HTTP API
Backend Server
    ↓
PostgreSQL
```

브라우저에서 실행되는 Vue 앱이 PostgreSQL에 직접 연결하는 방식은 보안상 권장하지 않는다.
다음 단계에서는 백엔드 서버를 어떤 기술로 만들지 정한 뒤 PostgreSQL 연결을 진행하면 된다.

추천 백엔드 후보:

- Node.js + Express
- Node.js + NestJS
- Spring Boot
- Python FastAPI

초보 단계에서는 `Node.js + Express + PostgreSQL` 조합으로 시작하기 쉽다.
