# 2026-06-13 22:54 - endpoint 적용과 서비스 종료 Q&A

## 질문 1. endpoint는 어떻게 적용해야 하나?

endpoint는 프론트엔드가 호출할 백엔드 API 주소다.

이번에 만든 API endpoint는 아래와 같다.

```text
GET http://localhost:8080/api/echo?id=a
```

Spring Boot 쪽에서는 `@GetMapping`으로 endpoint를 만든다.

```java
@GetMapping("/api/echo")
public Map<String, String> echo(@RequestParam String id) {
    return Map.of("id", id);
}
```

여기서 `/api/echo`가 endpoint 경로다.

`@RequestParam String id`는 URL 뒤에 붙는 query parameter를 받겠다는 뜻이다.

```text
http://localhost:8080/api/echo?id=a
```

위 주소에서 `id=a` 부분이 query parameter다.

Vue에서는 이 endpoint를 `fetch()`로 호출한다.

```ts
const response = await fetch(`http://localhost:8080/api/echo?id=${encodeURIComponent(id.value)}`)
```

응답 JSON은 아래처럼 받을 수 있다.

```ts
const data = await response.json()
result.value = data.id
```

전체 흐름은 다음과 같다.

```text
백엔드에서 endpoint를 만든다
-> 프론트에서 endpoint 주소를 fetch로 호출한다
-> 백엔드가 JSON으로 응답한다
-> 프론트가 응답 값을 화면에 표시한다
```

## 질문 2. Java에서 코드를 수정했으면 어떻게 해야 하나?

Java에서 아래 코드를 추가했다.

```java
System.out.println("id: " + id);
```

이 코드는 API가 호출될 때 Spring Boot 서버 터미널에 받은 `id` 값을 출력한다.

확인 순서는 다음과 같다.

```text
1. Spring Boot 서버를 실행한다
2. Vue 화면에서 API 호출 버튼을 누른다
3. Spring Boot 터미널에 로그가 찍히는지 확인한다
```

정상이라면 Spring Boot 터미널에 아래처럼 출력된다.

```text
id: a
```

직접 API를 호출해서 확인할 수도 있다.

```bash
curl "http://localhost:8080/api/echo?id=a"
```

정상 응답은 아래와 같다.

```json
{"id":"a"}
```

만약 Java 수정 내용이 반영되지 않으면 Spring Boot 서버를 껐다가 다시 실행한다.

Gradle 프로젝트라면 `mtools-api` 폴더에서 아래 명령어로 실행한다.

```bash
./gradlew bootRun
```

## 질문 3. 실행 중인 서비스를 종료하려면 어떤 단축키를 쓰나?

터미널에서 실행 중인 서비스를 종료하려면 아래 단축키를 사용한다.

```text
Ctrl + C
```

현재처럼 터미널에서 아래 명령어가 실행 중이라면:

```bash
./gradlew bootRun
```

해당 터미널을 클릭한 뒤 `Ctrl + C`를 누르면 Spring Boot 서버가 종료된다.

Vue 개발 서버도 마찬가지다.

```bash
npm run dev
```

위와 같은 명령어로 실행 중인 터미널에서 `Ctrl + C`를 누르면 종료된다.

## 기억할 점

코드를 수정한 뒤에는 아래 순서로 확인하면 된다.

```text
1. 서버가 실행 중인지 확인
2. 필요하면 서버 재시작
3. API 호출
4. 터미널 로그 확인
5. 브라우저 화면 결과 확인
```

서비스 종료는 항상 실행 중인 터미널에서 `Ctrl + C`다.
