# 2026-06-13 22:40 - UI에서 API 호출 테스트 구현

## 목표

Vue UI에서 버튼을 누르면 `id` 값을 Spring Boot API로 보내고, API가 받은 값을 그대로 응답하는 흐름을 만든다.

테스트 값:

```text
id = a
```

## 전체 흐름

```text
브라우저
  -> mtools-ui
  -> http://localhost:8080/api/echo?id=a
  -> mtools-api
  -> {"id":"a"}
  -> mtools-ui 화면에 결과 표시
```

## API 작업

새 파일을 추가했다.

```text
mtools-api/src/main/java/com/mtools/api/EchoController.java
```

역할:

- `GET /api/echo` 요청을 받음
- query parameter로 `id`를 받음
- 받은 `id`를 JSON으로 그대로 반환함

구현 내용:

```java
@RestController
@CrossOrigin(origins = {"http://localhost:5173", "http://localhost:5174"})
public class EchoController {

	@GetMapping("/api/echo")
	public Map<String, String> echo(@RequestParam String id) {
		return Map.of("id", id);
	}
}
```

`@CrossOrigin`을 추가한 이유:

- Vue 개발 서버는 `5173` 또는 `5174` 포트에서 실행됨.
- Spring Boot API는 `8080` 포트에서 실행됨.
- 브라우저 기준으로 서로 다른 origin이므로 CORS 허용이 필요함.

## UI 작업

기존 Vue 예제 화면을 API 호출 테스트 화면으로 변경했다.

수정 파일:

```text
mtools-ui/src/App.vue
```

추가한 기능:

- `id` 입력값 기본값을 `a`로 설정
- `API 호출` 버튼 추가
- 버튼 클릭 시 `http://localhost:8080/api/echo?id=a` 호출
- API 응답의 `id` 값을 화면에 표시
- 실패 시 오류 메시지 표시

핵심 호출 코드:

```ts
const response = await fetch(`http://localhost:8080/api/echo?id=${encodeURIComponent(id.value)}`)
const data = (await response.json()) as { id: string }
result.value = data.id
```

## 확인한 내용

API 직접 호출:

```bash
curl -s -i "http://localhost:8080/api/echo?id=a"
```

응답:

```json
{"id":"a"}
```

CORS 확인:

```bash
curl -s -i -H "Origin: http://localhost:5174" "http://localhost:8080/api/echo?id=a"
```

응답 헤더에 아래 값이 포함됨.

```text
Access-Control-Allow-Origin: http://localhost:5174
```

```bash
npm --prefix "/Users/yujinseok/project/mtools/mtools-ui" run build
```

결과:

```text
✓ built
```

