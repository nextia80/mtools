# 2026-06-13 21:46 - mtools 프로젝트 폴더 구조 메모

## 질문

mtools에서 Spring Boot + Vue 3 + PostgreSQL을 이용해서 만들려고 하는데, 폴더를 `mtools-ui`, `mtools-ser`, `mtools-db`처럼 나누는 것은 어떤지 질문함.

## 결정

`mtools-ser`보다는 `mtools-api`가 더 나은 선택지로 판단함.

추천 구조:

```text
mtools/
  mtools-ui/   # Vue 3 프론트엔드
  mtools-api/  # Spring Boot 백엔드 API 서버
  mtools-db/   # PostgreSQL, docker-compose, init.sql 등 DB 관련 파일
  MD/          # 프로젝트 메모와 작업 기록
```

## 이유

- `mtools-ui`는 화면/프론트엔드 역할이 명확함.
- `mtools-api`는 Spring Boot 서버가 API 서버 역할을 한다는 의미가 분명함.
- `mtools-db`는 PostgreSQL과 DB 초기화 파일을 따로 관리하기 좋음.
- 루트 `mtools`는 전체 작업공간으로 두고, 각 구성요소를 하위 폴더로 분리하는 방식이 적절함.

## Vue 3 생성 명령어

```bash
npm create vue@latest mtools-ui
```
