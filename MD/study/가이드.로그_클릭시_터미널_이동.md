# 가이드 — 로그 클릭 시 터미널 화면으로 이동

좌측 **일자별LOG**에서 MD 파일(로그)을 클릭했을 때, **일자별 MD 화면(`docs`)** 대신 **터미널 화면(`home`)** 으로 이동시키는 수정 가이드입니다.

> 소스는 직접 수정하는 전제입니다. 아래는 **어떤 파일·어떤 줄**을 바꿔야 하는지 정리한 문서입니다.

---

## 1. 먼저 확인할 것

「로그 클릭」이 **어디 클릭**인지에 따라 수정 위치가 달라집니다.

| 클릭 위치 | 현재 동작 | 관련 코드 |
|-----------|-----------|-----------|
| **A. 좌측 일자별LOG → 날짜 → 파일** | `docs`(일자별 MD) 화면 | `SidebarMenu.vue` → `App.vue` `selectMdFile` |
| **B. 터미널 `md g` 실행 후** | `docs` 화면으로 이동 | `mdCommand.ts` → `handleTerminalAction` |
| **C. 「일자별LOG」 메뉴 버튼(2번)** | `docs` 화면 | `SidebarMenu.vue` `@click setActiveView('docs')` |

이 가이드는 **A + B** 기준입니다. (가장 흔한 요구)

---

## 2. 현재 흐름 (참고)

### 좌측 파일 클릭

```text
좌측 파일 클릭
  → SidebarMenu: @click="$emit('selectMdFile', file.path)"
  → App.vue selectMdFile()
  → activeView = 'docs'   ← 여기서 일자별 MD 화면으로 감
  → MD 내용 API 로드 → DocsView 표시
```

### 터미널 `md g`

```text
md g 키워드
  → mdCommand.ts: action { type: 'open-md', path }
  → handleTerminalAction → setActiveView('docs') + selectMdFile
```

## 3. 수정할 파일

### 3-1. `mtools-ui/src/App.vue` — **가장 중요**

#### (1) `selectMdFile` 함수 (~1141행)

**지금:**

```typescript
const selectMdFile = async (path: string) => {
  activeView.value = 'docs'   // ← 이 줄
  selectedMdPath.value = path
  // ... MD content fetch ...
}
```

**바꿀 것:** `'docs'` → `'home'`

```typescript
activeView.value = 'home'
```

MD fetch·`selectedMdPath` 설정은 **그대로 두어도** 됩니다. (좌측 파일 highlight용)

**1단계**에서는 view만 바꿔도 동작 확인 가능합니다.

터미널 입력창 포커스까지 주려면:

```typescript
activeView.value = 'home'
await nextTick()
terminalViewRef.value?.focusInput()
```

> 같은 패턴: `focusHomeTerminal` 함수 (~1213행) 참고

---

#### (2) `handleTerminalAction` — `open-md` 분기 (~1250행)

**지금:**

```typescript
if (action.type === 'open-md') {
  setActiveView('docs')
  void selectMdFile(action.path)
  return
}
```

**바꿀 것:** `selectMdFile` 안에서 이미 `home`으로 바꾸면 `setActiveView('docs')` **삭제**

```typescript
if (action.type === 'open-md') {
  void selectMdFile(action.path)
  return
}
```

---

### 3-2. `mtools-ui/src/components/SidebarMenu.vue` — **UI 보조 (선택)**

#### (1) 중첩 메뉴 펼침 조건 (~68행)

**지금:**

```vue
<div class="nested-md-menu" :class="{ open: activeView === 'docs' }">
```

`home`으로 가면 메뉴가 **접힐 수 있음**. 파일 선택 후에도 목록을 보이게:

```vue
:class="{ open: activeView === 'docs' || !!selectedMdPath }"
```

(`selectedMdPath`는 이미 props로 있음)

---

#### (2) 「일자별LOG」 버튼 active 표시 (~55행)

파일 선택 후 터미널에 있어도 2번 메뉴가 active로 보이게:

```vue
:class="{ active: activeView === 'docs' || (activeView === 'home' && selectedMdPath) }"
```

---

### 3-3. `mtools-ui/src/terminal/commands/mdCommand.ts` — **메시지만 (선택)**

~110행:

```typescript
lines: [{ type: 'output', text: `일자별 LOG 페이지로 이동: ${file.path}` }],
```

터미널로 가면 문구 변경 예:

```typescript
text: `터미널에서 로그 선택: ${file.path}`
```

동작 자체는 `App.vue`의 `open-md` 처리에 따릅니다.

---

### 3-4. `mtools-ui/src/terminal/help.ts` — **도움말 (선택)**

~119행 `md g` 설명이 「일자별 LOG 페이지」라면, 터미널 이동으로 수정.

---

## 4. 건드리지 않아도 되는 파일

| 파일 | 이유 |
|------|------|
| `DocsView.vue` | `activeView === 'docs'`일 때만 표시 |
| `TerminalView.vue` | 화면 전환은 `App.vue`에서 처리 |
| API (`MdFileController` 등) | 화면 전환과 무관 |

---

## 5. 같이 정할 것 (동작 설계)

### Q1. 터미널로만 갈까, 내용도 보여줄까?

지금은 **DocsView에서만** MD HTML 렌더링합니다. `home`만 바꾸면 **내용은 안 보이고** 좌측 선택만 됩니다.

원하면 추가 작업:

| 방법 | 수정 위치 |
|------|-----------|
| 입력창에 `md g 파일키`만 채우기 | `TerminalView.vue` `defineExpose`에 `setInput` 추가 → `selectMdFile`에서 호출 |
| 터미널에 본문 출력 (`md cat` 등) | `mdCommand.ts` + `help.ts` 신규 서브커맨드 |

**권장:** 1단계는 **view만 `home`으로** → 확인 후 2단계.

---

### Q2. 「일자별LOG」 버튼(2번)은?

| 정책 | 수정 |
|------|------|
| **파일 클릭** → 터미널, **2번 버튼** → 기존 `docs` | `SidebarMenu.vue` 58행 `@click setActiveView('docs')` **그대로** |
| **2번 버튼**도 터미널 | 58행을 `setActiveView('home')` 등으로 변경 |

## 6. 수정 후 확인 순서

```text
1. npm run dev (UI) + API 실행
2. 로그인 (메뉴 2번은 로그인 필요)
3. 좌측 일자별LOG → 날짜 → 파일 클릭
   → 우측이 터미널(home)인지
   → 좌측 파일 highlight 유지되는지
4. 터미널에서 md g study → Enter
   → docs가 아니라 home 유지되는지
5. Cmd+2 → docs는 여전히 docs로 가는지 (의도대로인지)
```

## 7. 한 줄 요약

| 우선순위 | 파일 | 무엇을 |
|----------|------|--------|
| **필수** | `App.vue` `selectMdFile` | `activeView = 'docs'` → `'home'` |

| **권장** | `SidebarMenu.vue` | nested `open` / active 조건 |
| **선택** | `mdCommand.ts`, `help.ts` | 안내 문구 |
