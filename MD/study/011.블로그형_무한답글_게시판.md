# 011. 블로그형 무한답글 게시판

[008](./008.게시판_CRUD와_MyBatis.md)의 **평면 CRUD**를 **원글 + 무한 depth 답글** 구조로 확장합니다.

> 상세 일지: `MD/2026-06-16/2026-06-16_08-30_블로그형_무한답글_게시판_구현.md`

---

## 목표

- `st_pid` + `st_order`로 트리 표현
- SQL 정렬만으로 계층 목록 출력
- 원글 / 댓글 / 대댓글 무제한

---

## 1. 테이블 핵심 컬럼

| 컬럼 | 역할 |
|------|------|
| `id_board` | PK, Sequence (`1, 2, 3…`) |
| `st_pid` | **원글 그룹 ID** |
| `st_order` | **계층·순서** 문자열 |
| `st_title`, `tx_board` | 제목·본문 |
| `yn_use` | Y/N (논리 삭제) |

인덱스: `(st_pid, st_order)`

---

## 2. 데이터 규칙

### st_pid

| 구분 | 규칙 |
|------|------|
| 원글 | `st_pid = id_board` |
| 답글 | 최상위 원글의 `id_board` |

```text
id=2  pid=2   원글 "테스트2"
id=3  pid=2   테스트2 답글
id=5  pid=2   테스트2 대댓글
```

### st_order

3자리 세그먼트 `#001`:

| 글 | st_order |
|----|----------|
| 원글 | `#001` |
| 1번째 답글 | `#001#001` |
| 2번째 답글 | `#001#002` |
| 대댓글 | `#001#001#001` |

---

## 3. 조회 정렬

```sql
ORDER BY st_pid DESC, st_order ASC
```

애플리케이션에서 트리를 재구성하지 않고, **ORDER BY만**으로 들여쓰기 순서를 맞춥니다.

---

## 4. 등록 로직

### 원글

```text
id_board = nextId()
st_pid   = id_board
st_order = #001
```

### 답글 (`parentId` 전달)

```text
1. 부모 조회
2. st_pid = 부모.st_pid
3. st_order = 부모.st_order + # + (형제 max+1, 3자리)
```

Service에서 처리, Dao/MyBatis는 INSERT만.

---

## 5. API·UI

| 동작 | API |
|------|-----|
| 목록 | `GET /api/boards` (트리 순서) |
| 원글 등록 | `POST /api/boards` |
| 답글 | `POST /api/boards` + `parentId` |
| 수정·삭제 | `PUT` / `DELETE` |

터미널 `bd` 명령도 같은 API를 사용합니다.

---

## 6. 확인

```bash
# 원글
curl -s -X POST http://localhost:8080/api/boards \
  -H 'Content-Type: application/json' \
  -d '{"title":"원글","text":"본문"}'

# 답글 (parentId는 원글 id)
curl -s -X POST http://localhost:8080/api/boards \
  -H 'Content-Type: application/json' \
  -d '{"title":"답글","text":"답","parentId":"2"}'
```

psql:

```bash
PGPASSWORD=123456 psql -h localhost -U mtools -d mtools \
  -c "SELECT id_board, st_pid, st_order, st_title FROM mtools.t_board ORDER BY st_pid DESC, st_order ASC;"
```
