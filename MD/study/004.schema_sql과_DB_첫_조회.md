# 004. schema.sql과 DB 첫 조회

000에서 DB **연결**까지 했다면, 이번에는 Spring Boot **기동 시 테이블·데이터를 만들고**, API로 **PostgreSQL 값을 조회**합니다.

> 상세 일지: `MD/2026-06-13/2026-06-13_23-27_PostgreSQL조회와_MD뷰어_화면확장.md`

---

## 목표

- `schema.sql` / `data.sql`로 테이블·초기 데이터 생성
- `GET /api/tbl-test`로 DB 조회 결과를 JSON으로 받기

---

## 1. SQL 파일 위치

```text
mtools-api/src/main/resources/sql/schema.sql
mtools-api/src/main/resources/sql/data.sql
```

### schema.sql 예 (tbl_test)

```sql
CREATE TABLE IF NOT EXISTS tbl_test (
	v01 VARCHAR(100),
	v02 VARCHAR(100),
	-- ... v03 ~ v10
	v10 VARCHAR(100)
);
```

### data.sql 예

```sql
INSERT INTO tbl_test (v01, v02, ..., v10)
SELECT '1', '1', ..., '1'
WHERE NOT EXISTS (SELECT 1 FROM tbl_test);
```

`WHERE NOT EXISTS` — 서버 재시작할 때마다 데이터가 **중복 삽입되지 않게** 함.

---

## 2. application.properties

```properties
spring.sql.init.mode=always
spring.sql.init.schema-locations=classpath:sql/schema.sql
spring.sql.init.data-locations=classpath:sql/data.sql
```

| 설정 | 의미 |
|------|------|
| `init.mode=always` | 앱 기동 시 SQL 실행 |
| `schema-locations` | DDL(테이블 생성) |
| `data-locations` | DML(초기 데이터) |

> PostgreSQL 사용 시 위 설정이 없으면 `schema.sql`이 실행되지 않을 수 있습니다.

---

## 3. API — tbl_test 조회

흐름:

```text
GET /api/tbl-test
  → Controller
  → Service
  → Repository / JPA / MyBatis
  → PostgreSQL tbl_test
  → JSON 배열
```

확인:

```bash
curl -s http://localhost:8080/api/tbl-test
```

응답 예:

```json
[
  {
    "v01": "1",
    "v02": "1",
    "v03": "1"
  }
]
```

---

## 4. psql로 직접 확인

```bash
PGPASSWORD=123456 psql -h localhost -p 5432 -U mtools -d mtools \
  -c "SELECT * FROM mtools.tbl_test LIMIT 5;"
```

API 결과와 psql 결과가 같으면 **DB ↔ Spring Boot** 연결이 맞습니다.

---

## 5. JPA ddl-auto와 sql.init

| 방식 | 용도 |
|------|------|
| `spring.jpa.hibernate.ddl-auto=update` | Entity 기준 테이블 자동 변경 |
| `schema.sql` + `data.sql` | SQL 파일로 명시적 DDL/DML |

mtools는 **SQL 파일을 source of truth**로 두고, 게시판·회원 테이블도 같은 방식으로 추가합니다.

---

## 6. 막혔을 때

| 증상 | 확인 |
|------|------|
| 테이블 없음 | `init.mode`, SQL 경로, API 재시작 로그 |
| data 중복 | `WHERE NOT EXISTS` 조건 |
| 빈 배열 `[]` | data.sql 실행 여부, `\dt`로 테이블 확인 |
