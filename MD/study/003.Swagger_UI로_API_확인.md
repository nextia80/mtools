# 003. Swagger UI로 API 확인

Echo API가 동작하면, 다음은 **브라우저에서 API 목록을 보고 직접 호출**할 수 있는 Swagger UI를 붙입니다.

> 상세 일지: `MD/2026-06-14/2026-06-14_15-46_API스웨거_메뉴와_Swagger_UI_연결.md`

---

## 목표

- Spring Boot에 OpenAPI(Swagger) 문서 자동 생성
- `http://localhost:8080/swagger-ui/index.html`에서 API 테스트

---

## 1. 의존성 추가

`mtools-api/build.gradle`:

```gradle
implementation 'org.springdoc:springdoc-openapi-starter-webmvc-ui:3.0.1'
```

Spring Boot 4.x는 Jackson 3 기반이므로 springdoc도 **Boot 4 대응 버전**을 사용합니다.

---

## 2. application.properties

```properties
springdoc.api-docs.enabled=true
springdoc.swagger-ui.enabled=true
```

---

## 3. 접속 주소

API 실행 후 브라우저:

```text
http://localhost:8080/swagger-ui/index.html
```

Swagger UI에서:

- 등록된 Controller·endpoint 목록 확인
- `GET /api/echo` 선택 → `id` 입력 → Execute

---

## 4. Vue 화면에 embed (선택)

좌측 메뉴에 `API 스웨거`를 두고 iframe으로 표시할 수 있습니다.

```ts
const API_BASE_URL = 'http://localhost:8080'
const SWAGGER_UI_URL = `${API_BASE_URL}/swagger-ui/index.html`
```

```vue
<iframe :src="SWAGGER_UI_URL" title="API Swagger UI" />
```

별도 탭: `<a :href="SWAGGER_UI_URL" target="_blank">새 창으로 열기</a>`

---

## 5. Swagger vs Vue API 테스트

| 방법 | 장점 |
|------|------|
| Swagger UI | API만 빠르게 검증, 파라미터 UI |
| Vue 화면 | 실제 사용자 흐름, CORS·fetch 연습 |

둘 다 쓰는 것이 좋습니다.

---

## 6. 막혔을 때

| 증상 | 확인 |
|------|------|
| 404 on swagger-ui | springdoc 의존성, API 재시작 |
| 빈 API 목록 | `@RestController` 패키지 스캔 경로 |
| iframe 빈 화면 | X-Frame-Options — 새 창으로 열기 |
