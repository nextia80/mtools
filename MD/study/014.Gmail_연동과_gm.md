# 014. Gmail 연동과 gm

Calendar(`gc`)와 같은 패턴으로 **Gmail OAuth** + **메일관리 화면** + **`gm` 터미널 명령**을 추가합니다.

> 상세 일지: `MD/2026-06-22/2026-06-22_17-00_메일관리_Gmail_연동_정리.md`, `2026-06-22_19-00_gm_명령어_스펙_정리.md`

---

## 목표

- Gmail API OAuth (Calendar와 **별도 연결**)
- 메일 목록·상세·읽음·삭제
- 터미널 `gm` 명령

---

## 1. Calendar vs Mail OAuth

| 항목 | Calendar | Gmail |
|------|----------|-------|
| API | Google Calendar API | Gmail API |
| 토큰 테이블 | `t_google_oauth` | `t_google_gmail_oauth` |
| callback | `/api/calendar/oauth/callback` | `/api/mail/oauth/callback` |
| 터미널 | `gc` | `gm` |

**같은 OAuth 클라이언트 ID**를 쓰되, 메뉴·연결은 **각각** 해야 합니다.

---

## 2. Google Cloud Console

1. **Gmail API** 활성화
2. 리디렉션 URI **두 개** 등록:

```text
http://localhost:8080/api/calendar/oauth/callback
http://localhost:8080/api/mail/oauth/callback
```

3. Scope (데이터 액세스):
   - `gmail.modify` (읽기·읽음·삭제)
   - `userinfo.email`

`gmail.readonly`만 연결했다면 **연결 해제 → 재연결** 필요.

---

## 3. application.properties

```properties
app.google.mail-redirect-uri=http://localhost:8080/api/mail/oauth/callback
app.google.frontend-redirect-url=http://localhost:5173
```

Secret은 `application-local.properties`에만.

---

## 4. OAuth 흐름

```text
[메일관리] Gmail 연결
  ↓
GET /api/mail/auth-url
  ↓
Google 승인
  ↓
GET /api/mail/oauth/callback
  ↓
토큰 → t_google_gmail_oauth
  ↓
GET /api/mail/messages → 목록
```

---

## 5. gm 명령 (요약)

| 명령 | 동작 |
|------|------|
| `gm` | 받은편지함 최근 목록 |
| `gm -s unread` | 안 읽은 메일 |
| `gm -k 키워드` | 제목/본문 검색 |
| `gm <id>` | 메일 상세 |
| `/? gm` | 도움말 |

`cd gm` 모드에서 접두어 생략 가능 (gc와 동일 패턴).

## 7. 확인

1. Console에 Gmail API + mail callback URI
2. 메일관리 → Gmail 연결
3. 목록 표시
4. 로그인 후 터미널: `gm`

```bash
curl -s http://localhost:8080/api/mail/status
```

---

## 8. 막혔을 때

| 증상 | 확인 |
|------|------|
| Calendar는 되는데 Mail 안 됨 | Gmail API 활성화, 별도 OAuth 연결 |
| 삭제/읽음 실패 | scope `gmail.modify`, 재연결 |
| gm 연결 안 됨 | `/api/mail/status`, 로그인 여부 |
