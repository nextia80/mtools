# 002. CORS와 application.properties

[001](./001.Echo_API로_UI와_API_연결.md)에서 Vue(`5173`)가 API(`8080`)를 호출하면, 브라우저가 **다른 origin** 요청을 막을 수 있습니다. CORS 설정으로 해결합니다.

---

## 목표

Vue 개발 서버에서 Spring Boot API를 호출할 때 CORS 오류 없이 응답받기.

---

## 1. CORS가 필요한 이유

| 구분 | 주소 | origin |
|------|------|--------|
| Vue dev server | `http://localhost:5173` | `:5173` |
| Spring Boot API | `http://localhost:8080` | `:8080` |

브라우저는 **포트가 다르면 다른 사이트**로 취급합니다.

`fetch()`는 성공해도 브라우저가 응답을 JavaScript에 넘기지 않을 수 있습니다.

콘솔 예:

```text
Access to fetch at 'http://localhost:8080/api/echo?id=a'
from origin 'http://localhost:5173' has been blocked by CORS policy
```

> `curl`은 브라우저가 아니므로 CORS 검사 없음. curl 성공 + Vue 실패면 CORS 문제일 가능성이 큽니다.

---

## 2. 전역 CORS 설정 (권장)

컨트롤러마다 `@CrossOrigin`을 붙이기보다, **한 곳에서 `/api/**` 전체**를 허용합니다.

### CorsConfig.java

```text
mtools-api/src/main/java/com/mtools/api/config/CorsConfig.java
```

```java
package com.mtools.api.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class CorsConfig implements WebMvcConfigurer {

	private final String[] allowedOrigins;

	public CorsConfig(@Value("${app.cors.allowed-origins}") String[] allowedOrigins) {
		this.allowedOrigins = allowedOrigins;
	}

	@Override
	public void addCorsMappings(CorsRegistry registry) {
		registry.addMapping("/api/**")
				.allowedOrigins(allowedOrigins)
				.allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS")
				.allowedHeaders("*");
	}
}
```

### application.properties

```properties
app.cors.allowed-origins=http://localhost:5173,http://localhost:5174
```

Vite가 5173이 점유되어 있으면 **5174**로 뜨므로 둘 다 넣습니다.

---

## 3. application.properties 역할 정리

000에서 넣은 DB 설정 외에, 자주 쓰는 항목:

| 키 | 예시 | 용도 |
|----|------|------|
| `spring.datasource.url` | `jdbc:postgresql://...?currentSchema=mtools` | DB 연결 |
| `spring.datasource.username` | `mtools` | DB 계정 |
| `spring.datasource.password` | `123456` | DB 비밀번호 |
| `spring.jpa.show-sql` | `true` | 실행 SQL 로그 |
| `app.cors.allowed-origins` | `5173,5174` | CORS 허용 origin |

properties는 **코드 재컴파일 없이** 환경별 값만 바꿀 수 있다는 점이 장점입니다.

---

## 4. CORS 확인

```bash
curl -s -i -H "Origin: http://localhost:5173" \
  "http://localhost:8080/api/echo?id=a"
```

응답 헤더에 아래가 있으면 OK:

```text
Access-Control-Allow-Origin: http://localhost:5173
```

## 5. 막혔을 때

| 증상 | 확인 |
|------|------|
| 여전히 CORS 오류 | origin 포트(5173 vs 5174), API 재시작 |
| OPTIONS 403 | `allowedMethods`에 OPTIONS 포함 |
| curl OK, Vue NG | 거의 항상 CORS — properties·CorsConfig |
