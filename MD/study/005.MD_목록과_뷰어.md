# 005. MD 목록과 뷰어

mtools의 핵심 아이디어: **작업 기록 Markdown을 웹에서 본다.**  
Vue가 로컬 `MD/` 폴더를 직접 읽을 수 없으므로, Spring Boot API가 파일을 읽어줍니다.

> 상세 일지: `MD/2026-06-13/2026-06-13_23-31_MD뷰어_동작방식.md`

---

## 목표

- `GET /api/md-files` — MD 파일 목록 JSON
- `GET /api/md-files/content?path=...` — 파일 원문
- Vue에서 Markdown → HTML 렌더링

---

## 전체 흐름

```text
Vue 접속
  → GET /api/md-files
  → Spring Boot가 프로젝트 MD/ 폴더 스캔
  → JSON 목록
  → 좌측 메뉴에 파일 표시
  → 파일 클릭
  → GET /api/md-files/content?path=...
  → Markdown 원문
  → Vue에서 HTML 변환 → 우측 뷰어
```

---

## 1. 왜 API를 거치나?

Vue는 **브라우저**에서 실행됩니다. 보안상 로컬 디스크 경로에 직접 접근할 수 없습니다.

```text
Vue  ✗  로컬 MD/ 폴더
Vue  →  API  →  서버 파일시스템  ✓
```

---

## 2. API endpoint

### 목록

```text
GET http://localhost:8080/api/md-files
```

응답 예:

```json
[
  {
    "path": "2026-06-13/2026-06-13_22-54_endpoint_적용과_서비스종료_QA.md",
    "name": "2026-06-13_22-54_endpoint_적용과_서비스종료_QA.md",
    "size": 3079,
    "modifiedAt": "2026-06-13T13:55:15.963Z"
  }
]
```

| 필드 | 의미 |
|------|------|
| `path` | `MD/` 기준 상대 경로 |
| `name` | 파일명 |
| `size` | 바이트 크기 |
| `modifiedAt` | 마지막 수정 시각 |

### 내용

```text
GET http://localhost:8080/api/md-files/content?path=study/000.설치.md
```

응답: Markdown **원문 문자열** (JSON 또는 text)

---

## 3. 백엔드 구조

```text
MdFileController   ← HTTP 입구
MdFileService      ← MD/ 폴더 경로 해석, Files.readString
```

Controller 예:

```java
@GetMapping("/api/md-files")
public List<MdFileInfo> listMdFiles() { ... }

@GetMapping("/api/md-files/content")
public String getMdFileContent(@RequestParam String path) { ... }
```

---

## 4. Vue — 목록·선택·렌더링

상태 예:

```ts
type MdFile = {
  path: string
  name: string
  size: number
  modifiedAt: string
}

const mdFiles = ref<MdFile[]>([])
const selectedMdPath = ref('')
const mdContent = ref('')
```

목록 로딩:

```ts
const response = await fetch(`${API_BASE_URL}/api/md-files`)
mdFiles.value = await response.json()
```

내용 로딩 + Markdown 렌더링 라이브러리(marked, markdown-it 등)로 HTML 변환 후 `v-html` 또는 컴포넌트로 표시.

---

## 5. 확인

```bash
curl -s http://localhost:8080/api/md-files | head
curl -s "http://localhost:8080/api/md-files/content?path=study/README.md"
```

Vue 좌측에 `MD/study/` 파일이 보이고, 클릭 시 우측에 렌더링되면 성공.

---

## 6. 막혔을 때

| 증상 | 확인 |
|------|------|
| 목록 빈 배열 | `MD/` 폴더 위치, API를 `mtools-api`에서 실행했는지 |
| 404 on content | `path` 인코딩(`encodeURIComponent`) |
| HTML 깨짐 | Markdown 파서 설정, 코드 블록 이스케이프 |
