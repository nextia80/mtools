## 1. 테이블 (schema.sql)

```sql
CREATE TABLE IF NOT EXISTS t_board (
	id_board VARCHAR(100) PRIMARY KEY,
	st_pid VARCHAR(100) NOT NULL,
	st_order VARCHAR(10000) NOT NULL,
	st_title VARCHAR(1000),
	tx_board TEXT,
	yn_use VARCHAR(1) DEFAULT 'Y',
	id_insert VARCHAR(100),
	dt_insert TIMESTAMP,
	id_update VARCHAR(100),
	dt_update TIMESTAMP
);
```

004에서 `sql/schema.sql`에 추가하고 API 재시작하면 테이블이 생성됩니다.

## 3. 요청 흐름 (등록 예)

```text
Vue 화면 — 제목·내용 입력
  ↓ POST /api/boards  (JSON body)
BoardController
  ↓ @RequestBody BoardPostRequest
BoardService  — ID 생성, 검증, 트랜잭션
  ↓
BoardDao  (@Mapper 인터페이스)
  ↓
BoardMapper.xml  — INSERT SQL
  ↓
PostgreSQL t_board
  ↓
BoardPost (응답 JSON)
  ↓
Vue 목록 갱신
```

## 4. 각 레이어 역할

### Controller — API 입구

```java
@PostMapping("/api/boards")
public BoardPost createBoard(@RequestBody BoardPostRequest request) {
	return boardService.create(request);
}
```

### Request — **사용자가 보낸 입력**

```java
public record BoardPostRequest(String title, String text) {}
```

DB 조회 결과가 아니라 **화면 → 서버** 방향 데이터.

### Service — 비즈니스 흐름

- 다음 ID 조회
- insert 호출
- 저장 결과 다시 select

SQL은 직접 쓰지 않고 Dao에 위임.

### Dao — DB 접근 인터페이스

```java
@Mapper
public interface BoardDao {
	List<BoardPost> findAll();
	int insert(...);
	int update(...);
	int delete(...);
}
```

### Mapper.xml — 실제 SQL

```xml
<insert id="insert">
  INSERT INTO t_board (id_board, st_title, tx_board, ...)
  VALUES (#{idBoard}, #{title}, #{text}, ...)
</insert>
```

MyBatis가 `BoardDao.insert()` ↔ XML `id="insert"` 를 연결합니다.

## 5. Vue — POST 예

```ts
await fetch(`${API_BASE_URL}/api/boards`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title: title.value, text: text.value }),
})
```

GET은 query/body 없이 URL만, POST/PUT은 JSON body.
