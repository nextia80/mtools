# 000. 설치 — PostgreSQL · API · UI 세팅

Vue 3(UI), Spring Boot(API), PostgreSQL(DB)을 **처음부터 각각 설치하고 기본 세팅**하는 과정을 정리한 학습용 문서입니다.

> 이 글은 `git clone`으로 mtools 전체를 받아 실행하는 방법이 **아닙니다.**  
> 스택을 이해하기 위해 **구성 요소별로** 무엇을 설치하고, 어떤 명령·설정을 했는지 기록합니다.

> mtools 전체 실행 가이드: [INSTALL.md](../../INSTALL.md)

---

## 전체 그림

```text
브라우저
  ↓ HTTP (5173)
Vue 3 (mtools-ui)     ← Node.js 필요
  ↓ HTTP API (8080)
Spring Boot (mtools-api) ← Java JDK 21 필요
  ↓ JDBC (5432)
PostgreSQL              ← DB·스키마·계정 필요
```

작업 폴더 예시:

```text
mtools/                 # 원하는 이름의 작업 폴더
├─ mtools-ui/           # Vue 3
├─ mtools-api/          # Spring Boot
└─ MD/                  # 작업 기록 (선택)
```

## 1. PostgreSQL 설치 및 세팅

### 1-1. PostgreSQL 설치

- Mac: Homebrew — `brew install postgresql@16` 후 서비스 시작
- Windows: [postgresql.org](https://www.postgresql.org/download/windows/) 설치 프로그램

설치 확인:

```bash
psql --version
# 예: psql (PostgreSQL) 16.x
```

### 1-2. 실행 여부 확인

PostgreSQL이 5432 포트에서 대기 중인지 확인합니다.

```bash
pg_isready -h localhost -p 5432
```

정상이면:

```text
localhost:5432 - accepting connections
```

포트 사용 프로세스 확인:

```bash
lsof -nP -iTCP:5432 -sTCP:LISTEN
```

superuser(또는 로컬 OS 계정)로 접속 확인:

```bash
psql -h localhost -p 5432 -d postgres -c "select current_user, current_database(), version();"
```

### 1-3. DB · 스키마 · 전용 계정 만들기

API에서 쓸 설정 목표:

| 항목 | 값 |
|------|-----|
| 데이터베이스 | `mtools` |
| 스키마 | `mtools` |
| 사용자 | `mtools` |
| 비밀번호 | `123456` |
| 포트 | `5432` |

superuser로 접속한 뒤 실행:

```sql
CREATE DATABASE mtools;

\c mtools

CREATE SCHEMA IF NOT EXISTS mtools;

DO $$
BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_roles WHERE rolname = 'mtools') THEN
    CREATE ROLE mtools LOGIN PASSWORD '123456';
  ELSE
    ALTER ROLE mtools WITH LOGIN PASSWORD '123456';
  END IF;
END
$$;

ALTER DATABASE mtools OWNER TO mtools;
ALTER SCHEMA mtools OWNER TO mtools;
GRANT ALL PRIVILEGES ON DATABASE mtools TO mtools;
GRANT ALL PRIVILEGES ON SCHEMA mtools TO mtools;
```

스키마만 따로 만들 때:

```bash
psql -h localhost -p 5432 -d mtools -c "create schema if not exists mtools;"
psql -h localhost -p 5432 -d mtools -c "\dn"
```

### 1-4. 접속 확인

`mtools` 계정으로 실제 접속이 되는지 확인합니다.

```bash
PGPASSWORD=123456 psql -h localhost -p 5432 -U mtools -d mtools \
  -c "SELECT current_user, current_database(), current_schema();"
```

기대 결과:

```text
current_user     : mtools
current_database : mtools
current_schema   : mtools
```

### 1-5. 막혔을 때

| 증상 | 확인 |
|------|------|
| `connection refused` | PostgreSQL 서비스 실행 여부 |
| `password authentication failed` | 계정·비밀번호, `ALTER ROLE` 재실행 |
| `database "mtools" does not exist` | `CREATE DATABASE mtools` 재실행 |

## 2. API — Java · Spring Boot 설치 및 세팅

### 2-1. Java JDK 21 설치

- Mac: `brew install openjdk@21` 또는 [Adoptium](https://adoptium.net/)
- Windows: Adoptium 또는 Oracle JDK 21

설치 확인:

```bash
java -version
# openjdk version "21.x" ...
```

### 2-2. Spring Boot 프로젝트 생성

작업 폴더(`mtools` 등)에서 Spring Initializr를 curl로 호출합니다.

```bash
pwd
# 작업 폴더인지 확인

curl "https://start.spring.io/starter.zip" \
  -d type=gradle-project \
  -d language=java \
  -d baseDir=mtools-api \
  -d groupId=com.mtools \
  -d artifactId=mtools-api \
  -d name=mtools-api \
  -d description="mtools Spring Boot API server" \
  -d packageName=com.mtools.api \
  -d packaging=jar \
  -d javaVersion=21 \
  -d dependencies=web,devtools,data-jpa,postgresql,validation,lombok \
  -o mtools-api.zip

unzip mtools-api.zip
rm mtools-api.zip
cd mtools-api
```

선택한 설정:

| 항목 | 값 |
|------|------|
| Build | Gradle |
| Language | Java 21 |
| Group | `com.mtools` |
| Artifact | `mtools-api` |
| Package | `com.mtools.api` |

추가한 Dependencies:

| Dependency | 용도 |
|------------|------|
| Spring Web | REST API |
| DevTools | 개발 중 자동 재시작 |
| Spring Data JPA | Entity ↔ DB |
| PostgreSQL Driver | DB 연결 |
| Validation | 요청값 검증 |
| Lombok | 보일러플레이트 감소 |

> Spring Security는 처음에는 넣지 않음. API 구조와 DB 연결을 먼저 잡은 뒤 추가하는 편이 이해하기 쉽습니다.

생성 후 폴더:

```text
mtools-api/
├─ build.gradle
├─ gradlew
└─ src/
   └─ main/
      ├─ java/
      └─ resources/
         └─ application.properties
```

### 2-3. PostgreSQL 연결 설정

`src/main/resources/application.properties`에 DB 정보를 넣습니다.

```properties
spring.application.name=mtools-api

spring.datasource.url=jdbc:postgresql://localhost:5432/mtools?currentSchema=mtools
spring.datasource.username=mtools
spring.datasource.password=123456

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
```

의미:

- `url`의 `mtools`(앞) → 데이터베이스 이름
- `currentSchema=mtools` → 스키마 이름
- `username` / `password` → 1장에서 만든 계정

### 2-4. API 실행 확인

```bash
cd mtools-api
./gradlew bootRun
```

Windows: `gradlew.bat bootRun`

정상 기동 시:

- API: http://localhost:8080
- (Swagger 추가 후) http://localhost:8080/swagger-ui/index.html

터미널에 DB 연결 오류가 없고 `Started ...Application` 로그가 보이면 PostgreSQL 연결까지 성공한 것입니다.

### 2-5. 막혔을 때

| 증상 | 확인 |
|------|------|
| `Connection refused` | PostgreSQL 실행, 1장 세팅 |
| `password authentication failed` | `application.properties` 계정·비밀번호 |
| 포트 8080 사용 중 | `lsof -ti:8080 \| xargs kill -TERM` (Mac) |

## 3. UI — Node.js · Vue 3 설치 및 세팅

### 3-1. Node.js 설치

- [nodejs.org](https://nodejs.org/) — LTS(20.x 또는 22.x) 권장

설치 확인:

```bash
node -v    # v20.19+ 또는 v22.12+
npm -v
```

### 3-2. Vue 3 프로젝트 생성

작업 폴더에서 실행:

```bash
npm create vue@latest mtools-ui
```

터미널 질문 — mtools에서 실제 선택한 예:

| 질문 | 선택 | 이유 |
|------|------|------|
| TypeScript? | **Yes** | API 연동 시 타입 관리, 유지보수 |
| features (Router, Pinia 등) | **none** | 가볍게 시작, 나중에 추가 가능 |
| experimental features | **none** | 베타·실험 기능은 초반 비추 |
| blank project? | **No** | 예제 코드로 구조 파악에 도움 |

생성 후:

```bash
cd mtools-ui
npm install
npm run dev
```

정상 실행 시:

```text
http://localhost:5173/
```

브라우저에서 Vue 기본 화면이 보이면 UI 세팅 완료입니다.

### 3-3. 기본 폴더 구조

```text
mtools-ui/
├─ src/
│  ├─ assets/
│  ├─ components/
│  ├─ App.vue
│  └─ main.ts
├─ public/
├─ index.html
├─ package.json
└─ vite.config.ts
```

| 경로 | 역할 |
|------|------|
| `src/main.ts` | 앱 진입점 |
| `src/App.vue` | 최상위 컴포넌트 |
| `src/components/` | 재사용 UI |
| `vite.config.ts` | Vite(개발 서버) 설정 |

### 3-4. 막혔을 때

| 증상 | 확인 |
|------|------|
| `node: command not found` | Node.js 설치·PATH |
| `EACCES` npm 오류 | 권한, `nvm` 사용 여부 |
| 5173 포트 사용 중 | 다른 dev 서버 종료 |

## 4. 세 가지를 동시에 띄울 때

학습 단계에서 **각각이 정상인지** 확인한 뒤, 아래 순서로 실행합니다.

| 순서 | 구성 요소 | 명령 | 포트 |
|------|-----------|------|------|
| 1 | PostgreSQL | OS 서비스 실행 | 5432 |
| 2 | API | `cd mtools-api && ./gradlew bootRun` | 8080 |
| 3 | UI | `cd mtools-ui && npm run dev` | 5173 |

터미널은 API·UI 각각 **1개씩** 필요합니다.

Vue에서 API를 호출하려면 이후 단계(Echo API, CORS 설정 등)에서 연결합니다.  
→ 다음 글: [001. Echo API로 UI ↔ API 연결](./001.Echo_API로_UI와_API_연결.md)
