# 013. 로그인과 회원관리

터미널·메뉴에 **접근 제어**를 두려면 회원 테이블과 로그인 흐름이 필요합니다.

> 상세 일지: `MD/2026-06-23/2026-06-23_15-00_터미널_로그인_및_회원관리_정리.md`

---

## 목표

- `t_member` + `POST /api/members/login`
- 터미널 `@@login` / `@@exit`
- 비로그인 시 명령·메뉴 제한

---

## 1. 로그인 명령

| 명령 | 동작 |
|------|------|
| `@@login` | 아이디 → `(@@login pw)` 비밀번호 (2단계) |
| `@@exit` | 로그아웃, 환영 화면 복귀 |
| `cls` | 비로그인에서도 허용 |

로그인 성공 시 ASCII 로고 + 접속 메시지.  
실패 시 환영 영역에 **「정보가 올바르지 않습니다」**.

비밀번호 입력: `type="password"`, 기록에는 `**`만 남김.

---

## 2. 비로그인 제한

허용: `cls`, `@@login` (및 로그인 입력 단계)

차단: `?`, `bd`, `gc`, `gm`, `md`, `cd`, 도움말 등

```text
사용을 하시려면 로그인을 하셔야 합니다.
```

`terminal/index.ts`의 `requiresLogin()`에서 검사.

---

## 3. 세션 저장

```text
localStorage 키: mtools-current-member
```

새로고침 후에도 로그인 유지. `@@exit` 시 삭제.

---

## 4. 백엔드 API

### 로그인

```text
POST /api/members/login
```

```json
{ "memberId": "nextia80", "password": "123456" }
```

응답: 회원 정보 (비밀번호 제외).  
DB: `t_member` — `st_member`, `st_password`, `yn_use = 'Y'` 일치.

### 회원 CRUD

| 메서드 | 경로 |
|--------|------|
| GET | `/api/members` |
| GET | `/api/members/{id}` |
| POST | `/api/members` |
| PUT | `/api/members/{id}` |
| DELETE | `/api/members/{id}` |

---

## 5. 시드 계정 (data.sql 예)

| 아이디 | 이름 | 레벨 | 비밀번호 |
|--------|------|------|----------|
| `admin` | 관리자 | 0 | `123456` |
| `nextia80` | (이름) | 1 | `123456` |

레벨 `0` = admin, `1` = 일반.  
좌측 메뉴 2~8번은 **로그인 + 레벨 ≤ 1** 일 때 표시.

> 학습/로컬: 평문 비밀번호. **운영 환경에서는 BCrypt 등 해싱 필수.**

---

## 6. 확인

```bash
curl -s -X POST http://localhost:8080/api/members/login \
  -H 'Content-Type: application/json' \
  -d '{"memberId":"nextia80","password":"123456"}'
```

터미널:

```text
@@login
(nextia80 입력)
(@@login pw) |
(비밀번호 입력)
```

→ `bd`, `gc` 등 사용 가능.
