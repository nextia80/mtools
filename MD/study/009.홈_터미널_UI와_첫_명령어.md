# 009. 홈 터미널 UI와 첫 명령어

[008](./008.게시판_CRUD와_MyBatis.md)까지 게시판·MD·API가 준비됐다면, 홈 화면에 **터미널 형태 UI**를 두고 명령어로 데이터를 조회할 수 있습니다.

> 상세 일지: `MD/2026-06-14/2026-06-14_23-59_홈_터미널_명령어_구현.md`

---

## 목표

- 상단: 명령·결과가 쌓이는 출력 영역
- 하단: 명령 입력창 (고정)
- `board`, `md`, `help` 등으로 **기존 REST API** 호출

---

## 핵심 아이디어

**백엔드에 터미널 전용 API는 없습니다.**

프론트에서 문자열을 파싱 → 기존 API를 `fetch` → 결과를 텍스트로 포맷.

```text
[입력] board list
  → executeTerminalCommand()
  → GET /api/boards
  → 목록을 터미널 줄(text)로 변환
  → 화면 출력
```

---

## 1. 화면 구조

```text
App.vue (activeView === 'home')
  └── TerminalView.vue
        ├── terminal-output    ← 스크롤 영역
        └── terminal-input-bar ← 하단 입력
```

`App.vue` 연결:

```vue
<TerminalView
  v-if="activeView === 'home'"
  :api-base-url="API_BASE_URL"
  :active="activeView === 'home'"
/>
```

---

## 2. 실행 흐름

```text
사용자 Enter
  → TerminalView.runCommand()
  → 입력 줄을 lines에 type: 'input' 추가
  → executeTerminalCommand(input, apiBaseUrl)
  → fetch*() → REST API
  → TerminalLineDraft[] 반환
  → lines에 type: 'output' | 'error' 추가
  → 스크롤 맨 아래
```

---

## 3. 초기 명령어 예

| 명령 | 동작 | API |
|------|------|-----|
| `help`, `?` | 도움말 | (로컬) |
| `clear`, `cls` | 화면 지우기 | — |
| `board`, `bd` | 게시판 목록 | `GET /api/boards` |
| `md`, `docs` | MD 파일 목록 | `GET /api/md-files` |
| `echo {"id":"a"}` | Echo 테스트 | `GET /api/echo` |

파싱 예:

```typescript
const parts = trimmed.split(/\s+/)
const command = parts[0]?.toLowerCase() ?? ''
const subCommand = parts[1]?.toLowerCase()
```

---

## 4. 출력 한 줄 타입

```typescript
export type TerminalLineDraft = {
  type: 'input' | 'output' | 'error'
  text: string
  clickValue?: string   // 클릭 시 입력창에 붙일 값
}
```

---

## 5. 확인

1. API + UI 실행
2. 홈(터미널) 메뉴 선택
3. `help` → 도움말 출력
4. `bd` 또는 `board` → 게시판 목록
5. `md` → MD 파일 목록

---

## 6. 막혔을 때

| 증상 | 확인 |
|------|------|
| 명령 무응답 | `apiBaseUrl`, API 실행 |
| API 오류 줄 | Network 탭, CORS(002) |
| 입력창 포커스 없음 | `:active="true"` prop |
