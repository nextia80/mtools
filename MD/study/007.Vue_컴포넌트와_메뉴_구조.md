# 007. Vue 컴포넌트와 메뉴 구조

기능이 늘면 `App.vue` 한 파일이 비대해집니다. mtools-ui는 **메뉴별 View 컴포넌트**로 나눕니다.

> 상세 일지: `MD/2026-06-14/2026-06-14_14-21_UI_좌측메뉴_레이아웃_정리.md`, `2026-06-14_21-20_Vue_소스_구조_설명.md`

---

## 목표

- 좌측 사이드바 + 우측 작업 영역 레이아웃 이해
- `App.vue`와 `components/*View.vue` 역할 분리

---

## 1. 화면 레이아웃

```text
전체 화면
├─ 좌측 SidebarMenu
│  ├─ mtools 브랜드
│  ├─ 메뉴 (홈, API 스웨거, API 테스트, 일자별 MD, 게시판 …)
│  └─ API 서버 상태
└─ 우측 작업 영역 (선택한 View)
   ├─ HomeView / TerminalView
   ├─ SwaggerView
   ├─ ApiTestView
   ├─ DocsView
   └─ BoardView
```

문서 탐색 UX: **목록은 왼쪽, 내용은 오른쪽**.

---

## 2. 폴더 구조

```text
mtools-ui/src/
├── main.ts              # createApp → App.vue 마운트
├── App.vue              # 전역 상태 + View 전환
├── types.ts             # MdFile, BoardPost 등 공통 타입
├── assets/              # CSS
└── components/
    ├── SidebarMenu.vue
    ├── ApiTestView.vue
    ├── SwaggerView.vue
    ├── DocsView.vue
    ├── BoardView.vue
    └── ...
```

---

## 3. App.vue 역할

- **상태**: `activeView`, MD 목록, 선택 파일, API URL 등
- **API 호출**: `loadMdFiles()`, `callApi()` 등
- **자식에 props / emit**: View는 표시·입력만, 로직은 App 또는 composable

View 전환 예:

```ts
type ActiveView = 'home' | 'swagger' | 'api' | 'docs' | 'board'

const activeView = ref<ActiveView>('home')
```

```vue
<ApiTestView v-if="activeView === 'api'" ... />
<DocsView v-else-if="activeView === 'docs'" ... />
```

---

## 4. View 컴포넌트 패턴

`ApiTestView.vue` — **표시 전용**, 로직은 부모:

```vue
<script setup lang="ts">
defineProps<{ endpoint: string; result: string; ... }>()
defineEmits<{ callApi: [] }>()
</script>
```

장점:

- 메뉴별 UI 수정이 다른 화면에 영향 없음
- props/emits로 데이터 흐름이 명확

---

## 5. main.ts 실행 흐름

```text
main.ts
  → App.vue mount
  → onMounted
       ├── loadMdFiles()
       ├── 키보드 단축키 등록
       └── 주기적 MD 목록 갱신
```

---

## 6. 점진적 분리 순서 (학습용)

처음 Echo API 테스트는 `App.vue`에 두어도 됩니다.

기능 추가 시:

```text
1. App.vue에 프로토타입
2. 동작 확인
3. components/XxxView.vue로 템플릿·스타일 이동
4. props/emits로 연결
5. types.ts에 타입 추출
```

---

## 7. 막혔을 때

| 증상 | 확인 |
|------|------|
| 메뉴 눌러도 화면 안 바뀜 | `activeView` 값, `v-if` 조건 |
| props undefined | 부모에서 `:prop="ref값"` 전달 |
| emit 안 됨 | `@call-api="handler"` 이름 일치 |
