# 006. Markdown 편집·저장

[005](./005.MD_목록과_뷰어.md)에서 **읽기**만 했다면, 이번에는 화면에서 MD를 **수정하고 디스크에 저장**합니다.

> 상세 일지: `MD/2026-06-14/2026-06-14_14-48_Markdown_리뷰_편집_저장기능.md`

---

## 목표

- **리뷰** 모드: Markdown → HTML 렌더링
- **편집** 모드: textarea에서 원문 수정
- **저장**: API로 파일에 반영

---

## 1. UI 모드

```text
[ 리뷰 ] [ 편집 ] [ 저장 ]
```

| 버튼 | 동작 |
|------|------|
| 리뷰 | `editedMdContent`를 HTML로 표시 |
| 편집 | textarea에 원문 표시 |
| 저장 | PUT API 호출 → 파일 쓰기 → 리뷰로 전환 |

---

## 3. 저장 API 호출

```ts
await fetch(
  `${API_BASE_URL}/api/md-files/content?path=${encodeURIComponent(selectedMdPath.value)}`,
  {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ content: editedMdContent.value }),
  }
)
```

저장 성공 후:

```text
mdContent 갱신
저장 완료 메시지
리뷰 모드 전환
MD 목록 새로고침 (modifiedAt 반영)
```

---

## 4. 백엔드 — PUT endpoint

```java
@PutMapping("/api/md-files/content")
public Map<String, String> updateMdFileContent(
		@RequestParam String path,
		@RequestBody MdFileContentRequest request
) throws IOException {
	mdFileService.write(path, request.content());
	return Map.of("status", "ok");
}
```

Service:

```java
public void write(String relativePath, String content) throws IOException {
	Path targetPath = resolveMdFile(relativePath);
	Files.writeString(targetPath, content, StandardCharsets.UTF_8);
}
```

`resolveMdFile` — `MD/` 밖으로 나가는 경로(`../`) 차단 등 **안전 검증** 필요.
